# Web UI 滚动容器内浮层定位修复说明

## 问题描述

当一个页面中存在**可滚动的容器**（而非 `window`），在该容器底部或右侧触发弹出类浮层（如 Tooltip、Popover、Dropdown）时，浮层可能会**溢出容器边界**，导致：

- 内容被截断、不可见
- 视觉上脱离了上下文
- 用户体验严重下降

## 根本原因

浮层组件通常基于 `window` 视口计算定位，而非其**最近的滚动祖先容器**。当滚动容器不是 `body`/`window` 时，定位算法的参考坐标系就出错了。

## 修复方案

```
浮层定位时，检测 trigger 元素最近的可滚动祖先容器（overflow: auto/scroll），
以该容器的边界（而非 window 视口）作为约束，
当浮层即将溢出时自动翻转方向（top ↔ bottom，left ↔ right）。
