CampusOS 是一个基于 Go 构建的面向高校场景的开放式社区平台（Open Community Platform）。它采用 API First、Plugin First、Event Driven 的设计理念，以身份认证、权限管理、资源管理、事件总线和开放 API 为核心能力，通过插件系统实现业务扩展，并通过统一的 Integration 层接入 NapCat、AstrBot、MCP 等第三方生态，为开发者提供完整的 SDK、插件市场和开发者中心，使论坛、AI 助手、机器人、校园应用等都能够以插件的形式运行，最终构建一个可持续发展的校园数字生态系统，而不是一个仅具备发帖和评论功能的传统论坛

## 项目背景

传统论坛系统（如 Discuz、Flarum）在设计之初未充分考虑现代化云原生与高度集成需求，导致了诸多痛点：

| 痛点 | 描述 |
|:---|:---|
| **生态封闭** | 缺乏灵活的插件生态机制，二次开发成本极高 |
| **AI 时代脱节** | 难以优雅接入 LLM 能力与现代机器人（如微信、QQ、Discord 机器人） |
| **权限僵化** | 硬编码的权限模型无法满足多维度、细粒度的校园社区管理需求 |
| **扩展困难** | 功能堆砌导致系统臃肿，缺乏模块化与微服务解耦能力 |
| **数据孤岛** | 校园内各系统之间无法打通，信息流转效率低下 |

CampusOS 的诞生，正是为了解决这些根本性问题。

## 项目定位

**CampusOS 不是在做一个传统的 BBS 业务系统，而是在打造一个极其灵活的"社区操作系统 (Community OS)"。**

官方论坛仅仅是运行在这个操作系统上的一个基础 App。

用操作系统来类比：

| 操作系统概念 | CampusOS 对应 |
|:---|:---|
| **内核 (Kernel)** | Core + Platform（身份、权限、事件总线等原子能力） |
| **系统调用 (Syscall)** | Host API（插件调用核心能力的统一接口） |
| **驱动程序 (Driver)** | Integration 层（NapCat / AstrBot 等外部适配器） |
| **应用程序 (App)** | 插件（论坛、AI 助手、机器人、校园应用） |
| **应用商店 (App Store)** | 插件市场 |
| **开发者工具链 (SDK)** | 多语言 SDK + 开发者中心 + 沙盒环境 |

## 核心理念

### 🔮 万物皆事件 (Everything is an Event)

所有核心行为必须通过 Event Bus 广播。插件通过监听事件来改变系统行为，而非修改核心代码。这确保了：

- **零侵入扩展**：任何业务逻辑都可以通过插件事件订阅实现，无需改动核心代码
- **解耦通信**：各服务与插件之间通过事件松耦合通信
- **完整可追溯**：所有系统行为都有事件记录，支持审计与回放

### 🏗️ 双运行时插件架构 (Dual-Runtime)

系统不绑定特定插件运行时，而是根据插件的 Manifest 自动调度：

| 运行时 | 适用场景 | 核心优势 |
|:---|:---|:---|
| **Wasm Runtime** | 事件监听器、内容审核、AI 后处理、Markdown 扩展 | 极度轻量、严格沙箱安全、无状态、易于分发 |
| **gRPC Runtime** | IM 适配器、RAG 服务、长连接网关、定时任务 | 完整系统能力、丰富网络 I/O、适合长期驻留 |

### 🤖 AI 一等公民 (AI as a First-Class Citizen)

AI 并非外挂功能，而是系统的"一等公民"操作者：

- **原生 MCP 协议支持**：系统核心 API 封装为 MCP Server，AI Agent 可直接调用
- **统一大模型网关**：封装 Claude、DeepSeek、Gemini 等 API，切断对具体厂商的依赖
- **AI 代理机制**：AI 可作为管理员、创作者或客服深度参与社区运转

### 🔒 默认安全 (Secure by Default)

- 严苛的插件权限清单（Manifest）与 WebAssembly/gRPC 沙箱隔离
- 全局内置防 XSS、防 CSRF、防 SQL 注入中间件
- 细粒度请求频率限制 (Rate Limiting)

## 核心特性

### 🔐 身份与权限

- 多端统一登录（OAuth2、LDAP、校园网 SSO）
- RBAC + ABAC 混合权限模型，支持资源级和字段级细粒度授权
- User 与 Account 实体解耦，支持多账号绑定

### 📡 事件驱动架构

- 基于 NATS 的高性能 Event Bus
- 采用 CNCF CloudEvents 1.0 规范
- `.before`（同步拦截）与 `.after`（异步通知）双模式事件
- At-least-once 投递保障 + 死信队列 + 指数退避重试

### 🔌 插件系统

- 统一 Plugin Manager，Wasm / gRPC 双运行时自动调度
- 标准化 Manifest 规范，显式声明运行环境、事件订阅与权限
- 统一 Host API，插件可反向调用核心能力

### 🌐 开放生态

- 标准 RESTful API，统一响应体封装
- 强制 URL 路径版本号隔离
- 第一方 JWT + 第三方 API Key + HMAC 签名双认证体系
- 多语言 SDK（Go / TypeScript / Rust）

### 🤖 AI 能力

- 统一大模型网关（AIClient.ChatCompletion()）
- MCP Server 协议集成
- AI Agent 可直接操作论坛底层数据

### 📱 第三方集成

- 统一 Message 协议，标准化外部消息映射
- NapCat (QQ)、AstrBot 等作为独立 gRPC Adapter 运行
- Webhook 机制，支持 Zapier、飞书等外部系统联动

## 快速开始

### 环境要求

- Go 1.22+
- PostgreSQL 16+
- Redis 7+
- NATS 2.x
- Docker & Docker Compose（推荐）

### 一键启动

```bash
# 克隆项目
git clone https://github.com/your-org/CampusOS.git
cd CampusOS

# 使用 Docker Compose 一键启动
docker compose up -d

# 访问
# API:       http://localhost:8080/api/v1
# Web UI:    http://localhost:3000
# Grafana:   http://localhost:3001
```

### 本地开发

```bash
# 安装依赖
go mod download

# 运行数据库迁移
go run cmd/migrate/main.go up

# 启动服务
go run cmd/server/main.go

# 运行测试
go test ./...
```
