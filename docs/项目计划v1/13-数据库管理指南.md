## 数据库概览

CampusOS 使用三个核心数据存储服务：

| 服务 | 用途 | 端口 | 默认连接信息 |
|:---|:---|:---|:---|
| **PostgreSQL 16** | 核心业务数据持久化 | 5432 | host=localhost, user=campusos, db=campusos, password=campusos_dev |
| **Redis 7** | 缓存、会话、限流 | 6379 | host=localhost, 无密码 |
| **NATS 2** | 事件总线（消息队列） | 4222 | nats://localhost:4222 |

### 连接架构

```
Go Server
  ├── PostgreSQL（pgx 连接池，最大 25 连接）
  │     ├── users / accounts / sessions
  │     ├── categories / threads / posts / tags / likes
  │     └── audit_logs / notifications / configurations
  ├── Redis（缓存层）
  │     ├── 会话缓存
  │     ├── 热点数据缓存
  │     └── 限流计数
  └── NATS（事件总线）
        ├── user.created / user.updated
        ├── thread.created / thread.updated / thread.deleted
        ├── post.created
        └── category.created
```

## PostgreSQL 管理

### 基本连接

```bash
# 使用 psql 命令行连接
PGPASSWORD=campusos_dev psql -h localhost -U campusos -d campusos

# 或使用 sudo 切换到 postgres 超级用户
sudo -u postgres psql
```

### 常用 psql 命令

```sql
-- 查看所有数据库
\l

-- 切换到 campusos 数据库
\c campusos

-- 查看所有表
\dt

-- 查看表结构
\d users
\d threads
\d posts

-- 查看索引
\di

-- 查看表大小
\dt+

-- 退出
\q
```

### 通过 SQL 操作数据

```sql
-- 查看所有用户
SELECT id, username, nickname, email, status, created_at FROM users;

-- 查看所有帖子
SELECT id, title, author_name, category_id, status, view_count, created_at FROM threads;

-- 查看帖子及其回复数
SELECT t.id, t.title, t.view_count, t.reply_count, 
       (SELECT COUNT(*) FROM posts p WHERE p.thread_id = t.id) AS actual_replies
FROM threads t;

-- 查看所有版块
SELECT id, name, slug, description, thread_count, post_count FROM categories;
```

### 环境变量配置

所有数据库连接信息通过环境变量配置：

| 环境变量 | 默认值 | 说明 |
|:---|:---|:---|
| `DATABASE_DSN` | `postgres://campusos:campusos_dev@localhost:5432/campusos?sslmode=disable` | PostgreSQL 连接字符串 |
| `REDIS_ADDR` | `localhost:6379` | Redis 地址 |
| `NATS_URL` | `nats://localhost:4222` | NATS 地址 |
| `JWT_SECRET` | `campusos-dev-secret-key...` | JWT 签名密钥 |
| `SERVER_PORT` | `8080` | API 服务端口 |

## Redis 管理

### 基本连接

```bash
# 连接 Redis
redis-cli

# 测试连接

# 输出: PONG
```

### 常用 Redis 命令

```bash
# 查看所有键
KEYS *

# 查看服务器信息
INFO

# 查看内存使用
INFO memory

# 查看连接数
INFO clients

# 查看数据库大小
DBSIZE

# 清空当前数据库（谨慎！）
FLUSHDB

# 清空所有数据库（非常谨慎！）

```

### Redis 在 CampusOS 中的用途

| 用途 | Key 模式 | 说明 |
|:---|:---|:---|
| 会话缓存 | `session:{user_id}` | JWT Refresh Token |
| 热点数据 | `cache:thread:{id}` | 帖子详情缓存 |
| 限流计数 | `ratelimit:{ip}:{path}` | 请求频率限制 |
| 用户状态 | `user:online:{user_id}` | 在线状态 |

## NATS 管理

### 启动 NATS

```bash
# 使用 Docker Compose 启动
cd ~/bbs/bbs01/CampusOS
docker compose up -d nats

# 验证启动
curl -s http://localhost:8222/varz | jq .
```

### NATS 监控

| 地址 | 说明 |
|:---|:---|
| http://localhost:8222 | NATS 监控首页 |
| http://localhost:8222/varz | 服务器变量（连接数、消息数） |
| http://localhost:8222/connz | 连接详情 |
| http://localhost:8222/subsz | 订阅详情 |
| http://localhost:8222/routez | 路由详情（集群模式） |

### NATS 在 CampusOS 中的事件

| 事件类型 | 触发时机 | 数据 |
|:---|:---|:---|
| `user.created` | 用户注册 | 用户完整信息 |
| `thread.created` | 帖子创建 | 帖子完整信息 |
| `thread.updated` | 帖子更新 | 帖子完整信息 |
| `thread.deleted` | 帖子删除 | 帖子完整信息 |
| `post.created` | 回复创建 | 回复完整信息 |
| `category.created` | 版块创建 | 版块完整信息 |

### NATS 不可用时

系统自动回退到内存事件总线，所有功能正常运行，但：
- 事件不持久化（进程重启丢失）
- 不支持集群
- 日志中会显示：`⚠️ NATS 连接失败，回退到内存事件总线`

## 表结构说明

### 身份域（Identity）

#### users — 用户表

| 字段 | 类型 | 说明 |
|:---|:---|:---|

| status | VARCHAR(20) | active/suspended/deactivated |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |
| deleted_at | TIMESTAMP | 软删除标记 |

#### accounts — 账号凭据表

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| id | BIGINT (PK) | 同 users.id |
| user_id | BIGINT | 关联用户 |
| type | VARCHAR(20) | email/phone/oauth |
| identifier | VARCHAR(255) | 邮箱/手机号 |

| 字段 | 类型 | 说明 |

| id | BIGINT (PK) | 会话 ID |

| refresh_token | VARCHAR(255) | Refresh Token |
| device_type | VARCHAR(20) | web/mobile/desktop |
| expires_at | TIMESTAMP | 过期时间 |

#### categories — 版块表

| 字段 | 类型 | 说明 |
|:---|:---|:---|

| thread_count | BIGINT | 帖子计数 |

| view_count | BIGINT | 浏览计数 |
| reply_count | BIGINT | 回复计数 |

## 常用 SQL 操作

### 用户管理

```sql
-- 查看所有活跃用户
SELECT id, username, nickname, email, status, created_at 
FROM users WHERE deleted_at IS NULL ORDER BY created_at DESC;

-- 查看用户及其账号信息
SELECT u.id, u.username, u.email, a.type, a.verified
FROM users u
LEFT JOIN accounts a ON u.id = a.user_id AND a.deleted_at IS NULL
WHERE u.deleted_at IS NULL;

-- 封禁用户
UPDATE users SET status = 'suspended', updated_at = NOW() WHERE id = <user_id>;

-- 解封用户
UPDATE users SET status = 'active', updated_at = NOW() WHERE id = <user_id>;
```

### 帖子管理

```sql
-- 查看帖子列表（按最新回复排序）
SELECT id, title, author_name, status, view_count, reply_count, like_count, created_at
FROM threads WHERE deleted_at IS NULL AND status = 'published'
ORDER BY is_pinned DESC, created_at DESC LIMIT 20;

-- 搜索帖子
SELECT id, title, author_name, created_at
FROM threads 
WHERE deleted_at IS NULL AND (title ILIKE '%关键词%' OR content ILIKE '%关键词%')
ORDER BY created_at DESC;

-- 查看帖子回复
SELECT p.id, p.author_name, p.content, p.floor_number, p.created_at
FROM posts p
WHERE p.thread_id = <thread_id> AND p.deleted_at IS NULL
ORDER BY p.floor_number;

-- 软删除帖子
UPDATE threads SET deleted_at = NOW() WHERE id = <thread_id>;

-- 置顶帖子
UPDATE threads SET is_pinned = TRUE, updated_at = NOW() WHERE id = <thread_id>;
```

### 版块管理

```sql
-- 查看版块列表
SELECT id, name, slug, description, thread_count, post_count, is_closed
FROM categories WHERE deleted_at IS NULL ORDER BY sort_order;

-- 创建版块
INSERT INTO categories (id, name, slug, description, created_at, updated_at)
VALUES (<snowflake_id>, '版块名称', 'slug', '描述', NOW(), NOW());
```

### 统计查询

```sql
-- 系统概览
SELECT 
  (SELECT COUNT(*) FROM users WHERE deleted_at IS NULL) AS total_users,
  (SELECT COUNT(*) FROM threads WHERE deleted_at IS NULL) AS total_threads,
  (SELECT COUNT(*) FROM posts WHERE deleted_at IS NULL) AS total_posts,
  (SELECT COUNT(*) FROM categories WHERE deleted_at IS NULL) AS total_categories;

-- 每日新增用户
SELECT DATE(created_at) AS date, COUNT(*) AS new_users
FROM users WHERE deleted_at IS NULL
GROUP BY DATE(created_at) ORDER BY date DESC LIMIT 30;

-- 热门帖子（按浏览量）
SELECT id, title, view_count, reply_count, like_count
FROM threads WHERE deleted_at IS NULL AND status = 'published'
ORDER BY view_count DESC LIMIT 10;

-- 最近审计日志
SELECT trace_id, action, resource, resource_id, created_at
FROM audit_logs ORDER BY created_at DESC LIMIT 20;
```

## 迁移管理

### 执行迁移

```bash
# 执行迁移（创建表）
cd ~/bbs/bbs01/CampusOS
PGPASSWORD=campusos_dev psql -h localhost -U campusos -d campusos \
  -f migrations/000001_init_schema.up.sql
```

## 备份与恢复

### 备份

```bash
# 完整备份
pg_dump -h localhost -U campusos -d campusos > campusos_backup_$(date +%Y%m%d).sql

# 仅备份数据（不含表结构）
pg_dump -h localhost -U campusos -d campusos --data-only > campusos_data.sql

# 仅备份表结构
pg_dump -h localhost -U campusos -d campusos --schema-only > campusos_schema.sql

# 备份单个表
pg_dump -h localhost -U campusos -d campusos -t users > users_backup.sql
```

### 恢复

```bash
# 恢复完整备份
PGPASSWORD=campusos_dev psql -h localhost -U campusos -d campusos < campusos_backup.sql

# 恢复到新数据库
createdb -h localhost -U campusos campusos_restored
PGPASSWORD=campusos_dev psql -h localhost -U campusos -d campusos_restored < campusos_backup.sql
```

### 自动备份脚本（推荐）

```bash
#!/bin/bash
# scripts/backup.sh
BACKUP_DIR="/var/backups/campusos"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR
pg_dump -h localhost -U campusos -d campusos | gzip > "$BACKUP_DIR/campusos_$DATE.sql.gz"
# 保留最近 30 天
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete
echo "Backup completed: campusos_$DATE.sql.gz"
```

## 故障排查

### PostgreSQL 连接失败

```bash
# 检查 PostgreSQL 是否运行
sudo systemctl status postgresql

# 检查端口是否监听
ss -tlnp | grep 5432

# 尝试连接
PGPASSWORD=campusos_dev psql -h localhost -U campusos -d campusos -c "SELECT 1;"

# 查看日志
sudo tail -f /var/log/postgresql/postgresql-16-main.log
```

### 表不存在

```bash
# 检查是否有表
PGPASSWORD=campusos_dev psql -h localhost -U campusos -d campusos -c "\dt"

# 如果没有表，执行迁移
make migrate-up
```

### 连接数过多

```sql
-- 查看当前连接数
SELECT COUNT(*) FROM pg_stat_activity;

-- 查看连接详情
SELECT pid, usename, application_name, client_addr, state, query
FROM pg_stat_activity WHERE datname = 'campusos';

-- 终止空闲连接
SELECT pg_terminate_backend(pid) FROM pg_stat_activity
WHERE datname = 'campusos' AND state = 'idle' AND query_start < NOW() - INTERVAL '10 minutes';
```

### Redis 连接失败

```bash
# 检查 Redis 是否运行
redis-cli ping

# 查看 Redis 日志
sudo tail -f /var/log/redis/redis-server.log

# 检查端口
ss -tlnp | grep 6379
```

### NATS 连接失败

```bash
# 检查 NATS 容器状态
docker ps | grep nats

# 重启 NATS
docker compose restart nats

# 检查 NATS 日志
docker logs campusos-nats

# 测试连接
curl -s http://localhost:8222/varz | jq .connections
```
