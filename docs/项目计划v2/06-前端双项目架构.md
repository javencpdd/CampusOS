## 一、背景与目标

### 当前问题

当前项目只有 `web/` 一个前端项目，包含用户界面（首页/登录/注册/帖子）和管理后台（已部分创建）。两者混合在同一项目中存在以下问题：

| 问题 | 说明 |
|:---|:---|
| 依赖冗余 | 管理后台引入 Element Plus 全量组件，用户界面不需要 |
| 路由复杂 | `/admin/*` 和 `/threads/*` 混在一起，路由守卫逻辑复杂 |
| 权限耦合 | 管理后台需要 admin 权限，用户界面不需要，中间件混在一起 |
| 部署粒度 | 无法独立部署/更新管理后台，每次更新都影响用户 |
| 构建体积 | 用户界面打包了管理后台的代码，增加不必要的体积 |

### 目标

将前端拆分为两个独立项目，共用同一套后端 API：

```
CampusOS/
├── web/          # 用户前台（现有项目，重命名后保留）
├── admin/        # 管理后台（新建项目）
└── internal/     # 后端 API（不变，两个前端共用）
```

## 二、架构设计

### 整体架构

```
┌─────────────────┐     ┌─────────────────┐
│   用户前台       │     │   管理后台       │
│   web/          │     │   admin/        │
│   :3000         │     │   :3001         │
│                 │     │                 │
│  · 首页         │     │  · 仪表盘       │
│  · 登录/注册    │     │  · 用户管理     │
│  · 帖子列表/详情│     │  · 帖子管理     │
│  · 发帖         │     │  · 版块管理     │
│  · 个人中心     │     │  · 插件管理     │
│  · 用户主页     │     │  · 事件日志     │
└────────┬────────┘     └────────┬────────┘
         │                       │
         └───────────┬───────────┘
                     │
         ┌───────────▼───────────┐
         │     后端 API           │
         │     :8080              │
         │                       │
         │  /api/v1/*            │
         │  (19+33 个端点)       │
         └───────────────────────┘
```

### 共用后端接口

两个前端通过 Vite Proxy 转发请求到同一个后端：

**web/vite.config.ts:**
```ts
proxy: {
  '/api': { target: 'http://localhost:8080', changeOrigin: true }
}
```

**admin/vite.config.ts:**
```ts
proxy: {
  '/api': { target: 'http://localhost:8080', changeOrigin: true }
}
```

### 认证方式

两个前端共用同一套 JWT 认证：
- `Authorization: Bearer <access_token>`
- Token 存储在各自的 `localStorage` 中
- 管理后台额外验证用户角色（admin/moderator）

## 五、API 分层

### 用户前台使用的 API

| 模块 | 端点 | 说明 |
|:---|:---|:---|
| 认证 | `POST /auth/register` | 注册 |
| 认证 | `POST /auth/login` | 登录 |
| 认证 | `GET /auth/me` | 当前用户 |
| 用户 | `GET /users`, `GET /users/:id` | 查看用户 |
| 用户 | `PUT /users/:id` | 编辑个人信息 |
| 帖子 | `GET/POST /threads` | 列表/创建 |
| 帖子 | `GET/PUT/DELETE /threads/:id` | 详情/编辑/删除 |
| 回复 | `GET/POST /threads/:id/posts` | 回复列表/创建 |
| 版块 | `GET /categories` | 版块列表 |

### 管理后台额外使用的 API

| 模块 | 端点 | 说明 |
|:---|:---|:---|
| 用户管理 | `POST /users/:id/suspend` | 封禁用户 |
| 用户管理 | `POST /users/:id/activate` | 解封用户 |
| 帖子管理 | `POST /threads/:id/pin` | 置顶帖子 |
| 帖子管理 | `POST /threads/:id/unpin` | 取消置顶 |
| 帖子管理 | `POST /threads/:id/lock` | 锁定帖子 |
| 帖子管理 | `POST /threads/:id/unlock` | 解锁帖子 |
| 版块管理 | `POST /categories` | 创建版块 |
| 版块管理 | `PUT /categories/:id` | 编辑版块 |
| 版块管理 | `DELETE /categories/:id` | 删除版块 |
| 插件管理 | `GET /plugins` | 插件列表 |
| 插件管理 | `POST /plugins/:name/enable` | 启用插件 |
| 插件管理 | `POST /plugins/:name/disable` | 禁用插件 |
| 插件管理 | `DELETE /plugins/:name` | 卸载插件 |
| 事件 | `GET /events` | 事件历史 |

## 七、部署方案

### 开发环境

```bash
# 后端
cd CampusOS && go run cmd/server/main.go

# 用户前台（:3000）
cd web && pnpm dev

# 管理后台（:3001）
cd admin && pnpm dev --port 3001
```

### 生产环境（Nginx）

```nginx
# 用户前台
server {
    listen 80;
    server_name forum.campusos.io;
    root /var/www/web/dist;
    location /api/ {
        proxy_pass http://localhost:8080;
    }
}

# 管理后台
server {
    listen 80;
    server_name admin.campusos.io;
    root /var/www/admin/dist;
    location /api/ {
        proxy_pass http://localhost:8080;
    }
}
```

### Docker Compose 扩展

```yaml
services:
  web:
    build: ./web
    ports: ["3000:80"]

  admin:
    build: ./admin
    ports: ["3001:80"]

  api:
    build: .
    ports: ["8080:8080"]
```

## 八、共用代码管理

为避免代码重复，创建共享类型包：

```
shared/
├── types/
│   ├── user.ts          # User / LoginRequest / LoginResponse
│   ├── thread.ts        # Thread / CreateThreadRequest / ThreadListFilter
│   ├── post.ts          # Post / CreatePostRequest
│   ├── category.ts      # Category / CreateCategoryRequest
│   ├── plugin.ts        # Plugin / PluginManifest
│   ├── event.ts         # Event / EventMessage
│   └── response.ts      # ApiResponse / Pagination
└── package.json         # name: "@campusos/shared"
```

在各前端项目中引用：

```ts
// web/src/api/index.ts
import type { User, Thread, ApiResponse } from '@campusos/shared'

// admin/src/api/index.ts
import type { User, Thread, Plugin, ApiResponse } from '@campusos/shared'
```
