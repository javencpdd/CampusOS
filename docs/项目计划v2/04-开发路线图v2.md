## v0.2 — 权限与插件基础

**目标**：建立 RBAC 权限系统，实现 gRPC 插件运行时和 Host API 基础

### P0（必须完成）

| 任务 | 说明 |
|:---|:---|
| **RBAC 角色表** | roles / user_roles 数据库表 |
| **基础角色** | admin / moderator / member / guest |
| **管理后台 API** | 用户封禁/解封、帖子置顶/精华、角色分配 |
| **Plugin Manager 核心** | 插件注册、Manifest 解析、生命周期管理 |
| **gRPC Runtime** | 插件进程启动/停止、健康检查、自动重启 |
| **Host API 基础** | Identity / Data / Event / Notification / Storage 接口 |
| **插件 Manifest 解析** | plugin.yaml 读取和验证 |
| **补全 API** | 登出、刷新 Token、置顶、封禁等 |

### P1（应该完成）

| 任务 | 说明 |
|:---|:---|
| **Redis 缓存集成** | 热点数据缓存、会话缓存 |
| **插件 API Key** | 插件认证和权限控制 |
| **.before/.after 事件** | 同步拦截 + 异步通知 |
| **示例 gRPC 插件** | 统计分析插件、日志插件 |
| **Webhook 系统** | 用户配置 Webhook 接收事件 |

### 验收标准

- [ ] 管理员可以封禁/解封用户
- [ ] 管理员可以置顶/精华帖子
- [ ] 插件可以注册、启动、接收事件
- [ ] Host API 可以查询数据、发布事件
- [ ] 至少 1 个 gRPC 插件正常工作
- [ ] 补全所有缺失 API 端点

## v0.3 — Wasm 安全隔离

**目标**：引入 Wasm 运行时，实现轻量级安全插件

| 任务 | 说明 |
|:---|:---|
| **Wasm Runtime** | 基于 Wazero 的 Wasm 沙箱 |
| **Wasm Host Functions** | Host API 的 Wasm 绑定 |
| **对象池管理** | Wasm 实例复用和生命周期 |
| **资源限制** | 内存上限、执行超时 |
| **三层权限模型** | API 级 + 资源级 + 字段级 |
| **插件存储** | SQLite 独立存储 + PostgreSQL 共享存储 |
| **Go SDK** | Wasm + gRPC 双运行时 SDK |
| **CLI 工具** | 插件创建、构建、测试 |

### 验收标准

- [ ] Wasm 插件安全运行，不影响核心系统
- [ ] 三层权限模型正常工作
- [ ] Go SDK 完整可用
- [ ] 至少 2 个 Wasm 示例插件正常工作
