> **文档类型**：插件实例参考文档

## 一、当前实现方式

**帖子管理目前是通过核心层（Community Domain）直接实现的，而非插件。** 这符合 CampusOS 的设计理念："官方论坛仅仅是运行在操作系统上的一个基础 App"。核心层提供了帖子的原子操作，插件通过事件订阅来扩展和改变帖子行为。

## 二、帖子管理完整实现流程

### 2.1 架构分层

```
HTTP 请求
  │
  ▼
┌─────────────────────────────────┐
│  Handler 层 (thread_handler.go) │  ← 请求解析、参数验证、响应格式化
│  POST /threads                  │
│  GET  /threads                  │
│  PUT  /threads/:id              │
│  DELETE /threads/:id            │
│  POST /threads/:id/pin          │
│  POST /threads/:id/lock         │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  Service 层 (thread_service.go) │  ← 业务逻辑、权限检查、缓存、事件发布
│  CreateThread()                 │
│  ListThreads()  [缓存: 5min]    │
│  UpdateThread()                 │
│  DeleteThread()                 │
│  PinThread() / LockThread()     │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  Repository 层                  │  ← 数据持久化
│  ThreadRepository 接口          │
│  ├── PgThreadRepository (PG)    │
│  └── MemoryThreadRepository     │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  Event Bus (NATS + Memory)      │  ← 事件广播
│  thread.created                 │
│  thread.updated                 │
│  thread.deleted                 │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│  插件系统（监听事件扩展行为）      │
│  ├── 内容审核插件 (Wasm)         │
│  ├── 统计分析插件 (Wasm)         │
│  ├── 通知推送插件 (Wasm)         │
│  └── IM 同步插件 (gRPC)          │
└─────────────────────────────────┘
```

### 2.2 创建帖子流程

```
1. 用户点击"发布帖子"
       │
       ▼
2. HTTP POST /api/v1/threads
   Headers: Authorization: Bearer <JWT>
   Body: { "title": "...", "content": "...", "category_id": "..." }
       │
       ▼
3. Gateway Layer 中间件链
   ├── TraceID 注入
   ├── CORS 检查
   ├── JWT 鉴权验证 → 提取 user_id
   └── 路由分发到 ThreadHandler.CreateThread
       │
       ▼
4. ThreadHandler.CreateThread (handler)
   ├── 从 Header 获取 X-User-ID（JWT 中间件注入）
   ├── ShouldBindJSON 解析请求体
   └── 调用 ThreadService.CreateThread
       │
       ▼
5. ThreadService.CreateThread (service)
   ├── 生成 UUID 作为帖子 ID
   ├── 构建 Thread 领域实体
   │   ├── Status = "published"
   │   ├── IsPinned = false
   │   ├── IsLocked = false
   │   └── ViewCount = 0
   ├── ThreadRepository.Create → 写入 PostgreSQL
   ├── 清除列表缓存（invalidateListCache）
   └── EventBus.Publish("thread.created") → NATS/Memory
       │
       ▼
6. EventBus 分发到订阅者
   ├── 核心层：日志记录
   └── 插件管理器：DispatchEvent 到订阅的插件
       │
       ▼
7. 返回 HTTP 201 + 帖子数据
```

### 2.3 查询帖子列表流程（带缓存）

```
1. HTTP GET /api/v1/threads?page=1&page_size=20
       │
       ▼
2. ThreadHandler.ListThreads
   ├── 解析查询参数（page, page_size, category_id, keyword, status）
   └── 构建 ThreadListFilter
       │
       ▼
3. ThreadService.ListThreads
   ├── 生成缓存键: "threads:list:1:20:published"
   ├── 检查缓存条件（无 keyword/categoryID/authorID 时）
   │
   ├── 【缓存命中】→ 直接返回（跳过数据库查询）
   │   └── log: "📦 缓存命中: threads:list:1:20:published"
   │
   └── 【缓存未命中】→
       ├── ThreadRepository.List → 查询 PostgreSQL
       ├── 写入缓存（TTL 5分钟）
       └── 返回结果
       │
       ▼
4. 分页信息封装
   ├── Page, PageSize, Total, TotalPages
   └── response.List(c, threads, pagination)
```

### 2.4 管理员操作流程（置顶/锁定/删除）

```
1. 管理员点击"置顶"
       │
       ▼
2. HTTP POST /api/v1/threads/:id/pin
   Headers: Authorization: Bearer <JWT>
       │
       ▼
3. Gateway 中间件链
   ├── JWT 鉴权
   └── RequirePermission(permSvc, "thread", "pin")
       ├── 查询用户角色 → 展开权限列表
       ├── 匹配 "thread:pin" 权限
       ├── admin/moderator 角色有此权限 → 允许
       └── member 角色无此权限 → 返回 403
       │
       ▼
4. ThreadService.PinThread
   ├── ThreadRepository.GetByID → 获取帖子
   ├── thread.IsPinned = true
   ├── ThreadRepository.Update → 更新 PostgreSQL
   └── 返回更新后的帖子
```

## 三、插件如何扩展帖子管理

### 3.1 事件驱动扩展（当前已实现）

插件通过订阅事件来扩展帖子行为，无需修改核心代码：

```yaml
# plugin.yaml - 内容审核插件示例
apiVersion: campusos/v1alpha1
metadata:
  name: content-moderator
  version: "1.0.0"
  displayName: "内容审核插件"
spec:
  runtime: wasm
  wasm:
    entrypoint: moderator.wasm
    memoryLimit: 128MB
    timeout: 30s
permissions:
  events:
    subscribe:
      - thread.created.before    # 同步拦截，可阻止发布
      - thread.created.after     # 异步通知，记录日志
      - post.created.before      # 同步拦截回复
```

**审核插件工作流程：**

```
用户发帖 → thread.created.before 事件
       │
       ▼
Plugin Manager 分发到内容审核插件 (Wasm)
       │
       ▼
插件检查：
  ├── 敏感词检测 → 发现违规 → 返回 { allowed: false, message: "包含敏感词" }
  ├── 垃圾内容检测 → 可疑 → 返回 { allowed: false, message: "疑似垃圾内容" }
  └── 检测通过 → 返回 { allowed: true }
       │
       ▼
核心层根据插件响应决定：
  ├── allowed=false → 阻止帖子发布，返回错误
  └── allowed=true → 继续发布流程
```

### 3.2 Host API 扩展（计划中，v0.3）

插件可以通过 Host API 主动查询和操作帖子数据：

```go
// 插件通过 Host API 查询帖子
threads, err := hostAPI.QueryThreads(ctx, &pb.QueryThreadsRequest{
    CategoryId: "tech",
    Status:     "published",
    PageSize:   10,
})

// 插件通过 Host API 更新帖子元数据
err := hostAPI.UpdateThreadMetadata(ctx, &pb.UpdateMetadataRequest{
    ThreadId: "xxx",
    Metadata: map[string]string{
        "ai_summary": "这是一篇关于Go语言的文章...",
        "sentiment":  "positive",
    },
})
```

### 3.3 完整的插件扩展示例：AI 帖子摘要插件

```yaml
# plugin.yaml
apiVersion: campusos/v1alpha1
metadata:
  name: ai-summary
  version: "1.0.0"
  displayName: "AI 帖子摘要"
  description: "自动生成帖子摘要"
spec:
  runtime: grpc
  grpc:
    entrypoint: ./ai-summary-server
    port: 9001
    healthCheck:
      interval: 10s
      timeout: 5s
permissions:
  events:
    subscribe:
      - thread.created.after    # 帖子创建后自动生成摘要
  host_api:
    - thread:read              # 读取帖子内容
    - thread:write             # 写入摘要到帖子元数据
```

**插件工作流程：**

```
帖子创建 → thread.created.after 事件
       │
       ▼
AI 摘要插件收到事件（gRPC 长连接）
       │
       ▼
1. 通过 Host API 获取帖子完整内容
2. 调用 LLM 生成摘要
3. 通过 Host API 将摘要写入帖子 metadata
4. 发布 thread.updated 事件（触发缓存失效）
```

                    │  draft   │  草稿

                         │ 提交
                         ▼
                    ┌──────────────┐

         │                │

    ┌─────────┐     ┌───────────┐

## 五、数据库表结构（threads 表）

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| id | BIGINT | 雪花算法主键 |
| title | VARCHAR(255) | 帖子标题 |
| content | TEXT | 帖子内容（Markdown） |
| content_format | VARCHAR(20) | 内容格式，默认 markdown |
| author_id | BIGINT | 作者 ID |
| author_name | VARCHAR(64) | 作者名（冗余） |
| category_id | BIGINT | 所属版块 |
| status | VARCHAR(20) | 状态（draft/pending_review/published/archived） |
| is_pinned | BOOLEAN | 是否置顶 |
| is_locked | BOOLEAN | 是否锁定 |
| is_highlighted | BOOLEAN | 是否精华 |
| view_count | BIGINT | 浏览数 |
| reply_count | BIGINT | 回复数 |
| like_count | BIGINT | 点赞数 |
| tags | TEXT[] | 标签数组（GIN 索引） |
| metadata | JSONB | 扩展元数据（插件可写入） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |
| deleted_at | TIMESTAMP | 软删除时间 |

## 六、API 端点清单

| 端点 | 方法 | 认证 | 权限 | 说明 |
|:---|:---|:---|:---|:---|
| `/api/v1/threads` | GET | ❌ | — | 帖子列表（公开，支持搜索/分页/过滤） |

| `/api/v1/threads/:id/lock` | POST | ✅ JWT | thread:pin | 锁定帖子 |
| `/api/v1/threads/:id/unlock` | POST | ✅ JWT | thread:pin | 解锁帖子 |
| `/api/v1/threads/:id/posts` | GET | ❌ | — | 帖子回复列表 |
| `/api/v1/threads/:id/posts` | POST | ✅ JWT | — | 创建回复 |
