## 快速开始

### 5 分钟创建你的第一个插件

```bash
# 1. 安装 CLI 工具
go install github.com/campusos/cli@latest

# 2. 创建插件项目
campusos plugin init my-first-plugin --runtime wasm --lang go

# 3. 进入项目目录
cd my-first-plugin

# 4. 查看生成的文件
tree .
.
├── plugin.yaml          # 插件清单
├── main.go              # 入口文件
├── main_test.go         # 测试文件
├── go.mod
├── go.sum
└── README.md

# 5. 编写插件逻辑
# 编辑 main.go

# 6. 构建插件
campusos plugin build

# 7. 本地测试
campusos plugin test

# 8. 安装到 CampusOS
campusos plugin install ./dist/my-first-plugin.wasm
```

## 插件类型选择

### 选择决策树

```
你的插件需要什么能力？
    │
    ├── 仅需监听事件并做轻量处理
    │   (内容审核、数据统计、格式转换)
    │   └── → 选择 Wasm 插件
    │
    ├── 需要长连接 / 外部网络访问
    │   (IM 适配器、WebSocket 网关)
    │   └── → 选择 gRPC 插件
    │
    ├── 需要大量计算资源
    │   (RAG 服务、文档索引)
    │   └── → 选择 gRPC 插件
    │
    ├── 需要定时任务
    │   (数据同步、定期清理)
    │   └── → 选择 gRPC 插件
    │
    └── 需要调用外部 API
        (Webhook、第三方集成)
        └── → 选择 gRPC 插件
```

### 对比总结

| 维度 | Wasm 插件 | gRPC 插件 |
|:---|:---|:---|
| **开发语言** | Go / Rust / TypeScript | 任意支持 gRPC 的语言 |
| **构建产物** | `.wasm` 文件 | 可执行文件 + `plugin.yaml` |
| **运行方式** | 进程内沙箱 | 独立进程 |
| **资源消耗** | 极低（MB 级） | 中等（独立进程） |
| **启动速度** | 毫秒级 | 秒级 |
| **网络访问** | ❌ 仅通过 Host API | ✅ 完整网络能力 |
| **适用场景** | 事件监听、内容过滤、AI 调用 | IM 适配器、长连接、重型计算 |

### 完整示例：帖子字数统计插件

**plugin.yaml:**

```yaml
apiVersion: campusos/v1alpha1
kind: Plugin

metadata:
  name: "word-counter"
  version: "1.0.0"
  displayName: "帖子字数统计"
  description: "自动统计帖子字数并添加到元数据"
  author: "Your Name"
  license: "MIT"

spec:
  runtime: "wasm"
  wasm:
    entrypoint: "main.wasm"
    memoryLimit: "64MB"
    timeout: "10s"

events:
  subscriptions:
    - topic: "thread.created.before"
      handler: "onThreadBeforeCreate"
      priority: 50

permissions:
  hostAPI:
    - resource: "thread"
      actions: ["read", "update"]
```

**main.go:**

```go
package main

import (
    "encoding/json"
    "unicode/utf8"
    
    "github.com/campusos/sdk-go/wasm"
)

// ThreadData 帖子事件数据
type ThreadData struct {
    Title      string   `json:"title"`
    Content    string   `json:"content"`
    AuthorID   string   `json:"author_id"`
    CategoryID string   `json:"category_id"`
    Metadata   map[string]interface{} `json:"metadata"`
}

func main() {
    // 注册事件处理器
    wasm.Handle("onThreadBeforeCreate", onThreadBeforeCreate)
    
    // 启动插件
    wasm.Run()
}

// onThreadBeforeCreate 在帖子创建前统计字数
func onThreadBeforeCreate(ctx *wasm.Context, event *wasm.Event) *wasm.Result {
    // 解析事件数据
    var data ThreadData
    if err := json.Unmarshal(event.Data, &data); err != nil {
        ctx.Log.Error("failed to unmarshal event data: " + err.Error())
        return wasm.Allow() // 解析失败时放行
    }
    
    // 统计字数
    titleRunes := utf8.RuneCountInString(data.Title)
    contentRunes := utf8.RuneCountInString(data.Content)
    totalRunes := titleRunes + contentRunes
    
    // 添加字数到元数据
    if data.Metadata == nil {
        data.Metadata = make(map[string]interface{})
    }
    data.Metadata["word_count"] = totalRunes
    data.Metadata["title_length"] = titleRunes
    data.Metadata["content_length"] = contentRunes
    
    // 如果内容过短，拒绝发布
    if contentRunes < 10 {
        return wasm.Block("帖子内容过短，至少需要 10 个字符")
    }
    
    // 返回修改后的数据
    modifiedData, _ := json.Marshal(data)
    return wasm.Modify(modifiedData)
}
```

```yaml
    resources:
      memoryLimit: "256MB"
      cpuLimit: "0.5"

events:

    - topic: "thread.created.after"
      handler: "onThreadCreated"
      priority: 300
    - topic: "post.created.after"
      handler: "onPostCreated"
      priority: 300

  hostAPI:
    - resource: "thread"
      actions: ["read"]
    - resource: "user"
      actions: ["read"]

configuration:
  schema:
    type: object
    properties:
      webhook_url:
        type: string
        description: "Webhook 接收 URL"
      secret:
        type: string
        description: "Webhook 签名密钥"
    required: ["webhook_url"]
```

**main.go:**

```go
package main

import (
    "context"
    "crypto/hmac"
    "crypto/sha256"
    "encoding/hex"
    "encoding/json"
    "fmt"
    "net/http"
    "time"

    pb "github.com/campusos/sdk-go/proto/plugin"
    "github.com/campusos/sdk-go/grpc"
    "github.com/campusos/sdk-go/grpc/hostapi"
)

// WebhookPlugin Webhook 转发插件
type WebhookPlugin struct {
    pb.UnimplementedPluginServiceServer
    hostAPI     hostapi.Client
    webhookURL  string
    secret      string
    httpClient  *http.Client
}

// GetManifest 返回插件清单
func (p *WebhookPlugin) GetManifest(ctx context.Context, req *pb.GetManifestRequest) (*pb.GetManifestResponse, error) {
    return &pb.GetManifestResponse{
        Manifest: grpc.LoadManifest("plugin.yaml"),
    }, nil
}

// HealthCheck 健康检查
func (p *WebhookPlugin) HealthCheck(ctx context.Context, req *pb.HealthCheckRequest) (*pb.HealthCheckResponse, error) {
    return &pb.HealthCheckResponse{Status: pb.HealthCheckResponse_SERVING}, nil
}

// HandleEvent 处理事件
func (p *WebhookPlugin) HandleEvent(ctx context.Context, req *pb.HandleEventRequest) (*pb.HandleEventResponse, error) {
    // 构建 Webhook payload
    payload := map[string]interface{}{
        "event_type": req.Topic,
        "event_id":   req.EventId,
        "timestamp":  time.Now().UTC().Format(time.RFC3339),
        "data":       json.RawMessage(req.Payload),
    }
    
    // 如果配置了用户信息，尝试获取
    if p.hasUserLookup(req) {
        userID := p.extractUserID(req.Payload)
        if userID != "" {
            user, err := p.hostAPI.Identity().GetUser(ctx, userID)
            if err == nil {
                payload["user"] = user
            }
        }
    }
    
    // 发送到 Webhook URL
    if err := p.sendWebhook(ctx, payload); err != nil {
        return nil, fmt.Errorf("send webhook: %w", err)
    }
    
    return &pb.HandleEventResponse{
        Action: pb.HandleEventResponse_ALLOW,
    }, nil
}

// Shutdown 优雅关闭
func (p *WebhookPlugin) Shutdown(ctx context.Context, req *pb.ShutdownRequest) (*pb.ShutdownResponse, error) {
    p.httpClient.CloseIdleConnections()
    return &pb.ShutdownResponse{}, nil
}

// sendWebhook 发送 Webhook 请求
func (p *WebhookPlugin) sendWebhook(ctx context.Context, payload interface{}) error {
    body, err := json.Marshal(payload)
    if err != nil {
        return err
    }
    
    req, err := http.NewRequestWithContext(ctx, "POST", p.webhookURL, bytes.NewReader(body))
    if err != nil {
        return err
    }
    
    req.Header.Set("Content-Type", "application/json")
    req.Header.Set("User-Agent", "CampusOS-Webhook/1.0")
    
    // 计算签名
    if p.secret != "" {
        signature := p.computeSignature(body)
        req.Header.Set("X-CampusOS-Signature", signature)
    }
    
    resp, err := p.httpClient.Do(req)
    if err != nil {
        return err
    }
    defer resp.Body.Close()
    
    if resp.StatusCode >= 400 {
        return fmt.Errorf("webhook returned status %d", resp.StatusCode)
    }
    
    return nil
}

// computeSignature 计算 HMAC-SHA256 签名
func (p *WebhookPlugin) computeSignature(body []byte) string {
    mac := hmac.New(sha256.New, []byte(p.secret))
    mac.Write(body)
    return "sha256=" + hex.EncodeToString(mac.Sum(nil))
}

func main() {
    // 加载配置
    config := grpc.LoadConfig()
    
    plugin := &WebhookPlugin{
        hostAPI:    hostapi.NewClient(),
        webhookURL: config.GetString("webhook_url"),
        secret:     config.GetString("secret"),
        httpClient: &http.Client{Timeout: 10 * time.Second},
    }
    
    // 启动 gRPC 服务
    if err := grpc.Serve(plugin); err != nil {
        panic(err)
    }
}
```

## Host API 使用

### Go SDK 接口

```go
// Host API 客户端接口
type HostAPI interface {
    // Identity 身份服务
    Identity() IdentityAPI
    // Permission 权限服务
    Permission() PermissionAPI
    // Storage 存储服务
    Storage() StorageAPI
    // Data 数据服务
    Data() DataAPI
    // Event 事件服务
    Event() EventAPI
    // Notification 通知服务
    Notification() NotificationAPI
    // Config 配置服务
    Config() ConfigAPI
    // AI AI 网关
    AI() AIAPI
    // Log 日志服务
    Log() LogAPI
}

// IdentityAPI 身份 API
type IdentityAPI interface {
    GetUser(ctx context.Context, userID string) (*User, error)
    SearchUsers(ctx context.Context, query string, limit int) ([]*User, error)
}

// DataAPI 数据 API
type DataAPI interface {
    Get(ctx context.Context, resource string, id string) (json.RawMessage, error)
    Query(ctx context.Context, resource string, filter Filter) (*QueryResult, error)
    Update(ctx context.Context, resource string, id string, data json.RawMessage) error
}

// EventAPI 事件 API
type EventAPI interface {
    Publish(ctx context.Context, topic string, data json.RawMessage) error
}

// AIAPI AI 网关 API
type AIAPI interface {
    ChatCompletion(ctx context.Context, messages []Message, opts ...ChatOption) (*ChatResponse, error)
}

// LogAPI 日志 API
type LogAPI interface {
    Debug(msg string, args ...interface{})
    Info(msg string, args ...interface{})
    Warn(msg string, args ...interface{})
    Error(msg string, args ...interface{})
}
```

## 事件处理

### `.before` 事件处理

```go
// .before 事件可以返回三种结果
func onThreadBeforeCreate(ctx *wasm.Context, event *wasm.Event) *wasm.Result {
    var data ThreadData
    json.Unmarshal(event.Data, &data)
    
    // 方式 1: 允许操作继续
    return wasm.Allow()
    
    // 方式 2: 阻断操作
    return wasm.Block("包含敏感内容")
    
    // 方式 3: 修改数据后放行
    data.Content = sanitizeContent(data.Content)
    modifiedData, _ := json.Marshal(data)
    return wasm.Modify(modifiedData)
}
```

### `.after` 事件处理

```go
// .after 事件不需要返回结果，只需要执行副作用
func onThreadAfterCreate(ctx *wasm.Context, event *wasm.Event) *wasm.Result {
    var data ThreadCreatedData
    json.Unmarshal(event.Data, &data)
    
    // 执行副作用操作
    // 更新统计
    // 发送通知
    // 同步到外部系统
    
    return wasm.Allow() // .after 事件的返回值会被忽略
}
```

## 最佳实践

### 1. 幂等处理

```go
// ✅ 正确：插件处理应具备幂等性
func onThreadCreated(ctx *wasm.Context, event *wasm.Event) *wasm.Result {
    // 使用事件 ID 做去重
    processed, _ := ctx.Host.Config().Get("processed:" + event.ID)
    if processed != "" {
        return wasm.Allow() // 已处理，跳过
    }
    
    // 处理逻辑...
    
    // 标记已处理
    ctx.Host.Config().Set("processed:"+event.ID, "true")
    return wasm.Allow()
}
```

### 2. 优雅降级

## FAQ

### Q: Wasm 插件能否访问外部网络？

**A:** 不能直接访问。Wasm 插件运行在完全隔离的沙箱中，所有外部通信必须通过 Host API。如果需要访问外部网络，应使用 gRPC 插件。

### Q: 插件之间可以互相调用吗？

**A:** 不可以。插件之间不直接通信，而是通过 Event Bus 间接通信。插件 A 发布自定义事件，插件 B 订阅该事件。

### Q: 插件崩溃会影响核心系统吗？

**A:** 不会。Wasm 插件在沙箱中运行，崩溃不会影响宿主进程。gRPC 插件运行在独立进程中，崩溃后会被 Plugin Manager 自动重启。

### Q: 如何处理插件的数据库迁移？

**A:** 插件不应该直接访问核心数据库。所有数据操作应通过 Host API 完成。如果插件需要私有数据存储，可以使用 Host API 的 Key-Value 存储接口。

### Q: 如何调试 .before 事件的 MODIFY 操作？

**A:** 使用 `campusos plugin debug` 命令，它会输出 MODIFY 前后的数据对比。建议在测试中明确验证修改后的数据结构。

### Q: gRPC 插件如何实现热更新？

**A:** gRPC 插件支持滚动更新。Plugin Manager 会先启动新版本实例，健康检查通过后再停止旧版本实例，实现零停机更新。
