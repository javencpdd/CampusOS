# CampusOS 账号认证与管理员登录说明

> 更新日期：2026-07-02
> 适用范围：CampusOS 本地开发环境、Docker PostgreSQL、用户前台、管理后台

## 1. 账号数据保存在哪

CampusOS 不使用单独的 `user` 表，用户资料表名是 `users`。

| 表 | 保存内容 | 说明 |
| --- | --- | --- |
| `users` | 用户基础资料 | `id`、`username`、`nickname`、`email`、`status` 等 |
| `accounts` | 登录凭据 | 邮箱登录账号、bcrypt 密码哈希、验证状态 |
| `roles` | 系统角色 | 默认包含 `admin`、`moderator`、`member`、`guest` |
| `user_roles` | 用户和角色绑定 | 管理员也是普通用户记录，再绑定 `admin` 角色 |

查询用户和登录账号：

```bash
docker exec -it campusos-postgres psql -U campusos -d campusos
```

进入 `psql` 后执行：

```sql
SELECT id, username, nickname, email, status, created_at
FROM users
ORDER BY created_at DESC;

SELECT user_id, type, identifier, verified, created_at
FROM accounts
ORDER BY created_at DESC;
```

联查账号和角色：

```sql
SELECT u.id, u.username, u.email, r.name AS role
FROM users u
LEFT JOIN user_roles ur ON ur.user_id = u.id AND ur.deleted_at IS NULL
LEFT JOIN roles r ON r.id = ur.role_id AND r.deleted_at IS NULL
WHERE u.deleted_at IS NULL
ORDER BY u.created_at DESC;
```

## 2. 用户前台注册写入哪些表

用户在 `http://localhost:3000/register` 注册成功后，后端会写入：

| 表 | 写入内容 |
| --- | --- |
| `users` | 用户 ID、用户名、昵称、邮箱、状态 |
| `accounts` | `type=email`、`identifier=邮箱`、`credential=密码哈希` |

如果接口返回 `201 created`，但 pgAdmin 里看不到新用户，优先检查后端是否连接了 Docker PostgreSQL。

正常启动日志应包含：

```text
✅ PostgreSQL 连接成功
```

如果后端因为连不上数据库进入内存模式，注册仍可能返回成功，但数据只保存在进程内存中，重启后消失，pgAdmin 也查不到。

## 3. 管理后台默认账号

CampusOS 管理后台地址：

```text
http://localhost:3001/login
```

默认管理员账号：

| 项目 | 值 |
| --- | --- |
| 邮箱 | `admin@campusos.local` |
| 密码 | `Admin@123456` |
| 角色 | `admin` |

注意不要和 pgAdmin 的登录账号混淆：

| 系统 | 登录地址 | 邮箱 | 密码 |
| --- | --- | --- | --- |
| CampusOS 管理后台 | `http://localhost:3001/login` | `admin@campusos.local` | `Admin@123456` |
| pgAdmin 数据库管理页 | `http://localhost:5050` | `admin@campusos.dev` | `pgadmin123` |

## 4. 系统配置文件

本地开发主要配置文件是项目根目录的 `.env`：

```env
POSTGRES_PORT=5433
DATABASE_DSN=postgres://campusos:campusos_dev@localhost:5433/campusos?sslmode=disable
AUTH_PASSWORD_HASH_ENABLED=true
```

后端启动时读取配置的优先级：

| 优先级 | 来源 | 说明 |
| --- | --- | --- |
| 1 | 进程环境变量 | 例如 `DATABASE_DSN=... go run ./cmd/server` |
| 2 | 项目根目录 `.env` | 从项目根目录启动后端时自动读取 |
| 3 | 代码默认值 | 没有配置时使用默认值 |

如果 `.env` 中 Docker PostgreSQL 映射到 `5433`，但后端没有读取 `.env`，就可能错误连接 `localhost:5432`，连接失败后进入内存模式。

## 5. 密码哈希开关

CampusOS 支持通过 `.env` 选择密码凭据保存方式：

| 配置 | 行为 |
| --- | --- |
| `AUTH_PASSWORD_HASH_ENABLED=true` | 默认值。注册时使用 bcrypt 保存到 `accounts.credential`，登录时用 bcrypt 校验 |
| `AUTH_PASSWORD_HASH_ENABLED=false` | 本地调试模式。注册时把明文密码保存到 `accounts.credential`，登录时直接明文比较 |

关闭密码哈希：

```env
AUTH_PASSWORD_HASH_ENABLED=false
```

然后重启后端。

注意：该模式只适合本地临时调试，不建议提交到生产环境或共享测试环境。

原因：

| 风险 | 说明 |
| --- | --- |
| 数据风险 | 明文密码会直接进入数据库、备份和调试输出 |
| 逻辑风险 | 哈希模式和明文模式不能混用；切换后需要同步迁移已有 `accounts.credential` |
| 迁移风险 | 后续再恢复哈希时需要批量重写已有账号凭据 |
| 协作风险 | 临时配置容易被误提交或被他人复用 |

如果关闭哈希后注册了新用户，数据库中看到的 `accounts.credential` 就是明文密码。重新开启哈希前，应先把这些明文密码转换为 bcrypt 哈希。

## 6. 生成 bcrypt 哈希

Bash：

```bash
scripts/hash-password.sh 'Admin@123456'
```

也可以避免把密码留在 shell history 中：

```bash
scripts/hash-password.sh
```

PowerShell：

```powershell
.\scripts\hash-password.ps1 -Password 'Admin@123456'
```

或交互输入：

```powershell
.\scripts\hash-password.ps1
```

输出示例：

```text
$2a$10$xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

将明文账号改回哈希保存时，可以先生成哈希，再更新 `accounts.credential`。

示例：

```bash
hash="$(scripts/hash-password.sh '用户明文密码')"
docker exec -i campusos-postgres psql -U campusos -d campusos <<SQL
UPDATE accounts
SET credential = '$hash',
    updated_at = NOW()
WHERE type = 'email'
  AND identifier = 'user@example.com'
  AND deleted_at IS NULL;
SQL
```

## 7. 重置默认管理员密码

如果默认管理员密码被改乱，可以把密码重置为 `Admin@123456`。

哈希模式下执行：

执行：

```bash
docker exec -i campusos-postgres psql -U campusos -d campusos <<'SQL'
UPDATE accounts
SET credential = '$2a$10$fL4UMGXtNMprJEykAqvor.TJWB4MXECJUbrHs6dIFHW6TC8P2vhXS',
    verified = TRUE,
    updated_at = NOW()
WHERE type = 'email'
  AND identifier = 'admin@campusos.local'
  AND deleted_at IS NULL;
SQL
```

然后使用：

```text
admin@campusos.local
Admin@123456
```

登录管理后台。

明文模式下执行：

```bash
docker exec -i campusos-postgres psql -U campusos -d campusos <<'SQL'
UPDATE accounts
SET credential = 'Admin@123456',
    verified = TRUE,
    updated_at = NOW()
WHERE type = 'email'
  AND identifier = 'admin@campusos.local'
  AND deleted_at IS NULL;
SQL
```

## 8. 排查顺序

遇到“注册成功但数据库看不到用户”或“管理员登录失败”时，按下面顺序查：

```bash
docker ps --filter name=campusos-postgres
make migrate-status
curl http://127.0.0.1:8080/api/v1/health
```

再查用户表：

```bash
docker exec -i campusos-postgres psql -U campusos -d campusos -P pager=off -c \
"SELECT id, username, email, status FROM users ORDER BY created_at DESC;"
```

如果后端启动日志没有 `PostgreSQL 连接成功`，应先修复 `DATABASE_DSN` 或确认后端是否从项目根目录启动。
