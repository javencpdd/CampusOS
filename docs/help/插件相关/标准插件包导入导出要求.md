# CampusOS 标准插件包导入导出要求

本文说明 CampusOS 标准插件包的格式要求、导入导出规则和数据边界。具体插件的功能和使用方法仍以插件目录中的 README 为准。

## 1. 标准插件包结论

CampusOS 当前标准插件包使用：

- 文件后缀：`.campusos-plugin.tar.gz`
- 压缩格式：`tar.gz`
- 包根目录：必须直接包含 `plugin.yaml`
- 导入目标：默认安装到 `examples/plugins/<插件名>/`
- 目录覆盖：可通过 `PLUGINS_DIR` 调整插件根目录

示例插件包名：

```text
hello-wasm-0.1.0.campusos-plugin.tar.gz
```

标准插件包只保存插件运行所需文件，不保存运行期数据、数据库记录和日志。

## 2. 包内目录结构要求

标准插件包解压后的根目录必须是插件目录本身，而不是再套一层父目录。

推荐结构：

```text
plugin.yaml
README.md
plugin.wasm
src/
  handle_event.wat
```

不推荐结构：

```text
hello-wasm/
  plugin.yaml
  plugin.wasm
```

原因是导入逻辑会从包根目录读取 `plugin.yaml`，再根据 manifest 中的 `name` 安装到：

```text
examples/plugins/<name>/
```

## 3. `plugin.yaml` 必填要求

`plugin.yaml` 是插件包的入口文件，至少需要包含：

```yaml
name: hello-wasm
version: "0.1.0"
runtime: wasm
storage:
  type: none
config:
  module: "plugin.wasm"
```

字段要求：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| `name` | 必填 | 插件唯一名称。只能包含小写字母、数字、连字符 `-` 和下划线 `_`。 |
| `version` | 必填 | 插件版本号。建议使用语义化版本，例如 `0.1.0`。 |
| `runtime` | 必填 | 当前支持 `wasm` 和 `grpc`。 |
| `events.subscribe` | 可选 | 插件订阅的事件列表。 |
| `permissions.api` | 可选 | 插件声明的 Host API 权限。 |
| `storage` | 建议填写 | 当前支持 `none`、`sqlite` 等声明方式。 |
| `config` | 视运行时而定 | 运行时配置，例如 Wasm 模块路径、入口函数、超时时间。 |
| `config_schema` | 可选 | 用于 Admin 或 CLI 渲染配置表单。 |

## 4. Wasm 插件包要求

当 `runtime: wasm` 时：

- `config.module` 必须指向包内的 Wasm 文件。
- 如果未配置 `config.module`，默认按 `plugin.wasm` 查找。
- `config.module` 必须是相对路径，不能是绝对路径，也不能跳出插件目录。
- 被引用的 Wasm 文件必须实际存在。

示例：

```yaml
runtime: wasm
config:
  module: "plugin.wasm"
  entrypoint: "handle_event"
  event_timeout_ms: 1000
```

## 5. gRPC 插件包要求

当 `runtime: grpc` 时：

- `plugin.yaml` 中应声明启动命令或连接方式。
- 如果插件需要可执行文件，应将可执行文件放入插件包内。
- 如果插件依赖外部服务，应在 README 中说明部署要求。

示例：

```yaml
runtime: grpc
config:
  command: "./plugin"
  event_timeout_ms: 1000
```

## 6. 禁止打包的内容

标准导出会自动跳过以下内容：

| 内容 | 原因 |
| --- | --- |
| `.git/` | 避免把版本库元数据打入插件包。 |
| `data/` | 运行期数据不属于插件代码包。 |
| `node_modules/` | 依赖体积过大，应通过构建产物或 README 说明恢复方式。 |
| `*.log` | 日志属于运行期数据。 |
| `*.tmp` | 临时文件不应进入标准包。 |
| `.DS_Store` | 本地系统文件不应进入标准包。 |

导入时如果发现归档内包含这些禁止路径，会拒绝导入。

## 7. 安全限制

导入插件包时会进行以下检查：

- 拒绝绝对路径。
- 拒绝 `../` 路径逃逸。
- 拒绝符号链接、硬链接等非普通文件类型。
- 拒绝非法插件名。
- 拒绝缺少 `plugin.yaml` 的包。
- 拒绝缺少 Wasm 模块文件的 Wasm 插件包。
- 同名插件默认不能覆盖，需要显式选择覆盖。

## 8. Admin 导入导出

Admin 后台 `插件管理` 页面已支持：

- 导入插件包。
- 选择是否覆盖同名插件。
- 导出单个已安装插件。
- 导入后刷新插件列表。

对应接口：

| 动作 | 接口 | 说明 |
| --- | --- | --- |
| 导入插件包 | `POST /api/v1/plugin-packages/import` | 使用 `multipart/form-data`，字段名为 `file`，可传 `replace=true`。 |
| 导出插件包 | `GET /api/v1/plugins/:name/export` | 返回 `.campusos-plugin.tar.gz` 文件。 |

权限要求：

- 需要登录 Admin。
- 需要具备 `role:manage` 权限。

## 9. CLI 导入导出

CLI 仍然可以使用同一套标准插件包规则。

导出：

```bash
go run ./cmd/campusosctl plugin pack examples/plugins/hello-wasm
```

指定输出路径：

```bash
go run ./cmd/campusosctl plugin pack examples/plugins/hello-wasm --out /tmp/hello-wasm-0.1.0.campusos-plugin.tar.gz
```

导入：

```bash
go run ./cmd/campusosctl plugin install /tmp/hello-wasm-0.1.0.campusos-plugin.tar.gz --dir examples/plugins
```

覆盖同名插件：

```bash
go run ./cmd/campusosctl plugin install /tmp/hello-wasm-0.1.0.campusos-plugin.tar.gz --dir examples/plugins --replace
```

## 10. 数据边界

标准插件包会包含：

- `plugin.yaml`
- 插件 README
- Wasm 文件或 gRPC 可执行文件
- 插件源码或必要静态资源
- 插件运行所需的配置模板

标准插件包不会包含：

- `.campusos/plugin-data/<插件名>/plugin.db`
- PostgreSQL `plugins` 表记录
- PostgreSQL `plugin_logs` 日志
- PostgreSQL `plugin_permissions` 权限记录
- API Key
- 插件运行期缓存

这意味着插件包用于迁移“插件本身”，不用于备份“插件运行后的业务数据”。

## 11. 推荐发布检查清单

发布插件包前建议检查：

1. `plugin.yaml` 可以被 `campusosctl plugin inspect` 正常解析。
2. 插件名称符合小写命名规则。
3. 版本号已更新。
4. Wasm 或 gRPC 运行文件存在。
5. README 已说明插件作用、配置项和运行依赖。
6. 没有把 `data/`、日志、临时文件、API Key 打进包内。
7. 本地完成一次导出和重新导入验证。
