# CampusOS 个人主页风格包说明

> 适用阶段：v0.4-dev
> 更新时间：2026-07-01

## 1. 功能定位

个人主页风格包用于在不同用户之间分享、导入和迁移个人主页外观。当前阶段已完成风格包 manifest 格式、安全校验、导入预览、导出当前风格和一键应用到个人主页。

当前支持：

| 能力 | 状态 |
| --- | --- |
| 风格包 manifest JSON 格式 | 已支持 |
| 安全校验 | 已支持 |
| 登录用户校验 API | 已支持 |
| 风格包导入预览 | 已支持 |
| 当前主页风格导出 | 已支持 |
| 风格包一键应用 | 已支持 |
| 用户侧前端入口 | 已支持 |
| 应用前快照和回滚 | 后续任务 |

## 2. 校验 API

接口：

```text
POST /api/v1/spaces/me/styles/validate
```

该接口需要登录：

```bash
curl -X POST http://localhost:8080/api/v1/spaces/me/styles/validate \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "manifest": {
      "schema_version": "space-style.v1",
      "name": "clean-blog",
      "version": "0.1.0",
      "author": "Alice",
      "description": "Clean blog style for CampusOS personal spaces.",
      "preview_image": "assets/preview.png",
      "compatible_campusos": [">=0.4.0"],
      "layout": "blog",
      "components": [
        {
          "slot": "header",
          "type": "profile-header",
          "props": {
            "show_avatar": true,
            "title_size": "lg"
          }
        },
        {
          "slot": "main",
          "type": "content-list",
          "props": {
            "density": "comfortable"
          }
        }
      ],
      "tokens": {
        "color.primary": "#2563eb",
        "font.body": "system-ui",
        "radius.card": "8px",
        "space.section": "24px"
      },
      "assets": [
        {
          "name": "preview",
          "path": "assets/preview.png",
          "type": "image/png"
        }
      ]
    }
  }'
```

返回示例：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "valid": true
  }
}
```

如果校验失败，接口仍返回统一成功响应，但 `data.valid=false`，并列出错误：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "valid": false,
    "errors": [
      "manifest.components[0].type \"script\" is not supported"
    ]
  }
}
```

## 3. 导入预览 API

接口：

```text
POST /api/v1/spaces/me/styles/preview
```

该接口需要登录，请求体与校验 API 一致。接口会返回：

| 字段 | 说明 |
| --- | --- |
| `validation` | 校验结果 |
| `owner` | 当前登录用户的公开展示信息 |
| `space` | 当前用户的个人主页配置 |
| `manifest` | 校验通过后的规范化 manifest |
| `layout` | 预览布局 |
| `components` | 可渲染组件列表 |
| `tokens` | 可渲染样式 token |
| `assets` | 包内资源声明 |

如果 `validation.valid=false`，接口不会返回 `manifest`、`layout`、`components`、`tokens` 和 `assets`，前端应只展示错误信息，不能渲染风格包。

示例：

```bash
curl -X POST http://localhost:8080/api/v1/spaces/me/styles/preview \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d @style-package.json
```

返回示例：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "validation": {
      "valid": true
    },
    "owner": {
      "id": "1001",
      "username": "alice",
      "nickname": "Alice"
    },
    "space": {
      "user_id": "1001",
      "title": "Alice的个人主页",
      "layout": "blog",
      "theme": "default"
    },
    "layout": "blog",
    "components": [
      {
        "slot": "header",
        "type": "profile-header"
      }
    ],
    "tokens": {
      "color.primary": "#2563eb"
    }
  }
}
```

## 4. 风格导出 API

接口：

```text
POST /api/v1/spaces/me/styles/export
```

该接口需要登录。接口会读取当前登录用户的个人主页配置，将 `layout`、`theme` 和受控组件组合导出为标准风格包 JSON。

请求体可以为空对象：

```bash
curl -X POST http://localhost:8080/api/v1/spaces/me/styles/export \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{}'
```

也可以指定导出名称、版本和描述：

```bash
curl -X POST http://localhost:8080/api/v1/spaces/me/styles/export \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "alice clean blog",
    "version": "0.2.0",
    "description": "Alice 的个人主页风格"
  }'
```

返回示例：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "filename": "alice-clean-blog-0.2.0.space-style.json",
    "validation": {
      "valid": true
    },
    "package": {
      "manifest": {
        "schema_version": "space-style.v1",
        "name": "alice-clean-blog",
        "version": "0.2.0",
        "author": "Alice",
        "description": "Alice 的个人主页风格",
        "compatible_campusos": [">=0.4.0"],
        "layout": "blog",
        "components": [
          {
            "slot": "header",
            "type": "profile-header",
            "props": {
              "align": "center",
              "show_avatar": true,
              "show_cover": false
            }
          },
          {
            "slot": "main",
            "type": "content-list",
            "props": {
              "density": "comfortable",
              "show_excerpt": true,
              "show_meta": true
            }
          }
        ],
        "tokens": {
          "color.primary": "#2563eb",
          "color.background": "#ffffff",
          "color.surface": "#f8fafc",
          "font.body": "system-ui",
          "radius.card": "8px",
          "space.section": "24px"
        }
      }
    }
  }
}
```

导出规则：

| 项目 | 规则 |
| --- | --- |
| `name` | 未传时由用户名生成；传入值会转为小写短横线格式 |
| `version` | 未传时为 `0.1.0`；传入值必须是语义版本 |
| `layout` | 使用当前主页布局；不支持的布局回退为 `blog` |
| `theme` | 转换为受控 token，目前支持 `default`、`ink`、`dark`、`warm` |
| 个人内容 | 不导出帖子、标题正文、同步内容或运行数据 |
| 资源文件 | 当前导出只生成 manifest JSON，不打包图片资源 |

## 5. 风格应用 API

接口：

```text
POST /api/v1/spaces/me/styles/apply
```

该接口需要登录，请求体与校验 API 一致。接口会先执行完整安全校验，只有 `validation.valid=true` 时才会写入当前用户的个人主页配置。

示例：

```bash
curl -X POST http://localhost:8080/api/v1/spaces/me/styles/apply \
  -H "Authorization: Bearer <access_token>" \
  -H "Content-Type: application/json" \
  -d @style-package.json
```

应用成功后会更新：

| 字段 | 写入内容 |
| --- | --- |
| `theme` | 风格包 `manifest.name` |
| `layout` | 风格包 `manifest.layout` |
| `style_name` | 风格包名称 |
| `style_version` | 风格包版本 |
| `style_manifest` | 规范化后的完整 manifest |

如果风格包非法，接口返回 400，且不会写入 `user_spaces`。

返回示例：

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "validation": {
      "valid": true
    },
    "space": {
      "theme": "clean-blog",
      "layout": "blog",
      "style_name": "clean-blog",
      "style_version": "0.1.0"
    },
    "applied": {
      "schema_version": "space-style.v1",
      "name": "clean-blog",
      "version": "0.1.0",
      "layout": "blog"
    }
  }
}
```

## 7. Manifest 字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `schema_version` | 是 | 当前固定为 `space-style.v1` |
| `name` | 是 | 小写字母、数字和中划线，最长 63 字符 |
| `version` | 是 | 语义版本，例如 `0.1.0` |
| `author` | 否 | 作者 |
| `description` | 否 | 描述 |
| `preview_image` | 否 | 包内相对路径，支持 png/jpg/jpeg/webp |
| `compatible_campusos` | 否 | 兼容版本声明 |
| `layout` | 是 | 支持 `blog`、`grid`、`timeline`、`magazine` |
| `components` | 是 | 受控组件列表，最多 30 个 |
| `tokens` | 否 | CSS 变量式 token，最多 80 个 |
| `assets` | 否 | 包内资源声明，最多 20 个 |

## 8. 受控组件

组件由 `slot`、`type`、`props` 组成。

支持的 slot：

```text
header
main
sidebar
footer
```

支持的 type：

```text
profile-header
content-list
category-tabs
tag-cloud
rich-text
link-list
hero
footer
```

## 9. 安全规则

当前阶段不允许任意 JavaScript、HTML 或外链资源进入风格包。

校验会拒绝：

| 风险 | 规则 |
| --- | --- |
| 脚本组件 | `type=script`、`iframe`、未在白名单内的组件会被拒绝 |
| 事件处理 props | `onload`、`onclick` 等 `on*` 字段会被拒绝 |
| HTML 注入 | 包含 `html`、`script` 语义的 props 会被拒绝 |
| 危险字符串 | `<script`、`javascript:`、`data:text/html`、`<iframe` 等会被拒绝 |
| 外链资源 | `http://`、`https://`、`//` 资源路径会被拒绝 |
| 路径穿越 | `../`、绝对路径、空路径会被拒绝 |
| CSS 注入 | token 值中的 `url(`、`expression(`、`javascript:` 会被拒绝 |
| SVG | 当前阶段不允许 SVG 资源，避免脚本嵌入风险 |
