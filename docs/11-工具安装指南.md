| 项目 | 最低要求 | 推荐配置 |

## 一、基础开发工具

### 1.1 系统更新与基础依赖

```bash
# 更新系统软件包
sudo apt update && sudo apt upgrade -y

# 安装基础开发工具
sudo apt install -y \
    git \
    make \
    curl \
    wget \
    unzip \
    build-essential \
    ca-certificates \
    gnupg \
    lsb-release \
    software-properties-common \
    apt-transport-https \
    vim \
    jq \
    tree

# 更新 CA 证书（确保 HTTPS 连接正常）
sudo update-ca-certificates --fresh

# 验证基础工具
git --version
# 预期输出: git version 2.34.1 (Ubuntu 22.04 默认版本)

make --version | head -1
# 预期输出: GNU Make 4.3

curl --version | head -1
# 预期输出: curl 7.81.0
```

### 1.2 Git 配置

```bash
# 设置用户信息（替换为你的信息）
git config --global user.name "jack"
git config --global user.email "1904560862@qq.com"

# 设置默认分支名
git config --global init.defaultBranch main

# 设置换行符（Linux 环境）
git config --global core.autocrlf input

# 启用颜色输出
git config --global color.ui auto

# 查看配置
git config --list
```

### 1.3 代理配置（可选）

如果你在中国大陆地区，可能需要配置代理来加速访问 GitHub 等服务：


## 二、后端开发环境

### 2.1 Go 安装

CampusOS 后端使用 Go 1.22+ 编写。

```bash
# ─── 第一步：更新 SSL 证书（解决 wget/curl HTTPS 问题）───
sudo apt update && sudo apt upgrade -y openssl ca-certificates
sudo update-ca-certificates --fresh

# ─── 第二步：下载并安装 Go ───
GO_VERSION="1.22.5"
wget "https://go.dev/dl/go${GO_VERSION}.linux-amd64.tar.gz"

# 清理旧版本并安装到 /usr/local/go
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf "go${GO_VERSION}.linux-amd64.tar.gz"
rm "go${GO_VERSION}.linux-amd64.tar.gz"

# ─── 第三步：配置环境变量 ───
# 追加到 ~/.bashrc（仅在未配置时添加）
grep -q '/usr/local/go/bin' ~/.bashrc || {
    cat >> ~/.bashrc << 'ENVEOF'

# Go 环境变量
export PATH=$PATH:/usr/local/go/bin
export GOPATH=$HOME/go
export PATH=$PATH:$GOPATH/bin
ENVEOF
}

# 使环境变量生效
source ~/.bashrc

# ─── 第四步：配置 Go 代理（中国大陆用户必选）───
go env -w GOPROXY=https://goproxy.cn,direct
go env -w GOSUMDB=sum.golang.google.cn
go env -w GO111MODULE=on

# ─── 第五步：验证安装 ───
go version
# 预期输出: go version go1.22.5 linux/amd64

go env GOPATH
# 预期输出: /home/jack/go

go env GOPROXY
# 预期输出: https://goproxy.cn,direct
```

### 2.2 Go 开发工具链

```bash
# 安装 gopls（Go 官方语言服务器，VS Code 必需）
go install golang.org/x/tools/gopls@latest
```

## 三、前端开发环境

### 3.1 Node.js 安装

CampusOS 前端使用 Vue 3 + TypeScript，需要 Node.js 20 LTS。

#### 方式一：使用 nvm（推荐）

```bash
# 安装 nvm (Node Version Manager)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash

# 重新加载 shell 配置
source ~/.bashrc

# 安装 Node.js 20 LTS
nvm install 20
nvm use 20
nvm alias default 20

# 验证
node --version
# 预期输出: v20.x.x

npm --version
# 预期输出: 10.x.x
```

#### 方式二：使用 apt 直接安装

```bash
# 添加 NodeSource 仓库
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -

# 安装 Node.js 20 LTS
sudo apt install -y nodejs

# 验证
node --version
npm --version
```

### 3.2 pnpm 安装

pnpm 是推荐的前端包管理器，比 npm 更快、更节省磁盘空间。

```bash
# 通过 npm 安装 pnpm
npm install -g pnpm@8

# 验证
pnpm --version
# 预期输出: 8.x.x

# 配置国内镜像（中国大陆用户）
pnpm config set registry https://registry.npmmirror.com
npm config set registry https://registry.npmmirror.com
```

## 四、数据库与中间件

> **推荐方式**：以下中间件建议使用 Docker 方式安装（见第五章），以下是原生安装方式，供需要原生部署的场景参考。

### 4.1 PostgreSQL 16

```bash
# 添加 PostgreSQL 官方 APT 仓库
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
# 下载密钥并保存到指定的 keyring 目录
curl -fsSL https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/postgresql.gpg
# wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -
sudo apt update

# 安装 PostgreSQL 16
sudo apt install -y postgresql-16 postgresql-client-16

# 启动服务并设置开机自启
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 验证
psql --version
# 预期输出: psql (PostgreSQL) 16.x

# 创建 CampusOS 数据库和用户
sudo -u postgres psql << 'SQL'
CREATE USER campusos WITH PASSWORD 'campusos_dev';
CREATE DATABASE campusos OWNER campusos;
GRANT ALL PRIVILEGES ON DATABASE campusos TO campusos;
\q
SQL

# 验证数据库连接
PGPASSWORD=campusos_dev psql -h localhost -U campusos -d campusos -c "SELECT 1;"
# 预期输出: ?column? | 1
```

### 4.2 Redis 7

```bash
# Ubuntu 22.04 默认仓库中的 Redis 版本可能较旧
# 安装 Redis 7 需要添加官方仓库

# 添加 Redis 官方 APT 仓库
curl -fsSL https://packages.redis.io/gpg | sudo gpg --dearmor -o /usr/share/keyrings/redis-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/redis-archive-keyring.gpg] https://packages.redis.io/deb $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/redis.list
sudo apt update

# 安装 Redis 7
sudo apt install -y redis

# 启动服务并设置开机自启
sudo systemctl start redis-server
sudo systemctl enable redis-server

# 验证
redis-cli ping
# 预期输出: PONG

redis-server --version
# 预期输出: Redis server v=8.x.x
```

### 4.3 NATS

```bash
# 下载 NATS Server
NATS_VERSION="2.10.9"
wget "https://github.com/nats-io/nats-server/releases/download/v${NATS_VERSION}/nats-server-v${NATS_VERSION}-linux-amd64.zip"

# 安装 NATS CLI 工具（用于管理 JetStream）
go install github.com/nats-io/natscli/nats@latest

# 启动 NATS（启用 JetStream，开发调试用）
# nats-server -js

# 验证 JetStream
```

### 4.4 MinIO

```bash

wget https://dl.min.io/server/minio/release/linux-amd64/minio
```

## 五、Docker 容器化

### 5.1 Docker Engine 安装

```bash
# ─── 第一步：卸载旧版本 ───
sudo apt remove -y docker docker-engine docker.io containerd runc 2>/dev/null || true

# ─── 第二步：添加 Docker 官方 GPG 密钥 ───
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# ─── 第三步：添加 Docker APT 仓库（Ubuntu 22.04 = jammy）───
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  jammy stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# ─── 第四步：安装 Docker Engine ───
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# ─── 第五步：将当前用户添加到 docker 组（免 sudo 执行 docker）───
sudo usermod -aG docker $USER
newgrp docker

# ─── 第六步：验证安装 ───
docker --version
# 预期输出: Docker version 25.x.x

docker compose version
# 预期输出: Docker Compose version v2.x.x

# 测试 Docker 是否正常工作
docker run --rm hello-world
# 预期输出: Hello from Docker! ...
```

### 5.2 Docker 镜像加速

中国大陆用户建议配置 Docker 镜像加速器：

```bash
# 创建 Docker 配置目录
sudo mkdir -p /etc/docker

# 写入镜像加速配置
sudo tee /etc/docker/daemon.json << 'EOF'
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me"
  ],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  }
}
EOF

# 重启 Docker 服务
sudo systemctl daemon-reload
sudo systemctl restart docker

# 验证镜像加速配置
docker info | grep -A 5 "Registry Mirrors"
```

## 七、代码质量工具


```yaml
    - gosimple
    - govet
    - ineffassign
    - staticcheck
    - unused
    - gocritic
    - gofmt
    - goimports
    - misspell
    - revive

  govet:
    shadow: true
  gocritic:
    enabled-tags:
      - diagnostic
      - style
      - performance

issues:
  exclude-use-default: false
  max-issues-per-linter: 50
  max-same-issues: 10
```

## 十、一键安装脚本

以下脚本专为 **Ubuntu 22.04 LTS** 编写，可一键安装所有开发工具：

```bash
#!/bin/bash
# install-dev-tools.sh - CampusOS 开发环境一键安装脚本 (Ubuntu 22.04)
set -e

GREEN='\033[0;32m'
YELLOW='\033[1;33m'
RED='\033[0;31m'
NC='\033[0m'

info()  { echo -e "${GREEN}[INFO]${NC} $1"; }
warn()  { echo -e "${YELLOW}[WARN]${NC} $1"; }
error() { echo -e "${RED}[ERROR]${NC} $1"; }

echo "=========================================="
echo "  CampusOS 开发环境安装脚本"
echo "  目标系统: Ubuntu 22.04 LTS (amd64)"
echo "=========================================="
echo ""

# ─── 检查系统 ───
if ! grep -q "Ubuntu 22.04" /etc/os-release 2>/dev/null; then
    warn "当前系统不是 Ubuntu 22.04，部分命令可能不兼容"
fi

# ─── 1. 系统依赖 ───
info "[1/8] 安装系统依赖..."
sudo apt update && sudo apt upgrade -y
sudo apt install -y git make curl wget unzip build-essential \
    ca-certificates gnupg lsb-release software-properties-common \
    apt-transport-https vim jq tree
sudo update-ca-certificates --fresh

# ─── 2. Go ───
GO_VERSION="1.22.5"
info "[2/8] 安装 Go ${GO_VERSION}..."
if ! command -v go &> /dev/null || [[ "$(go version)" != *"${GO_VERSION}"* ]]; then
    wget -q "https://go.dev/dl/go${GO_VERSION}.linux-amd64.tar.gz"
    sudo rm -rf /usr/local/go
    sudo tar -C /usr/local -xzf "go${GO_VERSION}.linux-amd64.tar.gz"
    rm "go${GO_VERSION}.linux-amd64.tar.gz"

    grep -q '/usr/local/go/bin' ~/.bashrc || {
        cat >> ~/.bashrc << 'ENVEOF'

# Go 环境变量
export PATH=$PATH:/usr/local/go/bin
export GOPATH=$HOME/go
export PATH=$PATH:$GOPATH/bin
ENVEOF
    }
    export PATH=$PATH:/usr/local/go/bin:$HOME/go/bin

    go env -w GOPROXY=https://goproxy.cn,direct
    go env -w GOSUMDB=sum.golang.google.cn
    go env -w GO111MODULE=on
fi
info "Go $(go version) 已就绪"

# ─── 3. Go 工具 ───
info "[3/8] 安装 Go 开发工具..."
export PATH=$PATH:$(go env GOPATH)/bin
go install golang.org/x/tools/gopls@latest
go install github.com/go-delve/delve/cmd/dlv@latest
go install go.uber.org/mock/mockgen@latest
go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@latest
go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest
go install github.com/air-verse/air@latest

# ─── 4. Node.js 20 LTS ───
info "[4/8] 安装 Node.js 20 LTS..."
if ! command -v node &> /dev/null; then
    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
    sudo apt install -y nodejs
fi
info "Node.js $(node --version) 已就绪"

# ─── 5. pnpm ───
info "[5/8] 安装 pnpm..."
npm install -g pnpm@8
pnpm config set registry https://registry.npmmirror.com
npm config set registry https://registry.npmmirror.com

# ─── 6. Docker ───
info "[6/8] 安装 Docker..."
if ! command -v docker &> /dev/null; then
    sudo apt remove -y docker docker-engine docker.io containerd runc 2>/dev/null || true
    sudo install -m 0755 -d /etc/apt/keyrings
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
    sudo chmod a+r /etc/apt/keyrings/docker.gpg
    echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu jammy stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
    sudo apt update
    sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
    sudo usermod -aG docker $USER
    warn "Docker 已安装，需要重新登录才能免 sudo 使用 docker"
else
    info "Docker $(docker --version) 已就绪"
fi

# ─── 7. protoc ───
info "[7/8] 安装 protoc..."
PROTOC_VERSION="25.1"
if ! command -v protoc &> /dev/null; then
    wget -q "https://github.com/protocolbuffers/protobuf/releases/download/v${PROTOC_VERSION}/protoc-${PROTOC_VERSION}-linux-x86_64.zip"
    sudo unzip -o "protoc-${PROTOC_VERSION}-linux-x86_64.zip" -d /usr/local > /dev/null
    rm "protoc-${PROTOC_VERSION}-linux-x86_64.zip"
fi
info "protoc $(protoc --version) 已就绪"

# ─── 8. NATS CLI ───
info "[8/8] 安装 NATS CLI..."
go install github.com/nats-io/natscli/nats@latest

# ─── 完成 ───
echo ""
echo "=========================================="
echo "  安装完成！"
echo "=========================================="
echo ""
echo "已安装的工具:"
echo "  Go:            $(go version 2>/dev/null || echo '未安装')"
echo "  Node.js:       $(node --version 2>/dev/null || echo '未安装')"
echo "  pnpm:          $(pnpm --version 2>/dev/null || echo '未安装')"
echo "  Docker:        $(docker --version 2>/dev/null || echo '未安装')"
echo "  protoc:        $(protoc --version 2>/dev/null || echo '未安装')"
echo "  golangci-lint: $(golangci-lint --version 2>/dev/null || echo '未安装')"
echo "  air:           $(air -v 2>/dev/null || echo '未安装')"
echo ""
echo "下一步:"
echo "  1. 重新加载 shell: source ~/.bashrc"
echo "  2. 如需使用 Docker，请重新登录或执行: newgrp docker"
echo "  3. 克隆项目: git clone https://github.com/your-org/CampusOS.git"
echo "  4. 启动环境: cd CampusOS && docker compose up -d"
echo "  5. 安装依赖: go mod download && cd web && pnpm install"
echo "  5. 启动开发: make dev"
```

**使用方式：**

```bash
# 保存并执行
chmod +x install-dev-tools.sh
./install-dev-tools.sh

# 重新加载环境变量
source ~/.bashrc
```
