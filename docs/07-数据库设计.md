## 设计规范

### 强制规范

| 规范 | 说明 |
|:---|:---|
| **分布式 ID** | 所有核心业务表强制使用全局唯一的分布式 ID（雪花算法），禁止自增 ID |
| **审计字段** | 所有表必须包含 `created_at`、`updated_at` 字段 |
| **软删除** | 禁止物理删除，必须保留 `deleted_at` 字段用于逻辑软删除 |
| **字符集** | 使用 UTF-8 编码，PostgreSQL 默认支持 |
| **命名规范** | 表名使用 snake_case 复数形式，字段名使用 snake_case |
| **时区** | 所有时间字段统一使用 UTC 时区存储 |

### 命名规范

| 对象 | 命名规则 | 示例 |
|:---|:---|:---|
| 表名 | snake_case 复数 | `users`, `threads`, `post_replies` |
| 字段名 | snake_case | `created_at`, `user_id`, `is_deleted` |
| 主键 | `id` | 统一使用雪花算法生成 |
| 外键 | `{关联表单数}_id` | `user_id`, `category_id` |
| 索引 | `idx_{表名}_{字段}` | `idx_users_email`, `idx_threads_category_id` |
| 唯一索引 | `uk_{表名}_{字段}` | `uk_users_username` |
| 联合索引 | `idx_{表名}_{字段1}_{字段2}` | `idx_user_roles_user_id_role_id` |

## 通用字段规范

所有业务表必须包含以下通用字段：

```sql
-- 通用字段模板
id          BIGINT PRIMARY KEY,        -- 雪花算法生成的分布式 ID
created_at  TIMESTAMP NOT NULL DEFAULT NOW(),  -- 创建时间
updated_at  TIMESTAMP NOT NULL DEFAULT NOW(),  -- 更新时间
deleted_at  TIMESTAMP NULL,            -- 软删除时间（NULL 表示未删除）
```

### 雪花 ID 生成器

```go
// Snowflake ID 结构 (64-bit)
// ┌─────────┬───────────┬──────────┬────────────┐
// │ 1 bit   │ 41 bits   │ 10 bits  │ 12 bits    │
// │ (符号位) │ (时间戳)   │ (机器ID) │ (序列号)    │
// └─────────┴───────────┴──────────┴────────────┘

// ID 生成器配置
type SnowflakeConfig struct {
    MachineID  uint16  // 机器 ID (0-1023)
    Epoch      int64   // 起始纪元时间戳 (2024-01-01 00:00:00 UTC)
}

// 使用示例
generator := snowflake.NewGenerator(SnowflakeConfig{
    MachineID: 1,
    Epoch:     1704067200000, // 2024-01-01
})
id := generator.Generate() // 返回 snowflake.ID 类型
```

## 核心表结构

### 身份域 (Identity Domain)

#### users - 用户表

```sql
CREATE TABLE users (
    id          BIGINT PRIMARY KEY,
    username    VARCHAR(32) NOT NULL,
    nickname    VARCHAR(64) NOT NULL,
    avatar      VARCHAR(512) DEFAULT '',
    bio         VARCHAR(500) DEFAULT '',
    status      VARCHAR(20) NOT NULL DEFAULT 'active',  -- active / suspended / deactivated
    created_at  TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMP NOT NULL DEFAULT NOW(),
    deleted_at  TIMESTAMP NULL
);

CREATE UNIQUE INDEX uk_users_username ON users(username) WHERE deleted_at IS NULL;
CREATE INDEX idx_users_status ON users(status) WHERE deleted_at IS NULL;
CREATE INDEX idx_users_created_at ON users(created_at) WHERE deleted_at IS NULL;
```

#### accounts - 账号表

```sql
CREATE TABLE accounts (
    id          BIGINT PRIMARY KEY,
    user_id     BIGINT NOT NULL,
    type        VARCHAR(20) NOT NULL,  -- email / phone / oauth
    identifier  VARCHAR(255) NOT NULL, -- 邮箱/手机号/oauth_provider+uid
    credential  VARCHAR(512) NOT NULL, -- 密码哈希或 OAuth token（加密存储）
    verified    BOOLEAN NOT NULL DEFAULT FALSE,
    created_at  TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMP NOT NULL DEFAULT NOW(),
    deleted_at  TIMESTAMP NULL
);

CREATE UNIQUE INDEX uk_accounts_type_identifier ON accounts(type, identifier) WHERE deleted_at IS NULL;
CREATE INDEX idx_accounts_user_id ON accounts(user_id) WHERE deleted_at IS NULL;
```

#### sessions - 会话表

```sql
CREATE TABLE sessions (
    id            BIGINT PRIMARY KEY,
    user_id       BIGINT NOT NULL,
    device_id     VARCHAR(128) NOT NULL,
    device_name   VARCHAR(128) DEFAULT '',
    device_type   VARCHAR(20) NOT NULL,  -- web / mobile / desktop / api
    refresh_token VARCHAR(255) NOT NULL,
    ip_address    VARCHAR(45) DEFAULT '',
    user_agent    VARCHAR(512) DEFAULT '',
    last_active_at TIMESTAMP NOT NULL DEFAULT NOW(),
    expires_at    TIMESTAMP NOT NULL,
    created_at    TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMP NOT NULL DEFAULT NOW(),
    deleted_at    TIMESTAMP NULL
);

CREATE UNIQUE INDEX uk_sessions_refresh_token ON sessions(refresh_token) WHERE deleted_at IS NULL;
CREATE INDEX idx_sessions_user_id ON sessions(user_id) WHERE deleted_at IS NULL;
CREATE INDEX idx_sessions_expires_at ON sessions(expires_at) WHERE deleted_at IS NULL;
```

```sql
    id            BIGINT PRIMARY KEY,
```

```sql
    author_id     BIGINT NOT NULL,

    deleted_at    TIMESTAMP NULL
```

## 索引设计规范

### 索引原则

| 原则 | 说明 |
|:---|:---|
| **软删除过滤** | 所有索引必须包含 `WHERE deleted_at IS NULL` 条件 |
| **选择性** | 优先为高选择性字段创建索引 |
| **复合索引** | 遵循最左前缀原则 |
| **覆盖索引** | 对高频查询尽量使用覆盖索引 |
| **GIN 索引** | JSONB 和数组字段使用 GIN 索引 |

### 禁止事项

- 禁止在低选择性字段（如 boolean）上单独创建索引
- 禁止创建冗余索引
- 禁止在频繁更新的字段上创建过多索引

## 数据迁移规范

### 迁移文件命名

```
migrations/
├── 000001_create_users_table.up.sql
├── 000001_create_users_table.down.sql
├── 000002_create_accounts_table.up.sql
├── 000002_create_accounts_table.down.sql
├── 000003_create_roles_table.up.sql
├── 000003_create_roles_table.down.sql
└── ...
```

### 迁移规范

| 规范 | 说明 |
|:---|:---|
| **每个迁移必须有回滚** | `.up.sql` 和 `.down.sql` 成对出现 |
| **迁移不可修改** | 已提交的迁移文件禁止修改 |
| **向后兼容** | 新增字段必须有默认值或允许 NULL |
| **数据迁移分离** | 结构迁移和数据迁移分开执行 |
| **幂等性** | 迁移脚本应支持重复执行 |

### 迁移工具

使用 `golang-migrate/migrate` 作为迁移工具：

```bash
# 创建新迁移
migrate create -ext sql -dir migrations -seq create_users_table

# 执行迁移
migrate -path migrations -database "postgres://..." up

# 回滚一步
migrate -path migrations -database "postgres://..." down 1

# 查看当前版本
migrate -path migrations -database "postgres://..." version
```

## 性能优化策略

### 读写分离

```
┌──────────┐     ┌──────────────────┐     ┌──────────────────┐
│ 应用服务  │────→│  PostgreSQL      │────→│  PostgreSQL      │
│          │     │  Primary (写)    │     │  Replica (读)    │
│          │     │                  │←────│                  │
│ 写请求    │────→│  INSERT/UPDATE   │     │  SELECT          │
│ 读请求    │────→│                  │     │  (异步复制)       │
└──────────┘     └──────────────────┘     └──────────────────┘
```

### 缓存策略

| 数据类型 | 缓存层 | TTL | 说明 |
|:---|:---|:---|:---|
| 用户信息 | Redis | 5 分钟 | 热点数据缓存 |
| 角色权限 | 进程内 + Redis | 5 + 15 分钟 | 多级缓存 |
| 版块配置 | Redis | 10 分钟 | 变更时主动刷新 |
| 帖子详情 | Redis | 2 分钟 | 热帖子缓存 |
| 帖子列表 | Redis | 30 秒 | 列表缓存（短 TTL） |
| 配置项 | 进程内 | 事件驱动刷新 | 监听 config.updated 事件 |

### 连接池配置

```go
// PostgreSQL 连接池配置
dbConfig := &pgxpool.Config{
    MaxConns:          25,               // 最大连接数
    MinConns:          5,                // 最小连接数
    MaxConnLifetime:   1 * time.Hour,    // 连接最大生命周期
    MaxConnIdleTime:   30 * time.Minute, // 空闲连接最大时间
    HealthCheckPeriod: 1 * time.Minute,  // 健康检查周期
}
```

### 查询优化

- **避免 N+1 查询**：使用 JOIN 或批量查询
- **延迟加载**：非关键字段（如 content）在详情接口才返回
- **分页优化**：使用游标分页替代深度分页（OFFSET 大值性能差）
- **批量操作**：使用批量 INSERT/UPDATE 减少数据库往返
