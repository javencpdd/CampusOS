## 3. 主要代码与文档变更

### 3.1 迁移与数据库

- `scripts/migrate.sh` / `scripts/migrate.ps1`：支持 `up`、`down`、`reset`、`status`。
- `schema_migrations`：记录已应用版本，重复执行 `migrate-up` 会跳过已完成迁移。
- `migrations/000001` 到 `000004`：补齐基础幂等处理，便于旧开发库回填迁移版本。
- `migrations/000005_plugin_schema_alignment.*.sql`：新增插件权限表 `plugin_permissions` 和插件日志表 `plugin_logs`。

### 3.2 Docker 与跨平台脚本

- `Makefile`：`docker-up` 改为调用 `scripts/docker-up.sh`；新增/保留 `migrate-status`、`docker-infra-up`、`docker-tools-up`。
- `scripts/docker-up.sh` / `scripts/docker-up.ps1`：准备 PostgreSQL、Redis、NATS；遇到宿主机端口占用时提示跳过或改端口。
- `scripts/dev.sh` / `scripts/dev.ps1`：先准备依赖服务，再执行迁移并启动 Go 后端。
- `docker-compose.yml`：PostgreSQL、Redis、NATS、pgAdmin 端口支持通过环境变量覆盖。

### 3.3 插件与 ID

- `internal/plugin/pg_repository.go`：插件和 API Key 记录在应用侧生成 `BIGINT` ID。
- `docs/help/ID策略说明.md`：记录当前 ID 策略、API 暴露规则和后续待办。

### 3.4 API 与测试

- `docs/api/openapi-v0.3-pre.yaml`：建立 v0.3-dev-pre OpenAPI 基线。
- `internal/community/handler/thread_handler_test.go`：新增 JWT 创建帖子回归测试，覆盖不依赖 `X-User-ID` 的发帖路径。

## 4. 使用方式

Ubuntu/WSL2 推荐流程：

```bash
make docker-up
make migrate-up
make migrate-status
go run ./cmd/server/main.go
```

原生 Windows PowerShell 基础流程：

```powershell
.\scripts\docker-up.ps1
.\scripts\migrate.ps1 up
go run .\cmd\server\main.go
```

如果本机已经有 PostgreSQL 或 Redis 占用默认端口，`make docker-up` 会提示跳过对应 Docker 服务。若希望强制使用 Docker 服务，可以停止本机服务，或在 `.env` 中设置非默认端口，例如：

```env
POSTGRES_PORT=15432
DB_PORT=15432
REDIS_PORT=16379
REDIS_ADDR=localhost:16379
```

## 6. 当前环境提示

本次验证发现当前开发机的 `localhost:5432` 和 `localhost:6379` 已经被本机服务占用，因此 Docker Compose 的 PostgreSQL/Redis 没有重新绑定默认端口。新的 `docker-up` 脚本已经把这种情况转化为明确提示，避免启动流程直接失败。

如果后续希望恢复“依赖服务全部由 Docker 提供”的模式，建议先确认并停止本机 PostgreSQL/Redis，或使用 `.env` 为 Docker 服务和迁移脚本指定一组新端口。
