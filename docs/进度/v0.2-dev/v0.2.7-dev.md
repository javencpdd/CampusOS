## 核心功能详解

### 1. 帖子列表缓存

**缓存策略：**
- **缓存键**：`threads:list:{page}:{pageSize}:{status}`
- **缓存条件**：仅缓存无筛选条件（无 keyword/categoryID/authorID）的查询
- **TTL**：5 分钟
- **命中日志**：缓存命中时输出 `📦 缓存命中: threads:list:1:20:published`

**缓存失效：**
- 创建帖子 → 清除列表缓存
- 更新帖子 → 清除列表缓存
- 删除帖子 → 清除列表缓存
- 失效策略：清除常见的分页缓存 key（1:20、1:10、1:5）

**缓存层次：**
```
请求 → ThreadService.ListThreads()
        ├── 缓存命中 → 直接返回（跳过数据库查询）
        └── 缓存未命中 → 查询数据库 → 写入缓存 → 返回
```

### 2. server.go 集成

- 缓存实例 `appCache` 在 PostgreSQL 初始化之前创建
- 通过 `threadSvc.SetCache(appCache)` 注入到帖子服务
- Memory 模式下不注入缓存（使用默认的无缓存行为）
