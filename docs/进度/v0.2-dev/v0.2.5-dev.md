## 核心功能详解

### 1. JWT 登录响应注入角色信息

**后端改动：**
- `LoginResponse` 新增 `roles` 字段，类型为 `[]RoleInfo`
- `UserService` 新增 `RoleQuerier` 接口和 `SetRoleRepository()` 方法
- `UserHandler.Login()` 在生成 Token 后查询用户角色并返回
- 两种运行模式（PostgreSQL/Memory）均通过 `server.go` 注入 `roleRepo`

**登录响应示例：**
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "user": { "id": "...", "username": "admin", "nickname": "管理员" },
    "roles": [{ "id": 1, "name": "admin", "description": "系统管理员" }],
    "access_token": "eyJ...",
    "refresh_token": "...",
    "token_type": "Bearer",
    "expires_in": 7200
  }
}
```

### 2. 管理后台 Store 增强

- 登录时自动将角色信息注入到 `admin_user` 对象中
- 新增计算属性：
  - `primaryRole`：用户主角色名（第一个角色，默认 member）
  - `isAdmin`：是否为管理员（admin/super_admin）
  - `isModerator`：是否为版主或管理员

### 3. 路由守卫角色验证

- 读取 `admin_user` 中的 `roles[0].name` 进行前端角色验证
- 允许访问的角色：`admin`、`super_admin`、`moderator`
- 非授权角色访问时输出 console.warn 警告
- 后端 `RequirePermission` 中间件做真正的权限校验

### 4. 帖子审核管理页面（AdminReviews）

**三个标签页：**
- **待审核**：显示 `pending_review` 状态的帖子，支持通过/拒绝/查看详情
- **已发布**：显示 `published` 状态的帖子
- **已拒绝**：显示 `rejected`/`archived` 状态的帖子

**审核操作：**
- **通过**：调用 `PUT /threads/:id` 将状态改为 `published`
- **拒绝**：调用 `PUT /threads/:id` 将状态改为 `rejected`（带 Popconfirm 确认）
- **查看详情**：弹出对话框显示帖子完整信息（标题、作者、版块、时间、内容）

**侧边栏导航：**
- 新增"帖子审核"菜单项，使用 Stamp 图标，位于插件管理和事件日志之间
