## 本次更新内容

本次更新实现了**前端双项目架构拆分**，将管理后台从用户前台中独立出来，成为两个完全独立的 Vue 项目，同时完成了管理后台全部 6 个页面的开发。

### 架构变更

**从单体前端拆分为两个独立项目：**

```
CampusOS/
├── web/          # 用户前台（端口 :3000）— 纯用户界面
├── admin/        # 管理后台（端口 :3001）— 独立管理项目 [新增]
└── internal/     # 后端 API（端口 :8080）— 两个前端共用
```

## 核心功能详解

### 1. 前端双项目架构

- **用户前台 `web/`**：端口 3000，仅包含用户相关页面（首页/登录/注册/帖子列表/详情/发帖）
- **管理后台 `admin/`**：端口 3001，独立的登录界面和管理页面
- **共用后端 API**：两个前端都通过 Vite Proxy 代理 `/api` 到 `http://localhost:8080`
- **独立认证**：各自使用独立的 localStorage key 存储 Token（`access_token` vs `admin_token`）
- **独立构建**：各自有独立的 `package.json`、`vite.config.ts`、`tsconfig.json`

### 2. 管理后台布局（AdminLayout）

- **左侧导航栏**：深色主题（#304156），6 个菜单项，每项带 Element Plus 图标
- **顶部面包屑**：根据路由 `meta.title` 动态生成当前页面标题
- **顶部右侧**：显示当前登录用户名（带图标）、"退出"按钮
- **路由路径**：管理后台根路径为 `/`（非 `/admin`），子路由为 `/users`、`/threads` 等

### 3. 管理员登录页（LoginView）

- **独立登录界面**：渐变背景（#304156 → #1a2332），居中卡片布局
- **登录逻辑**：调用 `POST /auth/login`，Token 存储到 `admin_token`
- **已登录跳转**：路由守卫检测到已登录自动跳转到仪表盘
- **重定向**：支持 `?redirect=` 参数，登录后跳转到目标页面

### 4. 仪表盘（DashboardView）

- **统计卡片**：4 张卡片分别展示用户总数、帖子总数、版块数量、插件数量
- **快捷操作**：5 个按钮快速跳转到各管理模块
- **最新帖子/事件表格**：展示最近数据，带状态标签和事件类型标签

### 5. 用户管理（UserManageView）

- **用户列表**：展示 ID、用户名、昵称、邮箱、状态，分页（每页 20 条）
- **封禁/解封**：带 Popconfirm 二次确认
- **状态中文**：`active` → "正常"（绿色），其他 → "已封禁"（红色）

### 6. 帖子管理（ThreadManageView）

- **帖子列表**：展示 ID、标题（带置顶/锁定标签）、作者、浏览数、状态
- **搜索与筛选**：关键词搜索 + 版块下拉筛选
- **管理操作**：置顶/取消置顶、锁定/取消锁定、删除（均带确认）

### 7. 版块管理（CategoryManageView）

- **CRUD 对话框**：新建/编辑版块，支持名称、描述、图标（emoji）、颜色（颜色选择器）、排序
- **删除确认**：Popconfirm 二次确认

### 8. 插件管理（PluginManageView）

- **启用/禁用**：`el-switch` 切换，失败时自动回滚状态
- **卸载**：Popconfirm 二次确认
- **空状态**：无插件时显示 `el-empty` 提示

### 9. 事件日志（EventLogView）

- **类型筛选**：下拉选择器按事件类型过滤
- **数据详情**：弹出对话框展示事件完整信息和 JSON 数据
- **类型颜色**：user.* 蓝色、thread.* 绿色、post.* 橙色、category.* 灰色

## 启动方式

```bash
# 1. 启动后端（端口 8080）
cd CampusOS && go run cmd/server/main.go

# 2. 启动用户前台（端口 3000）
cd web && pnpm install && pnpm dev

# 3. 启动管理后台（端口 3001）
cd admin && pnpm install && pnpm dev
```

访问地址：
- 用户前台：`http://localhost:3000/`
- 管理后台：`http://localhost:3001/`
