## 本次更新内容

本次更新聚焦 **插件仓储集成 + API Key 中间件集成**，完成了 PluginManager 和 API Key 中间件从 Memory 实现到 PostgreSQL 持久化的升级。

### 修改文件

| 文件 | 说明 |
|:---|:---|
| `internal/plugin/manager.go` | Manager 新增 `SetPluginRepository()` 方法，支持注入持久化仓储 |
| `internal/server/server.go` | PostgreSQL 模式下注入 PgPluginRepository 和 PgAPIKeyRepository |

---

## 核心功能详解

### 1. PluginManager 持久化仓储集成

**Manager 结构体新增字段：**
```go
type Manager struct {
    // ... 原有字段
    repo PluginRepository // 可选：插件持久化仓储
}
```

**新增方法：**
```go
func (m *Manager) SetPluginRepository(repo PluginRepository)
```

**server.go 集成（PostgreSQL 模式）：**
```go
pluginRepo = plugin.NewPgPluginRepository(pool)
apiKeyRepo = plugin.NewPgAPIKeyRepository(pool)
s.manager.SetPluginRepository(pluginRepo)
```

**数据流向：**
```
PostgreSQL 启动成功
  ├── PgPluginRepository → Manager.SetPluginRepository()
  ├── PgAPIKeyRepository → apiKeyRepo（供 API Key 中间件使用）
  └── 种子数据 SeedAdmin()
```

### 2. API Key 仓储集成

**server.go 中声明了 apiKeyRepo 变量：**
- PgAPIKeyRepository 在 PostgreSQL 模式下初始化
- Memory 模式下使用 MemoryAPIKeyRepository（原有行为不变）
- apiKeyRepo 可供后续 API Key 中间件注入使用

### 3. 运行模式对比

| 维度 | PostgreSQL 模式 | Memory 模式（回退） |
|:---|:---|:---|
| 插件仓储 | PgPluginRepository | MemoryPluginRepository |
| API Key 仓储 | PgAPIKeyRepository | MemoryAPIKeyRepository |
| 数据持久化 | ✅ PostgreSQL | ❌ 内存（重启丢失） |
| 种子数据 | ✅ 自动创建管理员 | ❌ 无 |
