## 核心功能详解

### 1. 默认管理员账号

**登录信息：**

| 字段 | 值 |
|:---|:---|
| 邮箱 | `admin@campusos.local` |
| 密码 | `Admin@123456` |
| 用户名 | `admin` |
| 昵称 | 系统管理员 |
| 角色 | admin（系统管理员，拥有全部权限） |

### 2. 种子数据机制（seed.go）

**启动时自动执行流程：**

```
server.Run()
  └── PostgreSQL 连接成功
        └── SeedAdmin(pool)
              ├── 查询是否已有 admin 用户
              ├── 已存在 → 跳过，输出 "✅ 管理员账号已存在"
              └── 不存在 →
                    ├── 生成 bcrypt 哈希（密码 Admin@123456）
                    ├── 插入 users 表（固定雪花 ID）
                    ├── 插入 accounts 表（邮箱 + 密码哈希）
                    ├── 插入 user_roles 表（分配 admin 角色）
                    └── 输出 "✅ 默认管理员账号已创建"
```

**关键设计：**
- **幂等性**：使用 `ON CONFLICT DO NOTHING`，重复执行不会报错
- **自动哈希**：使用 `auth.HashPassword()` 生成 bcrypt 哈希，无需手动计算
- **固定 ID**：管理员使用固定雪花 ID（1000000000000000001），便于引用
- **角色自动分配**：启动时自动分配 admin 角色（role_id=1），确保管理后台可登录
- **容错**：种子数据失败仅输出警告日志，不影响服务启动

### 3. 数据库迁移脚本

**000004_seed_admin.up.sql：**
- 插入管理员用户到 `users` 表
- 插入账号凭据到 `accounts` 表
- 分配 admin 角色到 `user_roles` 表
- 使用 `ON CONFLICT DO NOTHING` 保证幂等

**000004_seed_admin.down.sql：**
- 按外键依赖顺序删除：`user_roles` → `accounts` → `users`

### 4. 数据持久化确认

**PostgreSQL 数据持久化机制：**
- Docker Compose 中 PostgreSQL 使用 volume 挂载持久化数据
- 种子数据通过 SQL 迁移和 Go 代码双重保障
- 服务重启不会丢失数据（`ON CONFLICT DO NOTHING` 保护）
