## 核心功能详解

### 1. Redis 缓存接入

**新增 RedisCache 实现：**
- 基于 `github.com/redis/go-redis/v9` 客户端
- 实现完整 `Cache` 接口：`Get`、`Set`、`Delete`、`Exists`、`Close`
- 新增 `Ping()` 方法用于连接测试

**工厂方法 `NewCache()` 升级：**
- 支持通过环境变量配置 Redis 连接
- 启动时自动 Ping 测试连接，失败时回退到内存缓存
- 日志输出连接状态（✅ 或 ⚠️ 回退提示）

**配置项：**

| 环境变量 | 默认值 | 说明 |
|:---|:---|:---|
| `REDIS_ENABLED` | `true` | 是否启用 Redis 缓存 |
| `REDIS_ADDR` | `localhost:6379` | Redis 服务地址 |
| `REDIS_PASSWORD` | （空） | Redis 密码 |

**Cache 接口新增 `Close()` 方法：**
- 所有缓存实现（Redis / Memory / Noop）均实现 `Close()` 方法
- 确保连接资源正确释放

### 2. PG 插件仓储

**PgPluginRepository（插件数据持久化）：**
- 实现 `PluginRepository` 接口的全部 5 个方法
- `Save`：INSERT ... ON CONFLICT（upsert 语义）
- `GetByName`：按名称查询插件记录
- `List`：查询所有插件，按安装时间倒序
- `UpdateStatus`：更新插件状态和错误信息
- `Delete`：软删除（设置 deleted_at）

**PgAPIKeyRepository（API Key 持久化）：**
- 实现 `APIKeyRepository` 接口的全部 6 个方法
- `Create`：创建 API Key，权限以 JSON 格式存储
- `GetByKey`：按 Key 查询，自动解析 JSON 权限
- `ListByUser`：查询用户的所有 API Key
- `Deactivate`：停用 API Key
- `UpdateLastUsed`：更新最后使用时间
- `Delete`：软删除

### 3. server.go 集成

**缓存初始化：**
- 在 PostgreSQL 初始化之前创建缓存实例
- 从 `Config.Redis` 读取配置（Addr、Password、DB、Enabled）
- Redis 不可用时自动回退到 MemoryCache

**配置管理增强：**
- `RedisConfig` 新增 `Password`、`DB`（默认 0）、`Enabled`（默认 true）字段
- 支持通过 `REDIS_ENABLED=false` 完全禁用 Redis
