# CampusOS v0.2 阶段总结

> **阶段**：v0.2.0 ~ v0.2.9（共 10 个版本）
> **核心目标**：RBAC 权限系统 + 插件系统增强 + 管理后台 + 前端双项目架构 + 基础设施完善

---

## 一、阶段概述

v0.2 阶段是 CampusOS 从"基础功能可用"迈向"具备完整管理能力"的关键阶段。在 v0.1 完成后端骨架、数据库、前端页面、JWT 认证、版块管理、回复系统、事件总线和插件框架的基础上，v0.2 围绕 **权限管控**、**插件系统增强**、**管理后台 UI**、**前端架构升级**、**基础设施完善** 五大方向展开了系统性建设。

本阶段共完成 **10 个版本迭代**（v0.2.0 ~ v0.2.9），API 端点从 19 个增长至 **40 个**，前端从单体项目拆分为 **用户前台 + 管理后台** 双项目架构，后端引入了 RBAC 权限系统、Redis 缓存、PostgreSQL 插件仓储、API Key 认证等关键基础设施。

---

```
v0.2.0 (Host API) → v0.2.1 (依赖注入重构) → v0.2.2 (API Key + 缓存层)

```

## 三、各版本详细交付

### v0.2.0 — Host API 完整实现

**核心交付**：为插件系统提供了完整的 Host API 能力，使插件可以反向调用核心系统功能。

- **Host API 8 个接口**：
  - `GetUser` — 查询用户信息
  - `GetThread` — 查询帖子信息
  - `PublishEvent` — 发布事件到 Event Bus
  - `SendNotification` — 发送通知
  - `GetConfig` / `SetConfig` — 配置读写
  - `CheckPermission` — 权限检查
  - `Log` — 日志记录
- **角色表迁移**（000002）：`roles`、`user_roles`、`role_permissions` 三张表
- **4 个内置角色**：admin / moderator / member / guest

### v0.2.1 — 插件系统依赖注入重构

**核心交付**：重构插件系统的依赖注入方式，提升可测试性和可维护性。

- Plugin Manager 依赖注入重构，解耦各组件
- 插件生命周期管理增强（注册→启动→停止→卸载）
- gRPC Runtime 连接管理优化

### v0.2.2 — API Key 认证 + 缓存层

**核心交付**：引入 API Key 认证机制和缓存抽象层。

- **插件表迁移**（000003）：`plugins`、`api_keys` 两张表
- **API Key 认证中间件**：`pkg/middleware/apikey.go`，支持第三方/插件调用
- **缓存层抽象**：`pkg/cache/cache.go`，定义统一 Cache 接口
  - `MemoryCache` — 内存缓存实现（开发环境）
  - `NoopCache` — 空操作缓存（测试环境）
  - `RedisCache` — 占位，待 v0.2.6 实现

### v0.2.3 — 前端双项目架构拆分 + 管理后台 UI

**核心交付**：将单体前端拆分为用户前台和管理后台两个独立项目，完成管理后台全部 7 个页面。

- **前端架构拆分**：
  - `web/`（:3000）— 用户前台，6 个页面（首页/登录/注册/帖子列表/详情/发帖）
  - `admin/`（:3001）— 管理后台，独立登录和管理界面
  - 两者独立 `package.json` / `vite.config.ts` / `tsconfig.json`，共用后端 API
- **管理后台 7 个页面**：
  1. **LoginView** — 渐变背景独立登录，admin_token 存储隔离
  2. **DashboardView** — 4 张统计卡片 + 快捷操作 + 最新数据表格
  3. **UserManageView** — 搜索 + 封禁/解封（Popconfirm 确认）+ 分页
  4. **ThreadManageView** — 搜索/筛选 + 置顶/锁定/删除
  5. **CategoryManageView** — 完整 CRUD 对话框 + 图标/颜色/排序
  6. **PluginManageView** — el-switch 启用/禁用（失败自动回滚）+ 卸载
  7. **EventLogView** — 类型筛选 + JSON 数据详情弹窗 + 类型颜色区分
- **AdminLayout**：左侧深色导航栏（#304156）+ 面包屑 + 用户信息
- **新增文件**：16 个（admin 项目完整结构）

### v0.2.4 — RBAC 权限系统前端集成

**核心交付**：完成 RBAC 角色管理的后端 API 和前端集成。

- **新增 4 个角色管理 API**（端点总数 36 → 40）：
  - `GET /api/v1/roles` — 列出所有角色
  - `GET /api/v1/users/:id/roles` — 获取用户角色
  - `POST /api/v1/users/:id/roles` — 分配角色
  - `DELETE /api/v1/users/:id/roles` — 撤销角色
- **24 条权限规则**的权限矩阵
- **前端集成**：
  - 用户管理页新增角色筛选下拉框
  - 用户列表显示当前角色标签
  - 角色分配对话框（多选 Checkbox）
- **路由守卫**：管理后台路由检查 admin_token（仅检查登录状态，角色检查待完善）

### v0.2.5 — 帖子审核队列

**核心交付**：实现帖子审核流程，支持管理员审核帖子发布。

- 帖子状态机增强：`draft` → `pending_review` → `published` / `rejected` → `archived`
- 管理后台审核页面：展示待审核帖子列表，支持通过/拒绝操作
- 审核操作触发事件（`thread.approved` / `thread.rejected`），可通过 Event Bus 被插件监听

### v0.2.6 — Redis 缓存接入 + PG 插件仓储

**核心交付**：完成 Redis 缓存的实际接入和插件系统的 PostgreSQL 持久化。

- **RedisCache 实现**：
  - 基于 `github.com/redis/go-redis/v9`
  - 完整 Cache 接口：`Get` / `Set` / `Delete` / `Exists` / `Close`
  - 新增 `Ping()` 连接测试方法
  - 工厂方法 `NewCache()` 支持环境变量配置
  - 启动时自动 Ping 测试，失败时回退到内存缓存
  - 环境变量：`REDIS_ENABLED`(默认true) / `REDIS_ADDR`(默认localhost:6379) / `REDIS_PASSWORD`
- **PgPluginRepository**：实现 `PluginRepository` 接口全部 5 个方法（Save/GetByName/List/UpdateStatus/Delete）
- **PgAPIKeyRepository**：实现 `APIKeyRepository` 接口全部 6 个方法（Create/GetByKey/ListByUser/Deactivate/UpdateLastUsed/Delete）
- **server.go 集成**：PostgreSQL 初始化前创建缓存实例，RedisConfig 新增 Password/DB/Enabled 字段

### v0.2.7 — 系统统计 API

**核心交付**：提供系统级统计数据接口，支撑管理后台仪表盘。

- 新增系统统计 API：返回用户总数、帖子总数、版块总数、插件总数等
- 管理后台仪表盘数据展示增强：统计卡片展示实时数据
- 快捷操作按钮功能完善

### v0.2.8 — 插件系统前端集成完善

**核心交付**：完善管理后台中插件管理和事件日志功能。

- 插件管理页面功能完善：插件详情展示、状态管理
- 事件日志查看增强：事件类型筛选、数据详情 JSON 格式化展示
- 事件类型颜色区分（user 蓝 / thread 绿 / post 橙 / category 灰）

### v0.2.9 — 整体优化 + Bug 修复

**核心交付**：v0.2 阶段收尾，整体优化和文档更新。

- 已知 Bug 修复
- 代码质量优化
- 文档更新（README、API 文档等）
- v0.2 阶段验收

## 六、关键技术决策

| 决策 | 选择 | 原因 |
|:---|:---|:---|
| ORM | pgx（原生 SQL） | 相比 GORM 性能更好、SQL 更可控、减少抽象层 |
| 缓存回退策略 | Redis → Memory → Noop | 渐进式集成，开发环境无需 Redis 依赖 |
| 前端架构 | 双项目独立部署 | 用户前台和管理后台关注点分离，独立迭代 |
| 认证存储隔离 | web: `token` / admin: `admin_token` | 防止前台用户误入后台，独立认证域 |
| 插件仓储 | PostgreSQL 持久化 | 统一数据源，支持软删除和审计 |
| 权限中间件 | JWT Claims → 角色查询 → 权限矩阵匹配 | 无外部依赖，权限矩阵存储在数据库中 |

## 九、v0.2 → v0.3 变更预期

| 维度 | v0.2 状态 | v0.3 目标 |
|:---|:---|:---|
| 插件运行时 | 仅 gRPC | gRPC + Wasm 双运行时 |
| 权限模型 | RBAC（角色级） | RBAC + 三层权限（API/资源/字段级） |
| 插件存储 | PostgreSQL 统一存储 | SQLite 独立 + PostgreSQL 共享 可选 |
| 开发者工具 | 手动创建插件 | SDK + CLI 脚手架工具 |
| 示例插件 | 1 个 gRPC (hello-plugin) | + 2 个 Wasm 示例 |
| 隔离级别 | 进程级（gRPC） | 内存级沙箱（Wasm） + 进程级（gRPC） |
