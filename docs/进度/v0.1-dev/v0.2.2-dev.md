### 核心功能

#### 1. plugins 数据库表

```sql
-- plugins 表：插件元数据持久化
CREATE TABLE plugins (
    id, name, display_name, version, description, author,
    runtime, manifest(JSONB), status, api_key, config(JSONB),
    error_message, installed_by, installed_at, updated_at, deleted_at
);

-- api_keys 表：独立 API Key 管理
CREATE TABLE api_keys (
    id, key, name, user_id, plugin_name,
    permissions(JSONB), is_active, last_used_at, expires_at,
    created_at, deleted_at
);
```

#### 2. 插件仓储层

- **PluginRepository 接口**：Save / GetByName / List / UpdateStatus / Delete
- **APIKeyRepository 接口**：Create / GetByKey / ListByUser / Deactivate / UpdateLastUsed / Delete
- **MemoryPluginRepository / MemoryAPIKeyRepository**：内存实现，服务启动时使用
- **GenerateAPIKey()**：生成 `pk_` 前缀的随机 API Key

#### 3. API Key 认证中间件

- 支持 `Authorization: Bearer pk_xxx` 和 `X-API-Key: pk_xxx` 两种方式
- 验证 Key 有效性、是否激活、是否过期
- 注入 API Key 信息到 Gin context（`api_key`、`api_key_name`、`api_key_plugin`）
- 异步更新最后使用时间

#### 4. 缓存层

- **Cache 接口**：Get / Set / Delete / Exists
- **MemoryCache**：带 TTL 的内存缓存（Redis 不可用时回退）
- **NoopCache**：空实现（缓存未启用时）
- **NewCache(cfg)**：工厂方法，根据配置自动选择实现
- Redis 实现预留接口，未来可无缝接入 `go-redis`
