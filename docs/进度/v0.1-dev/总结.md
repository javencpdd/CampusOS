# CampusOS 项目全景分析

## 一、项目定位
CampusOS 是基于 Go 构建的面向高校场景的**开放式社区平台**，定位为**"社区操作系统 (Community OS)"**而非传统论坛。官方论坛仅是运行在该 OS 上的一个基础 App。

## 二、四大设计理念
1. **API First**：OpenAPI 3.0 + Protobuf 定义先于实现
2. **Plugin First**：核心只提供原子能力，业务通过插件实现
3. **Event Driven**：所有核心行为通过 NATS Event Bus 广播，采用 CloudEvents 1.0 规范
4. **AI Native**：原生 MCP 协议 + 统一大模型网关，AI 作为一等公民操作者

## 三、技术栈
- **后端**：Go 1.22+ / Gin 1.9+ / GORM 1.25+ / pgx 5.x
- **数据库**：PostgreSQL 16+ / Redis 7+ / NATS 2.x (JetStream)
- **前端**：Vue 3.4+ / TypeScript 5.3+ / Vite 5.x / Element Plus 2.x / Pinia 2.x
- **插件运行时**：Wazero 1.7+ (Wasm) + gRPC-Go 1.60+
- **容器化**：Docker 25+ / Docker Compose v2
- **可观测性**：OpenTelemetry + Prometheus + Grafana + Jaeger

## 四、六层分层架构
```
L1 Presentation  → RESTful API / Web UI
L2 Gateway       → 统一鉴权 / 限流 / 路由
L3 Core          → Identity / Permission / Storage / Audit / Notification / Configuration
L4 Platform      → Event Bus (NATS) / Task Queue (Asynq) / Cache (Redis)
L5 Plugin Runtime → Wasm (Wazero) + gRPC 双运行时
L6 Infrastructure → PostgreSQL / Redis / MinIO / NATS / OpenTelemetry
```

## 五、核心服务设计

### 5.1 Identity 身份服务
- **User 与 Account 解耦**：User=人，Account=登录方式，支持邮箱/手机/OAuth多账号绑定
- **Token**：Access Token (JWT, 2h) + Refresh Token (256-bit, 30天, Redis)
- **多端管理**：Web浏览器指纹+IP、移动端设备ID、API Key

### 5.2 Permission 权限中心
- **RBAC + ABAC 混合模型**
- **7个内置角色**：super_admin(100) > admin(80) > moderator(60) > vip(40) > user(20) > guest(5) > banned(0)
- **三段式权限表达**：`resource:action:scope`（如 `thread:delete:moderated`）
- **三层权限控制**：API级 → 资源级 → 字段级
- **合并策略**：显式 DENY 优先于 ALLOW，默认拒绝

### 5.3 Storage 存储引擎
- **VFS 统一抽象**：支持 Local / S3(MinIO) / OSS
- **SHA-256 秒传检测** + 文件类型白名单 + 50MB 大小限制

### 5.4 Audit 审计中心
- **防篡改设计**：只允许 INSERT + HMAC-SHA256 签名 + 链式哈希
- 支持 Actor 类型：user / system / plugin / ai_agent

### 5.5 Notification 通知服务
- **多渠道**：站内信、邮件、WebSocket、短信、Webhook
- **模板引擎**：YAML 格式，支持变量插值
- **事件驱动**：自动监听事件触发通知

### 5.6 Configuration 配置中心
- **热更新**：PostgreSQL → Event Bus → 所有实例刷新本地缓存
- 双层缓存（本地+Redis），支持配置回滚

## 六、插件系统

### 6.1 双运行时架构
| 维度 | Wasm Runtime | gRPC Runtime |
|:---|:---|:---|
| 隔离 | 内存级沙箱 | 进程级网络隔离 |
| 资源 | 极低(MB级) | 中等(独立进程) |
| 场景 | 事件监听、内容审核 | IM适配器、长连接、RAG |
| 语言 | Go/Rust/TypeScript | 任意gRPC语言 |
| 生命周期 | 随事件触发,用完销毁 | 长期驻留,独立管理 |

### 6.2 Manifest 规范（plugin.yaml）
- apiVersion: campusos/v1alpha1
- 声明 metadata(名称/版本/作者)、spec(运行时配置)、permissions(事件订阅+Host API权限)、storage(存储类型)

### 6.3 插件数据存储
- **独立 SQLite**（推荐，文件级隔离，默认100MB上限）
- **共享 PostgreSQL**（表级隔离，plugin_ 前缀）
- **无存储**（无状态插件）

### 6.4 数据获取模式
- **事件驱动（被动）**：订阅 Event Bus 事件
- **查询模式（主动）**：通过 Host API 查询，三层权限过滤
- **实时流（长连接）**：gRPC StreamEvents

## 七、事件系统

- **基于 NATS JetStream** + **CloudEvents 1.0** 规范
- **双模式**：`.before`（同步拦截，可 Block/Modify，超时5s）+ `.after`（异步通知，超时30s）
- **可靠性**：At-least-once + 死信队列 + 指数退避重试（1s→2s→4s→8s，最大5次）
- **双实现**：NATS JetStream（生产）+ Memory（开发/测试回退）

## 八、API 设计

- **URL格式**：`/api/{version}/{resource}[/{id}][/{sub-resource}]`
- **统一响应**：`{code: 0, msg: "success", data: {...}}`
- **错误码体系**：1xxxx(认证)、2xxxx(用户)、3xxxx(帖子)、4xxxx(版块)、5xxxx(回复)、6xxxx(插件)、7xxxx(系统)
- **认证**：JWT + API Key + HMAC 签名

### 已实现 19 个 API 端点：
- 认证(4)：注册、登录、获取当前用户、健康检查
- 用户(3)：列表、详情、更新
- 帖子(5)：列表(搜索/分页/过滤)、详情(浏览计数)、创建、更新(仅作者)、删除(软删除)
- 版块(4)：列表、详情、创建、更新
- 回复(2)：列表(分页)、创建
- 事件(1)：事件历史查询

## 九、数据库设计（11张表，3大域）

- **身份域**：users、accounts、sessions
- **社区域**：categories、threads、posts、tags、likes
- **系统域**：audit_logs、notifications、configurations
- **设计约束**：全表雪花ID、全表软删除、全表审计字段、GIN索引、JSONB字段
