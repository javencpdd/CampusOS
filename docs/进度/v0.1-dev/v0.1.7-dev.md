# CampusOS v0.1.7-dev 发布说明

> **版本**：v0.1.7-dev
> **发布日期**：2024-06-24
> **上一版本**：v0.1.6-dev
> **状态**：开发中

---

## 版本概述

v0.1.7-dev 在 v0.1.6-dev（Event Bus 接口）基础上，将事件总线**集成到业务逻辑中**，实现了"万物皆事件"的运行时能力。当用户注册时，系统自动广播 `user.created` 事件；事件日志订阅器实时记录所有核心事件。

## 核心功能变更

### 1. 事件类型常量

```go
const (
    EventUserCreated     = "user.created"
    EventUserUpdated     = "user.updated"
    EventThreadCreated   = "thread.created"
    EventThreadUpdated   = "thread.updated"
    EventThreadDeleted   = "thread.deleted"
    EventPostCreated     = "post.created"
    EventCategoryCreated = "category.created"
)
```

### 2. 业务层事件发布

UserService 注册成功后自动发布事件：

```go
// 在 Register() 方法末尾
if s.bus != nil {
    _ = s.bus.Publish(ctx, eventbus.NewEvent(
        eventbus.EventUserCreated, "campusos.identity", "user."+userID, user,
    ))
}
```

### 3. 默认事件订阅器（Event Logger）

服务启动时自动注册 5 个事件订阅器，在控制台打印事件日志：

```
✅ 已注册 5 个默认事件订阅
📢 Event: user.created | Subject: user.xxx | Source: campusos.identity
📢 Event: thread.created | Subject: thread.xxx | Source: campusos.community
```

### 4. EventBus 初始化流程

```
启动 → 尝试连接 NATS
         ├── 成功 → 使用 NATSEventBus（持久化/集群）
         └── 失败 → 使用 MemoryEventBus（内存回退）
       → 注册默认事件订阅器
       → 传递给 UserService/ThreadService
```

## 快速体验

```bash
# 启动服务
cd ~/bbs/bbs01/CampusOS
./campusos-server

# 终端输出:
# ⚠️  NATS 连接失败，回退到内存事件总线: ...
# ✅ 已注册 5 个默认事件订阅
# 🚀 CampusOS API 监听 0.0.0.0:8080

# 注册用户（触发 user.created 事件）
curl -s -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"jack","nickname":"Jack","email":"jack@example.com","password":"123456"}' | jq .

# 终端会输出:
# 📢 Event: user.created | Subject: user.xxx | Source: campusos.identity
```
