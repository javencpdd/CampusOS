## 数据库设计亮点

### 遵循的设计规范

- ✅ 所有表使用 BIGINT 主键（雪花算法，预留）
- ✅ 所有表包含 `created_at`、`updated_at`、`deleted_at` 审计字段
- ✅ 禁止物理删除，使用软删除
- ✅ 部分索引（`WHERE deleted_at IS NULL`）优化查询性能
- ✅ GIN 索引支持数组字段（帖子标签）
- ✅ JSONB 字段支持灵活扩展（帖子元数据）
- ✅ 审计日志表无 `deleted_at`（只写不删）

### 索引覆盖

- 用户：用户名唯一、邮箱唯一、状态、创建时间
- 帖子：作者、版块、状态、置顶、创建时间、最后回复时间、标签（GIN）
- 回复：帖子、作者、父回复、创建时间
- 版块：slug 唯一、父版块、排序

## 迁移到 v0.1.1-dev

### 前提条件

- PostgreSQL 16 已安装并运行（原生或 Docker）
- 已创建 campusos 数据库和用户

### 数据库初始化

```bash
# 创建数据库和用户
sudo -u postgres psql << 'SQL'
CREATE USER campusos WITH PASSWORD 'campusos_dev';
CREATE DATABASE campusos OWNER campusos;
GRANT ALL PRIVILEGES ON DATABASE campusos TO campusos;
\q
SQL

# 执行迁移
cd ~/bbs/bbs01/CampusOS
PGPASSWORD=campusos_dev psql -h localhost -U campusos -d campusos -f migrations/000001_init_schema.up.sql

# 验证
PGPASSWORD=campusos_dev psql -h localhost -U campusos -d campusos -c "\dt"
```

### 启动其他依赖（Redis + NATS）

```bash
# 使用 Docker 启动 Redis 和 NATS
docker compose up -d redis nats
```
