# CampusOS v0.4.6-dev 进度说明

> 日期：2026-07-01
> 状态：已完成
> 范围：个人主页内容同步基础

## 1. 本次目标

继续推进 v4 计划中的“个人主页/博客园插件”，在 v0.4.5 已有主页模型和公开入口基础上，完成 P0 的内容同步与内容筛选基础能力。

本次目标是让用户在 CampusOS 社区发布或更新帖子后，系统可以根据个人主页配置把公开文章摘要同步到个人主页内容表，并提供公开列表接口供后续前端主页使用。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| 同步内容模型 | 新增 `SpaceContent`，记录来源帖子、标题、摘要、版块、标签和同步时间 |
| 同步内容表 | 新增 `user_space_contents`，按 `thread_id` 做唯一同步记录 |
| 事件订阅 | Space Service 订阅 `thread.created`、`thread.updated`、`thread.deleted` |
| 内容同步 | 已发布帖子命中规则时 upsert 到 `user_space_contents` |
| 内容移除 | 删除、非公开状态、关闭同步或筛选不命中时移除同步内容 |
| 分类筛选 | `sync_categories` 非空时只同步匹配 `category_id` 的帖子 |
| 标签筛选 | `sync_tags` 非空时只同步至少命中一个标签的帖子 |
| 公开列表接口 | 新增 `GET /api/v1/space/:user_id/contents` 和 `GET /api/v1/u/:username/contents` |
| NATS 兼容 | 事件 payload 支持内存总线的结构体和 NATS JSON 反序列化后的 map |
| NATS 分发修正 | 同一事件类型只创建一个底层 NATS 订阅，避免多个业务 handler 时重复分发 |
| 测试覆盖 | 覆盖默认主页同步、分类/标签过滤和 map payload 事件解码 |
| 帮助文档 | 更新个人主页 Space 说明，补充内容同步接口和规则 |
