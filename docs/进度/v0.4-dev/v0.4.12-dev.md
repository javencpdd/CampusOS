## 2. 完成内容

### 2.1 总体判断

v0.4-dev 已完成一组高价值的阶段性能力，包括：

| 模块 | 当前状态 | 依据 |
| --- | --- | --- |
| v3 前置收尾 | 已完成 | `v0.4.0-dev` 记录 SetConfig 持久化、SQLite-backed 插件 KV、CLI pack/install、SDK Harness |
| 插件配置 schema | 已完成基础能力 | `v0.4.1-dev` 记录 manifest `config_schema`、CLI/SDK 示例同步 |
| AI Gateway 最小闭环 | 已完成最小闭环 | `internal/ai/`、`/api/v1/ai/status`、`/api/v1/ai/logs`、`ai_call_logs` 迁移 |
| 插件交付闭环 | 已完成最小闭环 | `campusosctl plugin pack/install`、标准插件包导入导出、Admin 插件导入导出入口 |
| 个人主页/博客园能力 | 已完成核心闭环 | `internal/space/`、`/space/:user_id`、`/u/:username`、内容同步、`web` 公开主页 |
| 主页风格导入导出 | 已完成 P0 闭环 | 风格包校验、预览、导出、应用、4 个示例风格包、`/space/settings` 前端入口 |

但 v4 原计划同时包含 MCP Server、Webhook、IM/机器人、管理后台运营视图、部署观测等任务。当前代码结构中没有 `internal/mcp`、`internal/webhook`、`internal/im` 或对应后台页面，因此不能判定 v0.4-dev 已全部完成。

### 2.3 对 v0.4-dev 的建议定位

当前更合理的表述是：

```text
v0.4-dev 已完成阶段性收口：插件交付、AI Gateway 最小闭环、个人主页和风格包 P0 闭环已可验证。
v0.4-dev 未完成 v4 原计划全量任务：MCP、Webhook、IM/机器人、完整管理后台运营和部分发布观测能力应拆入 v0.5-dev 或后续任务。
```

不建议把当前状态写为“v0.4-dev 全部任务完成”。如果要严格完成 v4 原计划，应继续补齐 MCP、Webhook、IM/机器人、运营后台和回滚分享能力。
