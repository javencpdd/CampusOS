# CampusOS v0.4.17-dev 进度说明

> 日期：2026-07-02
> 状态：已完成
> 范围：修复后端未读取 `.env` 导致注册数据写入内存、管理员登录失败的问题

## 1. 本次目标

解决 UI 测试中出现的两个相关问题：

| 问题 | 表现 |
| --- | --- |
| 管理后台登录失败 | `http://localhost:3001/login` 返回 `invalid email or password` |
| 注册用户在 pgAdmin 查不到 | `http://localhost:3000/register` 返回 `201 created`，但 Docker PostgreSQL `users` 表只有 admin |

## 2. 原因分析

当前 `.env` 配置 Docker PostgreSQL 使用宿主机端口 `5433`：

```env
POSTGRES_PORT=5433
DATABASE_DSN=postgres://campusos:campusos_dev@localhost:5433/campusos?sslmode=disable
```

旧版 Go 后端 `config.Load()` 只读取进程环境变量，不读取项目根目录 `.env`。如果启动后端前没有手动 `export DATABASE_DSN=...`，后端会使用代码默认值：

```text
postgres://campusos:campusos_dev@localhost:5432/campusos?sslmode=disable
```

宿主机 PostgreSQL 已卸载且 Docker PostgreSQL 映射在 `5433`，所以后端连接 `5432` 失败并回退到内存模式。此时注册接口会成功，但数据只保存在进程内存中，不会进入 Docker PostgreSQL；默认管理员种子数据也不在内存模式中，因此管理后台登录失败。

## 6. 使用说明

管理后台默认账号：

| 项目 | 值 |
| --- | --- |
| 登录地址 | `http://localhost:3001/login` |
| 邮箱 | `admin@campusos.local` |
| 密码 | `Admin@123456` |

pgAdmin 账号不同：

| 项目 | 值 |
| --- | --- |
| 登录地址 | `http://localhost:5050` |
| 邮箱 | `admin@campusos.dev` |
| 密码 | `pgadmin123` |

注册用户会保存到 Docker PostgreSQL 的：

| 表 | 内容 |
| --- | --- |
| `users` | 用户基础资料 |
| `accounts` | 邮箱登录凭据和 bcrypt 密码哈希 |

不建议关闭密码哈希。需要重置管理员密码时，参考 `docs/help/账号认证与管理员登录说明.md`。
