# CampusOS v4 前置收尾功能说明

> 日期：2026-06-30
> 范围：插件配置持久化、SQLite KV、插件打包安装、SDK 测试 Harness
> 说明：Windows 实机验证本轮跳过，本文不声明 Windows + Docker Desktop 已跑通。

## 1. 插件配置持久化

插件通过 Host API 调用 `SetConfig` 时，系统会同时更新运行期 manifest 和插件仓储。

调用示例：

```bash
curl -X POST http://127.0.0.1:18080/api/host/SetConfig \
  -H "Content-Type: application/json" \
  -H "X-CampusOS-Plugin: hello-wasm" \
  -d '{"key":"threshold","value":3}'
```

要求插件 manifest 声明权限：

```yaml
permissions:
  api:
    - resource: "config"
      actions: ["read", "write"]
```

PG 模式下配置写入 `plugins.config`。写入成功后会记录插件日志，metadata 中包含被更新的 key。

## 2. SQLite-backed 插件 KV

插件 Storage API 现在可使用 SQLite-backed KV store。默认数据目录：

```text
.campusos/plugin-data
```

可通过 `.env` 覆盖：

```env
PLUGIN_DATA_DIR=.campusos/plugin-data
```

默认文件结构：

```text
.campusos/plugin-data/
└── hello-wasm/
    └── plugin.db
```

Host API 示例：

```bash
curl -X POST http://127.0.0.1:18080/api/host/StorageSet \
  -H "Content-Type: application/json" \
  -H "X-CampusOS-Plugin: hello-wasm" \
  -d '{"plugin_name":"hello-wasm","key":"greeting","value":"world"}'
```

权限要求：

```yaml
permissions:
  api:
    - resource: "storage"
      actions: ["read", "write", "delete"]
```

安全边界：

| 项目 | 规则 |
| --- | --- |
| 插件名 | 只允许字母、数字、`-`、`_` |
| 命名空间 | 调用插件只能访问自身 `plugin_name` |
| 路径 | 拒绝路径分隔符、空插件名和路径穿越 |
| Git | `.campusos/` 已加入 `.gitignore` |

## 3. 插件打包

命令：

```bash
go run ./cmd/campusosctl plugin pack examples/plugins/hello-wasm
```

指定输出路径：

```bash
go run ./cmd/campusosctl plugin pack examples/plugins/hello-wasm \
  --out /tmp/hello-wasm.campusos-plugin.tar.gz
```

打包时会校验：

| 项目 | 规则 |
| --- | --- |
| manifest | 必须存在 `plugin.yaml` 并可解析 |
| runtime | 必须是 `wasm` 或 `grpc` |
| wasm module | `runtime: wasm` 时，`config.module` 指向的文件必须存在且留在插件目录内 |
| 排除路径 | `data/`、`.git/`、`node_modules/`、日志和临时文件不进入包 |

## 4. 插件安装

安装本地插件包：

```bash
go run ./cmd/campusosctl plugin install /tmp/hello-wasm.campusos-plugin.tar.gz \
  --dir examples/plugins
```

覆盖已存在插件目录：

```bash
go run ./cmd/campusosctl plugin install /tmp/hello-wasm.campusos-plugin.tar.gz \
  --dir examples/plugins \
  --replace
```

安装安全规则：

| 项目 | 规则 |
| --- | --- |
| 解包路径 | 拒绝绝对路径和 `..` 路径穿越 |
| 默认覆盖 | 默认不覆盖已有插件目录 |
| 禁止内容 | 包内出现 `data/`、`.git/`、`node_modules/` 会被拒绝 |
| 最终目录 | 由 manifest 中的 `name` 决定 |

## 5. SDK 本地测试 Harness

`sdk/go` 提供 `NewHarness`，用于插件逻辑单元测试，不需要启动完整 CampusOS 服务。

示例：

```go
harness := campusos.NewHarness("hello-wasm")
defer harness.Close()

harness.Config["entrypoint"] = "handle_event"
harness.SetPermission("user-1", "thread", "read", true)

client := harness.Client()
value, found, err := client.GetConfig(ctx, "entrypoint")
```

当前支持：

| 能力 | 状态 |
| --- | --- |
| `GetConfig` / `SetConfig` | 支持 |
| `StorageGet` / `StorageSet` / `StorageDelete` | 支持 |
| `PublishEvent` | 支持，记录到 `harness.Events` |
| `Log` | 支持，记录到 `harness.Logs` |
| `CheckPermission` | 支持 |
| `GetUser` / `GetThread` / `GetReply` | 支持通过 harness map 预置数据 |
