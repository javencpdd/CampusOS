# CampusOS v0.4.1-dev 进度说明

> 日期：2026-07-01
> 状态：已完成
> 范围：v0.4-dev 插件配置 schema 草案

## 1. 本次目标

在 v0.4 插件交付闭环中补齐插件配置声明能力，让插件可以在 `plugin.yaml` 中声明可编辑配置项的结构，为后续后台插件配置表单、个人主页插件和主页风格插件提供基础约定。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| Manifest 扩展 | 新增 `config_schema`，支持声明配置字段列表 |
| Schema 字段 | 支持 `key`、`label`、`type`、`description`、`required`、`default`、`options` |
| 字段类型 | 初始支持 `string`、`text`、`number`、`boolean`、`select`、`json` |
| Manifest 校验 | 拒绝空 `key`、重复 `key`、未知类型、缺少 `options` 的 `select` |
| CLI inspect | `campusosctl plugin inspect` 输出 `config_schema` |
| CLI init | `campusosctl plugin init` 为 wasm/gRPC 插件生成对应的最小 schema |
| CLI pack | `campusosctl plugin pack` 复用 manifest 校验，打包前拒绝非法 schema |
| SDK 类型 | `sdk/go` 补齐 `ConfigSchema`、`ConfigField`、`ConfigOption` 类型 |
| SDK Harness | Harness 改为内存 HTTP transport，避免测试环境依赖本地监听端口 |
| 示例插件 | `examples/plugins/hello-wasm` 增加 `config_schema` 示例 |
| 帮助文档 | 新增插件配置 schema 说明文档 |
