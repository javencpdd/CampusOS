# CampusOS v0.4.4-dev 进度说明

> 日期：2026-07-01
> 状态：已完成
> 范围：AI Gateway 调用日志持久化

## 1. 本次目标

完成 `v0.4.3` 后续 P0 中的“设计并落地 AI 调用日志数据库表”，让 AI Gateway 在 PostgreSQL 模式下把调用日志写入数据库，同时保留内存模式回退，保证服务不可连接数据库时仍可运行。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| 数据库表 | 新增 `ai_call_logs`，记录 provider、model、source、status、耗时、token usage 和错误信息 |
| 索引 | 新增 created/provider/status/source 相关索引，支持管理端按时间和状态查看 |
| 回滚脚本 | 新增 `000006_add_ai_call_logs.down.sql` |
| 日志接口抽象 | 新增 `CallLogStore`，统一内存和 PG 日志读写能力 |
| PG 日志仓储 | 新增 `PgCallLogger`，实现 `Log` 和 `List` |
| 内存回退 | `MemoryCallLogger` 实现同一接口，内存模式继续可用 |
| 服务接入 | PostgreSQL 连接成功后，AI Service 自动切换为 PG logger |
| 管理接口 | `/api/v1/ai/logs` 继续使用统一读取接口，数据库错误会返回 500 |
| 测试覆盖 | 更新 AI Gateway 测试，覆盖日志仓储替换机制 |
| 帮助文档 | 更新 AI Gateway 文档，补充 `ai_call_logs` 说明 |
