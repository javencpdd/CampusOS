# CampusOS v0.4.15-dev 进度说明

> 日期：2026-07-02
> 状态：已完成
> 范围：清理宿主机 5432 本地数据库，统一使用 Docker PostgreSQL

## 1. 本次目标

解决宿主机 `127.0.0.1:5432` 与 Docker PostgreSQL `127.0.0.1:5433` 数据不一致的问题。按当前决策，删除宿主机 `5432` 上的本地 `campusos` 数据库，后续只使用 Docker PostgreSQL。

## 2. 完成内容

| 项目 | 说明 |
| --- | --- |
| 清理前确认 | `127.0.0.1:5432` 为宿主机 PostgreSQL，`users=5`、`threads=4`、`user_space_contents=2` |
| Docker 主库确认 | `127.0.0.1:5433` 与 `docker exec campusos-postgres` 指向同一套 Docker PostgreSQL |
| 本地库备份 | 已将宿主机 `5432` 的 `campusos` 数据库备份到 `/tmp/campusos-db-backups/campusos-host5432-20260702-180921.dump` |
| 本地库清理 | 已执行 `DROP DATABASE IF EXISTS campusos` 删除宿主机 `5432` 上的本地 `campusos` 数据库 |

| 命令 | 结果 |
| --- | --- |
| `PGPASSWORD=campusos_dev pg_dump -h 127.0.0.1 -p 5432 -U campusos -d campusos -F c -f /tmp/campusos-db-backups/campusos-host5432-20260702-180921.dump` | 通过，备份大小 54K |
| `DROP DATABASE IF EXISTS campusos` | 通过，宿主机 `5432` 本地库已删除 |

| `docker compose ps postgres pgadmin` | 通过，`campusos-postgres` 健康，端口为 `5433->5432` |
