# CampusOS v0.4.0-dev 进度说明

> 日期：2026-06-30
> 状态：已完成
> 范围：v4 前置收尾清单，跳过 Windows 实机验证

## 1. 本次目标

按 v4 计划中的“前置收尾清单”处理 v3 遗留项，但根据当前安排跳过 Windows + Docker Desktop 实机验证，不把该项作为本轮前置门槛。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| `SetConfig` 持久化 | Host API `SetConfig` 更新运行期 manifest 后，会通过插件仓储持久化配置；PG 模式写入 `plugins.config` |
| 配置审计 | `SetConfig` 成功后写入插件日志，记录变更 key |
| 插件仓储同步 | Manager 设置仓储时会同步已安装插件；若已有持久化配置，则合并回运行期 manifest |
| SQLite-backed 插件 KV | 新增 SQLite KV store，默认路径 `.campusos/plugin-data/<plugin>/plugin.db` |
| 插件 KV 路径隔离 | 拒绝空插件名、路径分隔符、`..` 等不安全插件名 |
| `campusosctl plugin pack` | 支持按规范打包插件目录为 `.campusos-plugin.tar.gz`，并排除 `data/`、`.git/`、`node_modules/`、日志和临时文件 |
| `campusosctl plugin install` | 支持从本地插件包安全解包安装到插件目录，默认拒绝覆盖，支持 `--replace` |
| SDK 测试 harness | `sdk/go` 新增 `NewHarness`，可本地模拟 Host API 的配置、存储、事件、日志、用户/主题/回复查询和权限检查 |
| 运行期数据忽略 | `.gitignore` 新增 `.campusos/`，避免插件 SQLite 数据入库 |
| 配置项 | `.env.example` 新增 `PLUGIN_DATA_DIR=.campusos/plugin-data` |
| Windows 实机验证 | 本轮明确跳过，未声明通过 |
