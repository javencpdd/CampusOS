# CampusOS v0.4.9-dev 进度说明

> 日期：2026-07-01
> 状态：已完成
> 范围：标准插件包导入导出与 Admin 管理入口

## 1. 本次目标

继续推进 v4 计划中的“插件交付闭环”，在已有 `campusosctl plugin pack/install` 基础上，将标准插件包能力抽成后端和 CLI 可复用实现，并补齐 Admin 插件管理页的导入、导出入口。

本次目标是让插件不只停留在本地源码目录或 CLI 操作，而是形成统一的 `.campusos-plugin.tar.gz` 标准包，管理员可以在后台导入插件包、导出已安装插件包，并通过同一套校验规则避免路径穿越、缺失 manifest、缺失 Wasm 模块等问题。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| 标准插件包实现 | 新增共享插件包逻辑，统一 `pack/install/import/export/inspect` 规则 |
| 包格式 | 固定 `.campusos-plugin.tar.gz`，包根目录必须直接包含 `plugin.yaml` |
| 安全校验 | 拒绝绝对路径、`../` 路径逃逸、非法插件名、非法归档类型和缺失 Wasm 模块 |
| 打包排除 | 自动排除 `.git/`、`data/`、`node_modules/`、日志、临时文件和 `.DS_Store` |
| CLI 复用 | `campusosctl plugin pack/install` 改为调用共享插件包实现 |
| Manager 导入导出 | 新增 `ImportPackage` 和 `ExportPackage`，支持覆盖同名插件 |
| 后端接口 | 新增 `POST /api/v1/plugin-packages/import` 和 `GET /api/v1/plugins/:name/export` |
| Admin 入口 | 插件管理页新增导入、覆盖同名插件、导出、刷新按钮 |
| 状态显示 | Admin 插件状态兼容 `installed/running/stopped/error` |
| 测试覆盖 | 新增插件包打包安装、覆盖导入、导出和恶意归档路径测试 |
| 帮助文档 | 新增标准插件包导入导出要求，并同步插件汇总文档 |
