# CampusOS v0.4.3-dev 进度说明

> 日期：2026-07-01
> 状态：已完成
> 范围：AI Gateway 服务启动接入与管理接口

## 1. 本次目标

补齐 `v0.4.2` 后续 P0 任务中的“将 Gateway 初始化接入服务启动流程”，让 AI Gateway 不只停留在内核包层面，而是在 API 服务启动时可按配置初始化，并提供管理员可查看的状态和内存日志接口。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| AI Service 容器 | 新增 `Service`，从 `config.AIConfig` 构建 Gateway、Provider、Limiter 和内存 Logger |
| 启动流程接入 | `internal/server` 启动时初始化 AI Gateway，PostgreSQL 和内存模式共用同一服务实例 |
| 状态查询 | 新增 `GET /api/v1/ai/status`，返回 enabled、ready、provider、脱敏配置和日志数量 |
| 日志查询 | 新增 `GET /api/v1/ai/logs?limit=100`，返回最近内存调用日志 |
| 权限控制 | AI 管理接口接入 JWT 和 `role:manage` 权限 |
| 密钥安全 | 状态接口只暴露 `api_key_configured`，不返回 `AI_API_KEY` 原文 |
| 测试覆盖 | 新增 Service 和 Handler 单元测试，覆盖启用、禁用、配置错误、脱敏状态和日志查询 |
| 帮助文档 | 更新 AI Gateway 文档，补充管理接口说明 |
