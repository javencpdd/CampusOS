# CampusOS v0.4.16-dev 进度说明

> 日期：2026-07-02
> 状态：已完成
> 范围：修复宿主机无 `psql` 时 migration 无法作用到 Docker PostgreSQL 的问题

## 1. 本次目标

解决卸载宿主机 PostgreSQL 后执行 `make migrate-up` 报错的问题：

```text
./scripts/migrate.sh: 行 21: psql: 未找到命令
```

当前项目数据库服务主要运行在 Docker 中，迁移脚本不应该强依赖宿主机安装 PostgreSQL 客户端。

## 2. 原因分析

| 项目 | 说明 |
| --- | --- |
| 根因 | 旧版 `scripts/migrate.sh` 直接调用宿主机 `psql` |
| 当前环境 | 宿主机 PostgreSQL 已卸载，`psql` 命令不存在 |
| Docker 状态 | `campusos-postgres` 正常运行，端口映射为 `5433->5432` |
| 实际问题 | 迁移 SQL 没有执行到数据库前就因找不到客户端失败 |

结论：`migrations/` 中的 SQL 文件本身有效，失败点在迁移执行器选择上。

## 3. 完成内容

| 项目 | 说明 |
| --- | --- |
| Bash 迁移脚本 | `scripts/migrate.sh` 增加 `PSQL_MODE=auto/host/docker` |
| Docker fallback | 宿主机没有 `psql` 时自动使用 `docker exec -i campusos-postgres psql` |
| SQL 文件处理 | Docker 模式下把 `-f migrations/xxx.sql` 转为标准输入，避免容器内找不到宿主机文件 |

| PowerShell 脚本 | `scripts/migrate.ps1` 同步增加相同的模式选择和 Docker fallback 逻辑 |

## 6. 使用说明

默认执行：

```bash
make migrate-up
```

脚本会自动选择：

| 条件 | 执行方式 |
| --- | --- |
| 宿主机有 `psql` | 连接 `DB_HOST:DB_PORT` |
| 宿主机没有 `psql`，`campusos-postgres` 正在运行 | 使用容器内 `psql` |

强制使用 Docker：

```bash
PSQL_MODE=docker make migrate-up
```

强制使用宿主机 `psql`：

```bash
PSQL_MODE=host make migrate-up
```

如果容器名变化：

```bash
POSTGRES_CONTAINER=新的容器名 PSQL_MODE=docker make migrate-up
```
