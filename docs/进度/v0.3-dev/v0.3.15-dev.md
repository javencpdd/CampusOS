## 3. 设计说明

当前项目的本地服务依赖主要由 Docker 提供，数据库迁移验证已经在 Ubuntu backend job 中覆盖。Windows CI 本轮先聚焦 Go 单元测试，原因如下：

| 难点 | 当前处理 |
| --- | --- |
| PostgreSQL service 在 Windows runner 上编排成本较高 | 迁移验证继续放在 Ubuntu job |
| Docker Desktop 与 GitHub Windows runner 的行为差异较大 | 不把 Docker 运行验证作为本轮 Windows CI 门槛 |
| 项目仍以 Ubuntu 作为参考开发环境 | Windows 先作为代码级兼容检查 |
| PowerShell 脚本实机验证依赖本地 Docker Desktop | 保留为后续人工或专门 runner 验证 |
