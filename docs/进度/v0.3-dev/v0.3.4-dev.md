# CampusOS v0.3.4-dev 进度说明

> 日期：2026-06-28
> 状态：已完成

## 1. 本次目标

完成 v0.3-dev 首轮 P0 中的错误隔离能力，确保 Wasm 插件在事件处理时发生 trap、panic 或普通调用错误时，不会影响主服务进程，并且异常插件会退出运行态。

本次聚焦 Runtime 与 Manager 的最小闭环：Runtime 隔离异常 module，Manager 将事件调用失败的插件状态标记为 `error`。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| Wasm 调用错误包装 | 新增 `ErrEventCallFailed`，普通 trap/error 返回可识别错误 |
| panic 保护 | `SendEvent` 使用 `recover` 捕获运行时 panic，避免向上击穿主流程 |
| module 隔离 | 事件调用失败或超时后关闭并移除对应 Wasm module |
| 锁粒度优化 | `SendEvent` 只在读取 module 状态时持锁，调用 Wasm 时不长期占用 runtime 锁 |
| Manager 状态更新 | 事件调用失败时将插件标记为 `StatusError` 并写入 `ErrorMsg` |
| 单元测试 | 覆盖 Wasm trap 隔离和 Manager 事件失败状态更新 |
