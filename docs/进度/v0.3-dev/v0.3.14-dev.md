## 2. 插件目录规范

插件源码目录建议使用以下结构：

```text
<plugin-name>/
├── plugin.yaml
├── README.md
├── plugin.wasm              # runtime: wasm 时推荐
├── bin/
│   └── plugin               # runtime: grpc 时推荐
├── assets/
│   └── ...
├── src/
│   └── ...
└── data/                    # 本地运行数据，不进入发布包
```

## 3. 发布包规范

发布包格式：

```text
<plugin-name>-<version>.campusos-plugin.tar.gz
```

归档包根目录必须直接包含：

| 文件/目录 | 要求 | 说明 |
| --- | --- | --- |
| `plugin.yaml` | 必须 | 插件 manifest |
| `README.md` | 推荐 | 插件说明 |
| `plugin.wasm` | `runtime: wasm` 必须或由 `config.module` 指定 | Wasm 主模块 |
| `bin/plugin` | `runtime: grpc` 推荐 | gRPC 插件可执行文件 |
| `assets/` | 可选 | 静态资源 |

发布包不得包含：

| 路径 | 原因 |
| --- | --- |
| `data/` | 运行期数据应由目标环境生成或恢复 |
| `.git/` | 避免泄露源码历史 |
| `node_modules/` | 避免包体积失控 |
| 临时文件、日志文件 | 避免污染安装环境 |
| 绝对路径或 `..` 路径 | 防止解包路径穿越 |

## 4. Manifest 最低要求

`plugin.yaml` 最低字段：

```yaml
name: hello-wasm
version: "0.1.0"
runtime: wasm

events:
  subscribe:
    - "thread.created"

permissions:
  api:
    - resource: "config"
      actions: ["read"]

storage:
  type: none

config:
  module: "plugin.wasm"
  entrypoint: "handle_event"
```

## 5. 校验规则

| 校验项 | 规则 |
| --- | --- |
| Manifest | 必须可由 `plugin.LoadManifest` 解析并通过 `Validate` |
| Runtime | 只允许 `wasm` 或 `grpc` |
| Wasm 模块 | `runtime: wasm` 时，`config.module` 指向的文件必须存在且留在插件目录内 |
| gRPC 可执行文件 | `runtime: grpc` 时，`config.command` 或默认 `bin/plugin` 应存在 |
| 权限声明 | Host API、storage、event 权限必须在 manifest 中显式声明 |
| 存储目录 | `data/` 不打包，安装时由运行环境创建 |
