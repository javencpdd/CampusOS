# CampusOS v0.3.5-dev 进度说明

> 日期：2026-06-28
> 状态：已完成

## 1. 本次目标

完成 v0.3-dev 首轮 P0 中的 Host API 权限检查，确保插件调用宿主能力前必须通过 `manifest.permissions.api` 授权校验。

本次聚焦 API 级和资源级权限，不做字段级权限、安装时权限审查 UI、插件日志落库。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| 方法权限映射 | 新增 Host API 方法到 `resource/action` 的固定映射 |
| 权限错误 | 新增 `ErrHostAPIPermissionDenied`，权限失败可被调用方识别 |
| manifest 校验 | `HandleHostAPIRequestForPlugin` 调用前检查插件 manifest 权限 |
| 旧入口收紧 | `HandleHostAPIRequest` 不再绕过权限，受保护方法缺少 manifest 会拒绝 |
| HTTP 身份解析 | Host API HTTP 入口支持通过 `X-CampusOS-Plugin` 或 `plugin_name` 解析插件身份 |
| 存储隔离 | `StorageGet/Set/Delete` 限制只能访问当前插件自己的 storage namespace |
| 单元测试 | 覆盖权限拒绝、声明权限允许、通配权限、存储隔离和 HTTP 入口身份校验 |
