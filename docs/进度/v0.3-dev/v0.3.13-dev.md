# CampusOS v0.3.13-dev 进度说明

> 日期：2026-06-30
> 状态：已完成

## 1. 本次目标

完成 v0.3-dev 剩余任务中的 CLI 雏形，新增 `campusosctl` 命令，先支持插件模板初始化和插件 manifest 检查。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| CLI 入口 | 新增 `cmd/campusosctl` |
| `plugin init` | 支持生成最小插件脚手架，包含 `plugin.yaml` 和 `README.md` |
| Runtime 选择 | `plugin init` 支持 `--runtime wasm|grpc`，默认 `wasm` |
| 目录选择 | `plugin init` 支持 `--dir` 指定输出目录 |
| 参数兼容 | 支持 `plugin init <name> --dir path` 和 `plugin init --dir path <name>` 两种常见写法 |
| `plugin inspect` | 支持读取插件目录中的 `plugin.yaml` 并输出 JSON 摘要 |
| 校验复用 | `plugin inspect` 复用现有 `internal/plugin.LoadManifest` 校验逻辑 |
| 单元测试 | 覆盖脚手架生成、manifest 摘要输出和非法插件名拒绝 |

## 3. 使用示例

```bash
go run ./cmd/campusosctl plugin init hello-demo --runtime wasm --dir /tmp/hello-demo
go run ./cmd/campusosctl plugin inspect examples/plugins/hello-wasm
```
