# CampusOS v0.3.2-dev 进度说明

> 日期：2026-06-28
> 状态：已完成

## 1. 本次目标

完成 v0.3-dev 首轮 P0 中的 Wasm 事件调用最小实现，让 `runtime: wasm` 插件不再只停留在生命周期骨架，而是可以被真实 Wasm 引擎实例化并响应 `SendEvent`。

本次采用临时最小 ABI：默认调用 Wasm 模块导出的无参数函数 `handle_event`，返回值非 0 表示允许事件，返回 0 表示拒绝事件。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| Wasm 引擎 | 引入 `github.com/tetratelabs/wazero` |
| 模块实例化 | `Start` 读取 `plugin.wasm` 或 `manifest.config.module` 指定文件，并通过 wazero 实例化 |
| 事件入口 | 默认查找 `handle_event`，支持 `manifest.config.entrypoint` 指定入口函数 |
| 事件调用 | `SendEvent` 调用入口函数，按首个返回值转换为 `PluginResponse.Allowed` |
| 停止释放 | `Stop` 关闭 wazero module 并移除运行状态 |
| 健康检查 | `IsRunning` 会排除已关闭 module |
| 单元测试 | 覆盖允许事件、拒绝事件、配置模块路径、入口缺失、模块缺失 |
