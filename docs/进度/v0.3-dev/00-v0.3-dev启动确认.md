# CampusOS v0.3-dev 启动确认

> 日期：2026-06-28
> 阶段状态：可以进入 v0.3-dev 开发
> 前置依据：`docs/项目计划v3/02-v0.3-dev计划书.md` 第 10 节、`docs/进度/v0.3-dev-pre/前置稳定化完成说明.md`

## 1. 结论

v0.3-dev 前置稳定化清单已经完成基础闭环，可以开始 v0.3-dev 阶段开发。

当前可进入的主线是：

1. Runtime 抽象收敛。
2. Wasm Runtime 最小闭环。
3. Host API 权限校验。
4. 插件 Manifest 权限模型。
5. SDK、CLI、示例插件和后台管理增强。

## 3. 当前开发基线

| 类别 | 当前基线 |
| --- | --- |
| 后端 | Go + Gin + pgx，模块边界为 identity/community/plugin/server/pkg |
| 前端 | `web` 用户前台、`admin` 管理后台，均使用 Vue/Vite/pnpm |
| 数据库 | PostgreSQL，迁移版本为 `000001` 到 `000005` |
| 基础服务 | PostgreSQL、Redis、NATS 由 Docker Compose 或本机服务提供 |
| 插件运行时 | gRPC Runtime 已存在，Wasm Runtime 待实现 |
| 插件 Manifest | 已支持 `runtime: grpc/wasm`、事件订阅、API 权限声明、存储配置 |
| Host API | 已有身份、数据、事件基础能力，权限校验和 Wasm 绑定待增强 |

## 5. 允许开始的开发范围

v0.3-dev 第一轮建议只进入以下范围：

| 范围 | 是否进入第一轮 |
| --- | --- |
| Runtime 接口梳理 | 是 |
| Wasm Runtime 最小实现 | 是 |
| hello-wasm 示例插件 | 是 |
| Host API 权限检查器 | 是 |
| 插件调用日志落库 | 是 |
| 完整插件市场 | 否 |
| 原生 Windows 生产部署 | 否 |
| 大规模前端重构 | 否 |

## 6. 非阻塞风险

| 风险 | 处理方式 |
| --- | --- |
| PowerShell 脚本尚未在 Windows 实机验证 | v0.3-dev 中期加入 Windows CI 或手工验证 |
| OpenAPI 仍是手写基线 | 后续考虑从路由或注解生成 |
| 插件存储暂未落地 SQLite | 放入 v0.3-dev 插件存储任务 |
| Host API v1/v2 命名和职责存在重叠 | Runtime/Host API 第一轮开发时同步收敛 |

## 7. 下一步

从 `docs/进度/v0.3-dev/01-v0.3-dev开发指南.md` 开始执行。第一轮开发不建议直接扩展后台 UI，应先把 Wasm Runtime、Host API 权限和日志链路跑通。
