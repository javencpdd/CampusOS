# CampusOS v0.3.3-dev 进度说明

> 日期：2026-06-28
> 状态：已完成

## 1. 本次目标

完成 v0.3-dev 首轮 P0 中的 Runtime 调用超时能力，确保 Wasm 插件在事件处理函数卡住时不会无限阻塞主服务调用线程。

本次聚焦 `internal/plugin/wasm` 的 `SendEvent` 调用 deadline，不引入 Host API、事件 payload ABI 或日志落库。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| wazero 超时终止 | `NewRuntime` 启用 `WithCloseOnContextDone(true)` |
| 默认调用超时 | Wasm 事件调用默认使用 `2s` deadline |
| manifest 配置 | 支持 `manifest.config.event_timeout_ms` 覆盖单个插件事件调用超时 |
| 超时错误 | 新增 `ErrEventCallTimeout`，调用超时时返回可识别错误 |
| 卡死隔离 | 超时后 wazero 会关闭对应 module，`IsRunning` 会返回 `false` |
| 单元测试 | 新增死循环 Wasm module 测试，确认超时能快速返回 |
