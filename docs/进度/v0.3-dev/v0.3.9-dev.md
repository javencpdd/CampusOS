# CampusOS v0.3.9-dev 进度说明

> 日期：2026-06-30
> 状态：已完成

## 1. 本次目标

完成 v0.3-dev 剩余任务中的 Host API 接入与插件配置读取，让主服务启动时可以同时启动本机 Host API，并让插件在声明 `config/read` 权限后通过 `GetConfig` 读取自身 manifest 配置。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| Host API 配置 | 新增 `HOST_API_ENABLED` 和 `HOST_API_ADDR`，默认启用并绑定 `127.0.0.1:18080` |
| 主服务接入 | PostgreSQL 模式和内存模式都会初始化 `HostAPIv2` 并启动 Host API server |
| 插件身份 | Host API server 继续通过 `X-CampusOS-Plugin` 或 body 中的 `plugin_name` 解析调用插件 |
| 权限校验 | `GetConfig` 复用 `config/read` 权限，未声明权限时返回 `ErrHostAPIPermissionDenied` |
| 配置读取 | 新增 `GetConfig`，支持读取整个 `manifest.config` 或按 `key` 读取单项 |
| 日志接入 | Host API 权限拒绝日志接入当前插件日志仓储 |
| 启动失败处理 | Host API 监听失败会在主服务启动阶段返回错误，不再只异步打印 |

## 3. Host API 当前契约

| 方法 | 请求 | 响应 | 权限 | 状态 |
| --- | --- | --- | --- | --- |
| `GetUser` | `{ "user_id": "..." }` | 用户基础字段 map | `user/read` | 已实现 |
| `QueryThreads` | `{ "category_id": "...", "author_id": "...", "keyword": "...", "page": 1, "page_size": 20 }` | `{ "threads": [...], "total": n }` | `thread/read` | 已实现 |
| `PublishEvent` | `{ "event_type": "...", "source": "...", "subject": "...", "data": {...} }` | `{ "success": true }` | `event/publish` | 已实现 |
| `SendNotification` | `{ "user_id": "...", "title": "...", "content": "...", "action_url": "..." }` | `{ "success": true }` | `notification/send` | 已实现为日志记录 |
| `GetConfig` | `{ "key": "..." }` 或 `{}` | `{ "value": ..., "found": true/false }` 或 `{ "config": {...}, "found": true }` | `config/read` | 本次完成 |
| `StorageGet` | `{ "plugin_name": "...", "key": "..." }` | `{ "value": "...", "found": true/false }` | `storage/read` | 已实现，限制只能访问自身命名空间 |
| `StorageSet` | `{ "plugin_name": "...", "key": "...", "value": "..." }` | `{ "success": true }` | `storage/write` | 已实现，限制只能访问自身命名空间 |
| `StorageDelete` | `{ "plugin_name": "...", "key": "..." }` | `{ "success": true }` | `storage/delete` | 已实现，限制只能访问自身命名空间 |
| `GetThread` / `GetReply` / `SetConfig` / `CheckPermission` / `Log` | 待固定 | 待固定 | 权限映射已预留 | 未实现 |

Host API 调用路径：

```text
POST /api/host/{Method}
X-CampusOS-Plugin: <plugin-name>
Content-Type: application/json
```
