# CampusOS v0.3.6-dev 进度说明

> 日期：2026-06-28
> 状态：已完成

## 1. 本次目标

完成 v0.3-dev 首轮 P0 中的 `plugin_logs` 写入路径，让插件启动、停止、事件处理、事件错误和 Host API 权限拒绝具备可落库的日志记录能力。

本次不新增 migration，因为 `000005_plugin_schema_alignment` 已创建 `plugin_logs` 表。

## 2. 完成内容

| 项目 | 结果 |
| --- | --- |
| 日志模型 | 新增 `PluginLogRecord` |
| 日志仓储接口 | 新增 `PluginLogRepository`，提供 `SaveLog` 和 `ListLogs` |
| 内存仓储 | `MemoryPluginRepository` 支持插件日志写入与查询 |
| PostgreSQL 仓储 | `PgPluginRepository` 写入和查询 `plugin_logs` |
| Manager 日志 | 启动、停止、before/after 事件成功、拒绝、失败都会写入日志 |
| Host API 日志 | 权限拒绝时写入 warn 级别插件日志 |
| 失败隔离 | 日志写入失败只打印服务日志，不影响插件主流程 |
| 单元测试 | 覆盖 Manager 日志写入和 Host API 权限拒绝日志写入 |
