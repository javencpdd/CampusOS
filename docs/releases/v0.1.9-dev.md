## 版本概述

v0.1.9-dev 是 v0.1 系列的最后一个版本，完成了**RBAC 权限系统、补全缺失 API、修复 PostgreSQL 模式下的关键 Bug**，使系统从"Demo 级"升级为"可用级"。v0.1 阶段正式收尾，下一阶段进入 v0.2 插件系统。

## 核心功能

### 1. RBAC 角色系统

| 组件 | 说明 |
|:---|:---|
| `roles` 表 | admin / moderator / member / guest 四个内置角色 |
| `user_roles` 表 | 用户-角色关联，支持按版块分配版主 |
| `permissions` 表 | 角色-权限关联（resource + action），30 条初始权限 |
| `PermissionService` | 权限检查服务（Check 方法） |
| `RequirePermission` 中间件 | 路由级权限保护 |

### 2. 补全 API 端点

| 方法 | 端点 | 说明 | 认证 |
|:---|:---|:---|:---|
| `POST` | `/api/v1/users/:id/suspend` | 封禁用户 | ✅ admin/moderator |
| `POST` | `/api/v1/users/:id/activate` | 解封用户 | ✅ admin/moderator |
| `POST` | `/api/v1/threads/:id/pin` | 置顶帖子 | ✅ admin/moderator |
| `POST` | `/api/v1/threads/:id/unpin` | 取消置顶 | ✅ admin/moderator |
| `POST` | `/api/v1/threads/:id/lock` | 锁定帖子 | ✅ admin/moderator |
| `POST` | `/api/v1/threads/:id/unlock` | 解锁帖子 | ✅ admin/moderator |
| `PUT` | `/api/v1/threads/:thread_id/posts/:post_id` | 编辑回复 | ✅ 作者 |
| `DELETE` | `/api/v1/threads/:thread_id/posts/:post_id` | 删除回复 | ✅ 作者 |
| `DELETE` | `/api/v1/categories/:id` | 删除版块 | ✅ admin |

补全后 API 端点总数：**30 个**

### 3. 修复的 Bug

| Bug | 修复方案 |
|:---|:---|
| PG 模式下 Category/Post Handler 传 nil 导致 panic | 创建 PgCategoryRepository 和 PgPostRepository，正确注入 |
| 事件历史 API 仅内存模式可用 | NATS 模式下也创建 MemoryEventBus 用于事件历史 |
| domain.Thread 依赖 lib/pq（未安装） | Tags 字段改为 []string，移除 pq 依赖 |
