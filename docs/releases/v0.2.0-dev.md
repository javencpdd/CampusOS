## 版本概述

v0.2.0 是 v0.2 阶段的首个版本，目标是**建立 gRPC 插件运行时基础架构**，实现插件可以注册、启动、接收事件并通过 Host API 与核心系统交互。这是 CampusOS 从"单体论坛"向"可扩展社区引擎"转变的关键里程碑。

---

## 核心目标

1. **Plugin Manager** — 统一插件管理器（注册、Manifest 解析、生命周期管理）
2. **gRPC Runtime** — 插件进程启动/停止/健康检查/自动重启
3. **Host API** — 插件调用核心能力的统一接口（Identity/Data/Event/Notification/Storage）
4. **示例 gRPC 插件** — 至少 1 个可工作的示例插件

## 架构设计

```
CampusOS Go Server
    │
    ├── Plugin Manager
    │   ├── Manifest Parser (plugin.yaml)
    │   ├── Plugin Registry (插件注册表)
    │   └── Lifecycle Manager (installed → enabled → running → stopped)
    │
    ├── gRPC Runtime
    │   ├── Process Manager (启动/停止/重启插件进程)
    │   ├── gRPC Client (进程间通信)
    │   └── Health Checker (每 10s)
    │
    ├── Host API (gRPC Server，插件反向调用)
    │   ├── host.Identity.GetUser()
    │   ├── host.Data.Query("thread", filter)
    │   ├── host.Event.Publish(event)
    │   ├── host.Notification.Send(userID, template, data)  [P1]
    │   └── host.Storage.Get/Set/Delete()                   [P1]
    │
    └── Event Bus
        ├── 内部事件分发（Service → EventBus → 订阅器）
        └── 插件事件分发（EventBus → Plugin Manager → 插件）
```

## 示例 Manifest (plugin.yaml)

```yaml
name: hello-plugin
display_name: "Hello Plugin"
version: "0.1.0"
description: "一个简单的事件日志插件"
author: "CampusOS Team"
runtime: grpc

events:
  subscribe:
    - "user.created"
    - "thread.created"
    - "post.created"

permissions:
  api:
    - resource: "thread"
      actions: ["read"]
    - resource: "user"
      actions: ["read"]

storage:
  type: sqlite
  sqlite:
    filename: "data.db"
```
