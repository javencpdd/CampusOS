### 核心功能

#### 1. Protobuf 协议定义
- **Host API**：7 个 RPC 方法（GetUser/QueryThreads/PublishEvent/SendNotification/StorageGet/Set/Delete）
- **Plugin API**：3 个 RPC 方法（Ping/HandleEvent/GetInfo）
- 当前使用 HTTP JSON-RPC 风格实现，未来可无缝切换到 gRPC

#### 2. Host API 补全
- **Notification**：`hostAPI.Notification().Send(userID, title, content, actionURL)`
- **Storage**：`hostAPI.Storage().Get/Set/Delete(pluginName, key, value)`
- **Host API Server**：HTTP 服务端，监听端口，插件通过 `/api/host/{method}` 调用

#### 3. .before/.after 事件模式
- **DispatchBeforeEvent**：同步分发，插件可返回 `allowed=false` 阻止操作
- **DispatchEvent（原 .after）**：异步分发，不阻塞主流程
- Manager 自动根据事件类型查找订阅的插件并分发

#### 4. 真实通信
- gRPC Runtime 的 SendEvent 改为真实 HTTP POST 调用插件的 `/event` 端点
- 插件不可达时降级为日志记录，不影响主流程
