## 项目概述

CampusOS 是一个面向高校场景的开放式社区平台，定位为"社区操作系统"而非传统论坛。

本次 Demo 是项目的第一个可运行版本，实现了最基础的**用户系统**和**帖子系统**，采用内存存储，不依赖任何外部数据库，用于验证整体架构设计的可行性。

## 运行指南

### 编译

```bash
cd ~/bbs/bbs01/CampusOS

# 编译
go build -o campusos-server ./cmd/server/main.go

# 或使用 go run 直接运行
go run ./cmd/server/main.go
```

### 启动服务器

```bash
# 使用默认配置启动（监听 0.0.0.0:8080）
./campusos-server

# 自定义端口
SERVER_PORT=3000 ./campusos-server
```

启动后会看到：
```
🚀 CampusOS v0.1.0-dev starting on 0.0.0.0:8080
📖 API docs: http://0.0.0.0:8080/api/v1/health
[GIN-debug] Listening and serving HTTP on 0.0.0.0:8080
```

### 停止服务器

在服务器运行的终端按 `Ctrl+C`。

## API 测试示例

### 1. 健康检查

```bash
curl -s http://localhost:8080/api/v1/health | jq .
```

预期返回：
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "service": "CampusOS",
    "status": "ok",
    "version": "0.1.0-dev"
  }
}
```

### 2. 注册用户

```bash
curl -s -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "jack",
    "nickname": "Jack",
    "email": "jack@example.com",
    "password": "123456"
  }' | jq .
```

预期返回：
```json
{
  "code": 0,
  "msg": "created",
  "data": {
    "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "username": "jack",
    "nickname": "Jack",
    "email": "jack@example.com",
    "status": "active",
    "created_at": "2024-06-24T12:00:00Z",
    "updated_at": "2024-06-24T12:00:00Z"
  }
}
```

### 3. 用户登录

```bash
curl -s -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "jack@example.com", "password": "123456"}' | jq .
```

### 4. 创建帖子

```bash
# 将 USER_ID 替换为注册返回的用户 ID
USER_ID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"

curl -s -X POST http://localhost:8080/api/v1/threads \
  -H "Content-Type: application/json" \
  -H "X-User-ID: $USER_ID" \
  -H "X-User-Name: Jack" \
  -d '{
    "title": "我的第一篇帖子",
    "content": "Hello CampusOS! 这是一个可运行的 Demo。",
    "category_id": "cat_001",
    "tags": ["demo", "hello"]
  }' | jq .
```

### 5. 获取帖子列表

```bash
# 获取所有帖子
curl -s http://localhost:8080/api/v1/threads | jq .

# 分页查询
curl -s "http://localhost:8080/api/v1/threads?page=1&page_size=5" | jq .

# 关键词搜索
curl -s "http://localhost:8080/api/v1/threads?keyword=CampusOS" | jq .

# 按分类过滤
curl -s "http://localhost:8080/api/v1/threads?category_id=cat_001" | jq .
```

### 6. 更新帖子

```bash
THREAD_ID="xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"

curl -s -X PUT "http://localhost:8080/api/v1/threads/$THREAD_ID" \
  -H "Content-Type: application/json" \
  -H "X-User-ID: $USER_ID" \
  -d '{
    "title": "更新后的标题",
    "content": "更新后的内容"
  }' | jq .
```

### 7. 删除帖子

```bash
curl -s -X DELETE "http://localhost:8080/api/v1/threads/$THREAD_ID" \
  -H "X-User-ID: $USER_ID"
```

## 当前限制与已知问题

### 数据存储

| 问题 | 影响 | 说明 |
|:---|:---|:---|
| 使用内存存储 | 重启丢失数据 | 尚未接入 PostgreSQL |
| 无数据持久化 | 不适合生产环境 | 仅用于 Demo 验证 |

### 认证与安全

| 问题 | 影响 | 说明 |
|:---|:---|:---|
| 无 JWT Token 验证 | 认证仅通过 Header 模拟 | 需实现完整的 JWT 签发/验证 |
| 密码未使用 bcrypt | 安全性不足 | 当前使用 SHA-256，生产需 bcrypt |
| 无 Rate Limiting | 可能被暴力攻击 | 需实现限流中间件 |
| 无 XSS/CSRF 防护 | Web 安全风险 | 需添加安全中间件 |

### 功能缺失

| 缺失功能 | 优先级 | 说明 |
|:---|:---|:---|
| 无版块管理 | P0 | Category CRUD 尚未实现 |
| 无回复系统 | P0 | Post（楼中楼回复）尚未实现 |
| 无权限系统 | P0 | RBAC 权限模型尚未实现 |
| 无事件总线 | P0 | NATS Event Bus 尚未集成 |
| 无通知系统 | P1 | 站内信/邮件通知尚未实现 |
| 无文件上传 | P1 | MinIO/S3 存储尚未集成 |
| 无前端 UI | P1 | Vue 3 前端尚未开发 |
| 无插件系统 | P1 | Plugin Manager / Runtime 尚未实现 |
| 无 AI 能力 | P2 | LLM 网关 / MCP Server 尚未实现 |
| 无 IM 适配器 | P2 | NapCat / AstrBot 尚未接入 |

## 下一步开发计划

### 第一步：PostgreSQL 数据持久化（优先级最高）

**目标**：替换内存存储，实现真正的数据持久化

**工作内容**：

1. 安装配置 PostgreSQL（可通过 Docker）
2. 创建数据库表结构（users、threads、posts、categories 等）
3. 实现 `golang-migrate` 数据库迁移
4. 编写 PostgreSQL 版的 UserRepository 和 ThreadRepository
5. 通过依赖注入替换内存实现
6. 编写集成测试验证数据持久化

**预计工期**：3-5 天

**验收标准**：
- 服务器重启后数据不丢失
- 所有 API 端点正常工作
- 数据库迁移脚本可正常执行

### 第二步：JWT 认证体系

**目标**：替换模拟 Token，实现真正的身份认证

**工作内容**：

1. 实现 JWT Token 签发（Access Token + Refresh Token）
2. 实现认证中间件（从 Token 中提取用户信息）
3. 实现 Token 刷新机制
4. 实现密码 bcrypt 哈希（替换 SHA-256）
5. 更新所有需要认证的 API 端点

**预计工期**：2-3 天

### 第三步：版块管理 + 回复系统

**目标**：完善社区核心功能

**工作内容**：

1. 实现 Category（版块）领域模型和 CRUD API
2. 实现 Post（回复）领域模型和 CRUD API
3. 实现帖子回复计数
4. 实现楼中楼回复（parent_id）

**预计工期**：3-4 天

### 第四步：Event Bus 初步集成

**目标**：引入事件驱动架构的基础

**工作内容**：

1. Docker 启动 NATS（JetStream）
2. 实现 EventBus 接口和 NATS 实现
3. 实现 CloudEvents 事件封装
4. 在关键操作（用户注册、帖子创建）中发布事件
5. 实现简单的事件消费者示例

**预计工期**：3-4 天

### 时间线总览

```
Week 1:  PostgreSQL 数据持久化 + JWT 认证
Week 2:  版块管理 + 回复系统 + Event Bus 初步集成
Week 3:  角色权限 + Docker Compose + 集成测试
Week 4:  文档完善 + Bug 修复 + v0.1 发布
```
