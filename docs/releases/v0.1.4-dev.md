## 核心功能变更

### 1. PostgreSQL 数据持久化

系统启动时自动连接 PostgreSQL，成功则使用 PostgreSQL 存储，失败自动回退到内存模式：

```go
pool, err := database.New(cfg.Database.DSN)
if err != nil {
    log.Printf("⚠️  PostgreSQL 连接失败，回退到内存模式: %v", err)
    return s.runMemoryMode()
}
```

**PostgreSQL 模式特性：**
- 用户注册：bcrypt 密码哈希 → 存入 `users` 表 + `accounts` 表
- 用户登录：从 `accounts` 表获取密码哈希，bcrypt 验证
- 帖子 CRUD：完整的 SQL 查询，支持分页、关键词搜索（ILIKE）、分类过滤
- 软删除：所有删除操作设置 `deleted_at`，查询自动过滤

### 2. 真实 JWT 认证

替换之前的模拟 Token，实现完整的 JWT 签发/验证流程：

```
注册 → bcrypt 哈希密码 → 存入 PostgreSQL
登录 → 验证密码 → 签发 JWT Access Token (2h) + Refresh Token (30d)
请求 → Authorization: Bearer <token> → JWT 中间件验证 → 注入 user_id
```

**认证中间件保护的接口：**
- `GET /api/v1/auth/me` — 获取当前用户
- `PUT /api/v1/users/:id` — 更新用户信息
- `POST /api/v1/threads` — 创建帖子
- `PUT /api/v1/threads/:id` — 更新帖子
- `DELETE /api/v1/threads/:id` — 删除帖子

### 3. 内存模式回退

如果 PostgreSQL 不可用，系统自动回退到内存模式（兼容 v0.1.0-dev），无需任何配置即可运行。

## 架构变化

```
v0.1.3-dev:
  Go Server → 内存存储（map） + 模拟 Token

v0.1.4-dev:
  Go Server → PostgreSQL 持久化 + 真实 JWT 认证
           ↘ 内存存储（PostgreSQL 不可用时自动回退）
```

### 依赖注入链

```
config.Load()
    ↓
database.New(dsn) → pgxpool.Pool
    ↓
PgUserRepository(pool)  ──→  UserRepository 接口
PgThreadRepository(pool) ──→  ThreadRepository 接口
    ↓
auth.NewJWTManager(config)
    ↓
UserService(userRepo, jwtMgr, pgUserRepo)
ThreadService(threadRepo)
    ↓
UserHandler(userSvc)
ThreadHandler(threadSvc)
    ↓
Gin Router + Middleware
```

## API 变更

### 登录响应变更

```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "user": { "id": "...", "username": "jack", ... },
    "access_token": "eyJhbGciOiJIUzI1NiIs...",    // ← 真实 JWT Token（之前是模拟的）
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",   // ← 真实 Refresh Token
    "token_type": "Bearer",
    "expires_in": 7200
  }
}
```

### 认证方式变更

```bash
# 之前（v0.1.0 ~ v0.1.3）：通过 X-User-ID Header 模拟
curl -H "X-User-ID: user123" http://localhost:8080/api/v1/threads

# v0.1.4 起：使用真实 JWT Token
curl -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." http://localhost:8080/api/v1/threads
```

## 启动方式

### PostgreSQL 模式（推荐）

```bash
# 1. 确保 PostgreSQL 已运行
sudo -u postgres psql -c "SELECT 1"

# 2. 创建数据库（首次）
sudo -u postgres psql << 'SQL'
CREATE USER campusos WITH PASSWORD 'campusos_dev';
CREATE DATABASE campusos OWNER campusos;
GRANT ALL PRIVILEGES ON DATABASE campusos TO campusos;
\q
SQL

# 3. 执行迁移
cd ~/bbs/bbs01/CampusOS
PGPASSWORD=campusos_dev psql -h localhost -U campusos -d campusos -f migrations/000001_init_schema.up.sql

# 4. 启动后端
./campusos-server
# 输出: ✅ PostgreSQL 连接成功
#       🚀 CampusOS API 监听 0.0.0.0:8080

# 5. 启动前端
cd web && pnpm dev
# 访问 http://localhost:3000
```

### 内存模式（无需数据库）

```bash
# 直接启动，PostgreSQL 不可用时自动回退
./campusos-server
# 输出: ⚠️  PostgreSQL 连接失败，回退到内存模式: ...
#       🚀 CampusOS API 监听 0.0.0.0:8080
```

### 快速测试

```bash
# 注册用户（密码使用 bcrypt 哈希存储）
curl -s -X POST http://localhost:8080/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"jack","nickname":"Jack","email":"jack@example.com","password":"123456"}' | jq .

# 登录（返回真实 JWT Token）
curl -s -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"jack@example.com","password":"123456"}' | jq .

# 使用 Token 发帖
TOKEN="<从登录返回的 access_token>"
curl -s -X POST http://localhost:8080/api/v1/threads \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"title":"Hello CampusOS","content":"第一篇帖子","category_id":"cat_001"}' | jq .

# 获取当前用户（JWT 认证）
curl -s http://localhost:8080/api/v1/auth/me \
  -H "Authorization: Bearer $TOKEN" | jq .
```
