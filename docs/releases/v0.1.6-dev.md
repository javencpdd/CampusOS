# CampusOS v0.1.6-dev 发布说明

> **版本**：v0.1.6-dev
> **发布日期**：2024-06-24
> **上一版本**：v0.1.5-dev
> **状态**：开发中

## 版本概述

v0.1.6-dev 引入了**事件总线（Event Bus）**——这是 CampusOS 插件架构的核心基础设施。实现"万物皆事件"设计理念的第一步：所有核心行为通过 Event Bus 广播，插件通过监听事件来改变系统行为。

## Event Bus 设计

### 接口定义

```go
type EventBus interface {
    Publish(ctx context.Context, event Event) error
    Subscribe(eventType string, handler EventHandler) error
    Close() error
}
```

### Event 结构（CloudEvents 1.0 子集）

```go
type Event struct {
    ID         string            `json:"id"`       // 事件唯一 ID
    Source     string            `json:"source"`    // 事件来源
    Type       string            `json:"type"`      // 事件类型: user.created
    Subject    string            `json:"subject"`   // 资源标识: user.123
    Time       string            `json:"time"`      // 时间戳
    Data       interface{}       `json:"data"`      // 事件数据
    Extensions map[string]string `json:"extensions"` // 扩展字段
}
```

### 事件类型命名规范

| 事件类型 | 触发时机 |
|:---|:---|
| `user.created` | 用户注册 |
| `user.updated` | 用户信息更新 |
| `thread.created` | 帖子创建 |
| `thread.updated` | 帖子更新 |
| `thread.deleted` | 帖子删除 |
| `post.created` | 回复创建 |
| `category.created` | 版块创建 |

### 双运行时设计

```
┌─────────────────────────────────────────────┐
│              EventBus 接口                    │
├──────────────────┬──────────────────────────┤
│  NATSEventBus    │  MemoryEventBus          │
│  (生产环境)       │  (开发/测试回退)           │
│  NATS Server     │  内存 goroutine           │
│  持久化/可靠      │  无持久化                  │
└──────────────────┴──────────────────────────┘
```

系统启动时尝试连接 NATS：
- **成功** → 使用 NATSEventBus（支持持久化、集群）
- **失败** → 回退到 MemoryEventBus（仅内存，进程重启丢失）

## 快速测试

```bash
# 启动 NATS（可选，不启动会自动回退到内存模式）
docker compose up -d nats

# 启动服务
cd ~/bbs/bbs01/CampusOS
./campusos-server

# 输出:
# ✅ NATS 连接成功: nats://localhost:4222  (NATS 可用时)
# 或
# ⚠️  NATS 连接失败，回退到内存事件总线  (NATS 不可用时)
```
