## 编码规范

### Go 编码规范

遵循 [Effective Go](https://go.dev/doc/effective_go) 标准规范，以下是额外的项目约定：

#### 命名规范

```go
// ✅ 正确
type UserService struct { ... }          // 大驼峰：导出类型
type userService struct { ... }          // 小驼峰：非导出类型
func (s *UserService) GetByID(...)       // 大驼峰：导出方法
func (s *UserService) validateInput(...) // 小驼峰：非导出方法
const MaxRetryCount = 3                  // 大驼峰：导出常量
const defaultTimeout = 30                // 小驼峰：非导出常量

// ❌ 错误
type user_service struct { ... }         // 不使用下划线
func (s *UserService) Get_by_id(...)     // 不使用下划线
```

#### 包命名

```go
// ✅ 正确：简短、小写、无下划线
package thread
package permission
package eventbus

// ❌ 错误
package thread_service    // 不使用下划线
package ThreadService     // 不使用大写
package utils             // 过于通用，避免使用
```

#### 错误处理

```go
// ✅ 正确：使用 fmt.Errorf 包装错误，保留错误链
func (s *ThreadService) GetByID(ctx context.Context, id snowflake.ID) (*Thread, error) {
    thread, err := s.repo.FindByID(ctx, id)
    if err != nil {
        if errors.Is(err, ErrNotFound) {
            return nil, ErrThreadNotFound
        }
        return nil, fmt.Errorf("get thread by id %s: %w", id, err)
    }
    return thread, nil
}

// ❌ 错误
thread, err := s.repo.FindByID(ctx, id)
if err != nil {
    return nil, err  // 丢失上下文信息
}
```

#### 接口设计

```go
// ✅ 正确：接口定义在消费方
// internal/community/service/thread_service.go
type ThreadRepository interface {
    FindByID(ctx context.Context, id snowflake.ID) (*Thread, error)
    Create(ctx context.Context, thread *Thread) error
    Update(ctx context.Context, thread *Thread) error
}

// ❌ 错误：接口定义在实现方
// internal/infrastructure/postgres/thread_repo.go
type ThreadRepository interface { ... }  // 不要在这里定义
```

#### 依赖注入

```go
// ✅ 正确：通过构造函数注入依赖
type ThreadService struct {
    repo      ThreadRepository
    eventBus  EventBus
    permission PermissionService
}

func NewThreadService(
    repo ThreadRepository,
    eventBus EventBus,
    permission PermissionService,
) *ThreadService {
    return &ThreadService{
        repo:      repo,
        eventBus:  eventBus,
        permission: permission,
    }
}

// ❌ 错误：使用全局变量
var globalDB *gorm.DB
var globalEventBus EventBus
```

#### 上下文传递

```go
// ✅ 正确：ctx 作为第一个参数
func (s *ThreadService) CreateThread(ctx context.Context, req CreateThreadRequest) (*Thread, error) {
    // 使用 ctx 传递 trace ID、认证信息、超时控制
    user := auth.UserFromContext(ctx)
    traceID := tracing.TraceIDFromContext(ctx)
    ...
}

// ❌ 错误
func (s *ThreadService) CreateThread(req CreateThreadRequest) (*Thread, error) {
    // 缺少 context
}
```

#### 注释规范

```go
// ThreadService 提供帖子相关的业务操作。
// 它是社区应用的核心服务之一，负责帖子的创建、编辑、删除和查询。
type ThreadService struct { ... }

// CreateThread 创建新帖子。
// 它会发布 thread.created.before 事件供插件拦截，
// 审核通过后写入数据库并发布 thread.created.after 事件。
//
// 返回以下错误：
//   - ErrCategoryNotFound: 版块不存在
//   - ErrPermissionDenied: 无权在此版块发帖
//   - ErrOperationBlocked: 插件拦截了操作
func (s *ThreadService) CreateThread(ctx context.Context, req CreateThreadRequest) (*Thread, error) {
    ...
}
```

## Git 工作流

### 分支模型

采用 Git Flow 分支管理模型：

```
main (生产分支)
 │
 ├── develop (开发分支)
 │    │
 │    ├── feature/xxx  (功能分支)
 │    ├── feature/yyy
 │    └── ...
 │
 ├── release/v1.0.0   (发布分支)
 │
 └── hotfix/xxx       (紧急修复分支)
```

### 分支命名规范

| 类型 | 命名格式 | 示例 |
|:---|:---|:---|
| 功能分支 | `feature/{模块}-{简短描述}` | `feature/plugin-wasm-runtime` |
| 修复分支 | `fix/{issue号}-{简短描述}` | `fix/123-login-token-expired` |
| 热修复 | `hotfix/{简短描述}` | `hotfix/security-xss-patch` |
| 发布分支 | `release/v{版本号}` | `release/v0.1.0` |
| 文档分支 | `docs/{简短描述}` | `docs/api-reference` |

### 分支工作流程

```
1. 从 develop 创建功能分支
   git checkout develop
   git pull origin develop
   git checkout -b feature/plugin-wasm-runtime

2. 在功能分支上开发和提交
   git add .
   git commit -m "feat(plugin): implement wasm runtime sandbox"

3. 推送到远程并创建 PR
   git push origin feature/plugin-wasm-runtime
   → 创建 PR: feature/plugin-wasm-runtime → develop

4. Code Review + CI 通过后合并到 develop

5. 发布时从 develop 创建 release 分支
   git checkout develop
   git checkout -b release/v0.1.0

6. release 分支合并到 main 并打 Tag
   git checkout main
   git merge release/v0.1.0
   git tag -a v0.1.0 -m "Release v0.1.0"
```

## Commit 规范

### Conventional Commits

严格执行 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

```
<type>(<scope>): <description>

[optional body]

[optional footer(s)]
```

### Type 类型

| Type | 说明 | 示例 |
|:---|:---|:---|
| `feat` | 新功能 | `feat(plugin): add wasm runtime` |
| `fix` | Bug 修复 | `fix(auth): fix token refresh race condition` |
| `docs` | 文档更新 | `docs(api): update thread API spec` |
| `style` | 代码格式（不影响逻辑） | `style: run gofmt` |
| `refactor` | 重构（既不修复 Bug 也不添加功能） | `refactor(eventbus): extract nats adapter` |
| `perf` | 性能优化 | `perf(cache): implement multi-level cache` |
| `test` | 测试相关 | `test(thread): add unit tests for create flow` |
| `build` | 构建系统或外部依赖 | `build: update go to 1.22` |
| `ci` | CI 配置 | `ci: add GitHub Actions workflow` |
| `chore` | 其他杂项 | `chore: update dependencies` |

### Scope 范围

| Scope | 说明 |
|:---|:---|
| `core` | 核心服务 |
| `identity` | 身份服务 |
| `permission` | 权限系统 |
| `community` | 社区应用 |
| `plugin` | 插件系统 |
| `eventbus` | 事件总线 |
| `api` | API 接口 |
| `db` | 数据库 |
| `auth` | 认证授权 |
| `ai` | AI 能力 |

### 提交示例

```bash
# 功能
git commit -m "feat(plugin): implement gRPC runtime with health check"
git commit -m "feat(community): add thread pinning feature"
git commit -m "feat(ai): implement LLM gateway with model fallback"

# 修复
git commit -m "fix(auth): prevent refresh token reuse after rotation"
git commit -m "fix(permission): correct ABAC condition evaluation"

# 重构
git commit -m "refactor(eventbus): decouple NATS from event bus interface"

# 破坏性变更（在 footer 中标记 BREAKING CHANGE）
git commit -m "feat(api)!: change pagination response format

BREAKING CHANGE: pagination fields renamed from total/limit to total/page_size"
```

## Code Review 规范

### Review 检查清单

| 维度 | 检查项 |
|:---|:---|
| **功能正确性** | 代码是否正确实现了需求？ |
| **边界条件** | 是否处理了空值、零值、负值等边界情况？ |
| **错误处理** | 错误是否被正确处理和传播？ |
| **安全性** | 是否存在 SQL 注入、XSS、信息泄露等安全问题？ |
| **性能** | 是否存在 N+1 查询、内存泄漏、不必要的锁？ |
| **可测试性** | 代码是否易于编写单元测试？ |
| **命名规范** | 变量、函数、类型命名是否清晰准确？ |
| **注释** | 复杂逻辑是否有足够的注释？ |
| **依赖管理** | 新增的第三方依赖是否必要和安全？ |

### Review 流程

1. 创建 PR 后自动触发 CI 检查
2. 至少需要 **1 位** 核心开发者 Approve
3. CI 全部通过后方可合并
4. 使用 **Squash Merge** 合并到 develop

## 测试规范

### 测试分层

```
┌─────────────────────────────────┐
│       E2E 测试 (test/e2e/)      │  少量，验证完整流程
├─────────────────────────────────┤
│    集成测试 (test/integration/)  │  中量，验证服务间协作
├─────────────────────────────────┤
│    单元测试 (*_test.go)          │  大量，验证函数逻辑
└─────────────────────────────────┘
```

### 单元测试规范

```go
// 文件命名：与被测文件同目录，以 _test.go 结尾
// internal/community/service/thread_service_test.go

func TestThreadService_CreateThread(t *testing.T) {
    // Arrange - 准备
    mockRepo := NewMockThreadRepository()
    mockEventBus := NewMockEventBus()
    svc := NewThreadService(mockRepo, mockEventBus, nil)
    
    req := CreateThreadRequest{
        Title:      "Test Thread",
        Content:    "This is a test thread content",
        AuthorID:   snowflake.Generate(),
        CategoryID: snowflake.Generate(),
    }
    
    // Act - 执行
    thread, err := svc.CreateThread(context.Background(), req)
    
    // Assert - 断言
    require.NoError(t, err)
    assert.Equal(t, req.Title, thread.Title)
    assert.Equal(t, req.Content, thread.Content)
    assert.Equal(t, req.AuthorID, thread.AuthorID)
    mockEventBus.AssertCalled(t, "PublishAsync", mock.Anything, mock.Anything)
}

func TestThreadService_CreateThread_CategoryNotFound(t *testing.T) {
    // 测试异常路径
    mockRepo := NewMockThreadRepository()
    mockRepo.On("FindByID", mock.Anything, mock.Anything).Return(nil, ErrNotFound)
    
    svc := NewThreadService(mockRepo, nil, nil)
    
    _, err := svc.CreateThread(context.Background(), CreateThreadRequest{
        CategoryID: snowflake.Generate(),
    })
    
    require.Error(t, err)
    assert.ErrorIs(t, err, ErrCategoryNotFound)
}
```

## 安全规范

### 代码安全

| 规范 | 说明 |
|:---|:---|
| **输入验证** | 所有外部输入必须验证和消毒 |
| **SQL 注入** | 禁止字符串拼接 SQL，必须使用参数化查询 |
| **XSS 防护** | 所有用户输入在输出时必须转义 |
| **密码存储** | 使用 bcrypt 进行密码哈希，禁止明文存储 |
| **敏感信息** | 禁止在日志中打印密码、Token、密钥等敏感信息 |
| **依赖审计** | 定期使用 `govulncheck` 检查依赖漏洞 |

### 安全中间件

```go
// 全局安全中间件
router.Use(middleware.SecurityHeaders())
router.Use(middleware.CORS(corsConfig))
router.Use(middleware.XSSProtection())
router.Use(middleware.CSRFProtection())
router.Use(middleware.SQLInjectionProtection())
router.Use(middleware.RateLimit(rateLimitConfig))
```
