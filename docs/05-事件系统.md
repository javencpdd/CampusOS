## 设计目标

| 目标 | 描述 |
|:---|:---|
| **万物皆事件** | 所有核心行为必须通过 Event Bus 广播 |
| **标准化** | 采用 CNCF CloudEvents 1.0 规范 |
| **高可靠** | At-least-once 投递保障，事件不丢失 |
| **高性能** | 单节点支持百万级消息/秒 |
| **可观测** | 全链路事件追踪与监控 |
| **可回放** | 支持事件重放，便于调试与数据恢复 |

## CloudEvents 规范

CampusOS 全面采用 [CNCF CloudEvents 1.0](https://cloudevents.io/) 规范作为标准事件封装格式。

### 事件结构

```json
{
  "specversion": "1.0",
  "id": "evt_01HXYZ1234567890",
  "source": "campusos://community/thread-service",
  "type": "io.campusos.thread.created.v1",
  "datacontenttype": "application/json",
  "dataschema": "https://api.campusos.io/schemas/events/thread.created.v1.json",
  "subject": "thread:1234567890",
  "time": "2024-01-15T08:30:00.123456Z",
  "traceid": "abc123def456",
  "data": {
    "thread_id": "1234567890",
    "title": "欢迎来到 CampusOS 社区",
    "content": "这是你的第一篇帖子...",
    "author_id": "9876543210",
    "category_id": "5555555555",
    "status": "published",
    "tags": ["announcement", "welcome"]
  }
}
```

### 必填字段说明

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `specversion` | string | CloudEvents 规范版本，固定为 `"1.0"` |
| `id` | string | 事件唯一 ID（全局唯一，使用雪花算法） |
| `source` | string | 事件来源标识（URI 格式） |
| `type` | string | 事件类型（反向域名命名） |
| `time` | string | 事件产生时间（RFC 3339 格式） |

### 扩展字段

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| `traceid` | string | 链路追踪 ID（用于跨服务关联） |
| `subject` | string | 事件关联的资源标识 |
| `datacontenttype` | string | 数据内容类型 |
| `dataschema` | string | 数据 JSON Schema 地址 |

### Go 实现

```go
// CloudEvent CloudEvents 1.0 标准事件结构
type CloudEvent struct {
    SpecVersion     string      `json:"specversion"`
    ID              string      `json:"id"`
    Source          string      `json:"source"`
    Type            string      `json:"type"`
    DataContentType string      `json:"datacontenttype,omitempty"`
    DataSchema      string      `json:"dataschema,omitempty"`
    Subject         string      `json:"subject,omitempty"`
    Time            time.Time   `json:"time"`
    TraceID         string      `json:"traceid,omitempty"`
    Data            interface{} `json:"data"`
}

// NewCloudEvent 创建新的 CloudEvent
func NewCloudEvent(source, eventType string, data interface{}) *CloudEvent {
    return &CloudEvent{
        SpecVersion:     "1.0",
        ID:              snowflake.Generate().String(),
        Source:          source,
        Type:            eventType,
        DataContentType: "application/json",
        Time:            time.Now().UTC(),
        TraceID:         tracing.CurrentTraceID(),
        Data:            data,
    }
}
```

## 事件类型与模式

### `.before` 模式（同步拦截）

`.before` 事件在核心操作**执行前**同步触发，允许插件阻断或修改操作。

**特征：**

- 同步执行，阻塞主流程
- 允许返回 ALLOW / BLOCK / MODIFY
- 有严格超时限制（默认 5 秒）
- 用于内容审核、权限二次校验等

**流程：**

```
核心服务即将执行操作
    │
    ▼
发布 .before 事件到 Event Bus
    │
    ▼
Plugin Manager 同步等待所有订阅插件处理完毕
    │
    ├── 任一返回 BLOCK → 阻断操作，返回错误
    ├── 有返回 MODIFY  → 使用修改后的 payload
    └── 全部返回 ALLOW → 继续执行核心操作
    │
    ▼
核心服务执行操作
    │
    ▼
发布 .after 事件
```

### `.after` 模式（异步通知）

`.after` 事件在核心操作**执行后**异步触发，用于副作用处理。

**特征：**

- 异步执行，不阻塞主流程
- 不关心处理结果
- 用于通知推送、统计分析、数据同步等

**流程：**

```
核心服务执行操作完成
    │
    ▼
发布 .after 事件到 Event Bus
    │
    ▼
Plugin Manager 并行通知所有订阅插件（Fire and Forget）
    │
    ├── Plugin A: 发送通知
    ├── Plugin B: 更新统计
    └── Plugin C: 同步到外部系统
```

### 完整事件生命周期

```
                    ┌─────────────────────────────────────────────┐
                    │           thread.created 完整生命周期         │
                    │                                              │
                    │  ┌──────────────────────┐                   │
                    │  │ 1. thread.created    │                   │
                    │  │    .before           │ ← 同步拦截点       │
                    │  │                      │                   │
                    │  │  插件可以:            │                   │
                    │  │  - 审核内容 → BLOCK   │                   │
                    │  │  - 过滤敏感词 → MODIFY│                   │
                    │  │  - 放行 → ALLOW       │                   │
                    │  └──────────┬───────────┘                   │
                    │             │                                │
                    │             ▼ ALLOW                          │
                    │  ┌──────────────────────┐                   │
                    │  │ 2. 核心服务执行操作    │                   │
                    │  │    (写入数据库)       │                   │
                    │  └──────────┬───────────┘                   │
                    │             │                                │
                    │             ▼ 操作完成                        │
                    │  ┌──────────────────────┐                   │

                    │  │    .after            │ ← 异步通知         │
                    │  │                      │                   │
                    │  │  插件可以:            │                   │
                    │  │  - 发送通知           │                   │
```

## 事件发布机制

### 发布接口

```go
// EventBus 事件总线接口
type EventBus interface {
    // Publish 发布事件
    // .before 事件：同步等待所有消费者处理完成
    // .after 事件：异步发布，不等待
    Publish(ctx context.Context, event *CloudEvent) error
    
    // PublishSync 同步发布事件（等待所有消费者确认）
    PublishSync(ctx context.Context, event *CloudEvent) error
    
    // PublishAsync 异步发布事件（Fire and Forget）
    PublishAsync(ctx context.Context, event *CloudEvent) error
    
    // Subscribe 订阅事件
    Subscribe(ctx context.Context, eventType string, handler EventHandler, opts ...SubscribeOption) error
    
    // Unsubscribe 取消订阅
    Unsubscribe(ctx context.Context, eventType string, consumerID string) error
    
    // Close 关闭事件总线
    Close() error
}
```

### 发布示例

```go
// 在帖子服务中发布事件
func (s *ThreadService) CreateThread(ctx context.Context, req CreateThreadRequest) (*Thread, error) {
    // 1. 发布 .before 事件（同步）
    beforeEvent := NewCloudEvent(
        "campusos://community/thread-service",
        "campusos.community.thread.created.before.v1",
        &ThreadCreatedData{
            Title:      req.Title,
            Content:    req.Content,
            AuthorID:   req.AuthorID,
            CategoryID: req.CategoryID,
        },
    )
    
    result, err := s.eventBus.PublishSync(ctx, beforeEvent)
    if err != nil {
        return nil, fmt.Errorf("publish before event: %w", err)
    }
    
    // 2. 检查是否有插件阻断
    if result.Blocked {
        return nil, ErrOperationBlocked(result.Reason)
    }
    
    // 3. 使用可能被插件修改后的 payload
    data := result.ModifiedData.(*ThreadCreatedData)
    
    // 4. 执行核心业务逻辑
    thread := &Thread{
        ID:         snowflake.Generate(),
        Title:      data.Title,
        Content:    data.Content,
        AuthorID:   data.AuthorID,
        CategoryID: data.CategoryID,
        Status:     ThreadStatusPublished,
        CreatedAt:  time.Now(),
    }
    
    if err := s.repo.Create(ctx, thread); err != nil {
        return nil, err
    }
    
    // 5. 发布 .after 事件（异步）
    afterEvent := NewCloudEvent(
        "campusos://community/thread-service",
        "campusos.community.thread.created.after.v1",
        &ThreadCreatedAfterData{
            ThreadID:   thread.ID,
            Title:      thread.Title,
            AuthorID:   thread.AuthorID,
            CategoryID: thread.CategoryID,
        },
    )
    s.eventBus.PublishAsync(ctx, afterEvent)
    
    return thread, nil
}
```

## 事件消费机制

### 消费者接口

```go
// EventHandler 事件处理函数类型
type EventHandler func(ctx context.Context, event *CloudEvent) (*EventResult, error)

// EventResult 事件处理结果
type EventResult struct {
    Action         EventAction  // ALLOW / BLOCK / MODIFY
    ModifiedData   interface{}  // action=MODIFY 时的修改后数据
    Reason         string       // 操作原因
}

type EventAction string

const (
    EventActionAllow  EventAction = "ALLOW"
    EventActionBlock  EventAction = "BLOCK"
    EventActionModify EventAction = "MODIFY"
)
```

### 订阅示例

```go
// 内容审核插件订阅 .before 事件
func RegisterModerationPlugin(eventBus EventBus, aiGateway AIClient) error {
    return eventBus.Subscribe(
        context.Background(),
        "campusos.community.thread.created.before.v1",
        func(ctx context.Context, event *CloudEvent) (*EventResult, error) {
            data := event.Data.(*ThreadCreatedData)
            
            // AI 内容审核
            result, err := aiGateway.ChatCompletion(ctx, []Message{
                {Role: "system", Content: "审核内容是否包含敏感信息"},
                {Role: "user", Content: data.Content},
            })
            if err != nil {
                // AI 失败时放行
                return &EventResult{Action: EventActionAllow}, nil
            }
            
            if !result.Safe {
                return &EventResult{
                    Action: EventActionBlock,
                    Reason: "内容审核未通过: " + result.Reason,
                }, nil
            }
            
            return &EventResult{Action: EventActionAllow}, nil
        },
        WithConsumerID("content-moderator"),
        WithPriority(100),
        WithTimeout(5*time.Second),
    )
}

// 通知服务订阅 .after 事件
func RegisterNotificationPlugin(eventBus EventBus, notificationSvc NotificationService) error {
    return eventBus.Subscribe(
        context.Background(),
        "campusos.community.thread.created.after.v1",
        func(ctx context.Context, event *CloudEvent) (*EventResult, error) {
            data := event.Data.(*ThreadCreatedAfterData)
            
            // 获取帖子作者的关注者
            followers, _ := userService.GetFollowers(ctx, data.AuthorID)
            
            // 发送通知给关注者
            for _, follower := range followers {
                notificationSvc.Send(ctx, Notification{
                    UserID:   follower.ID,
                    Type:     "thread_created",
                    Title:    "你关注的人发布了新帖子",
                    Content:  data.Title,
                    ThreadID: data.ThreadID,
                })
            }
            
            return &EventResult{Action: EventActionAllow}, nil
        },
        WithConsumerID("notification-service"),
        WithPriority(200),
    )
}
```

## 可靠性保障

### 投递保证

CampusOS 提供 **At-least-once** 投递保证：

```
┌─────────────────────────────────────────────────────────────┐
│                    At-least-once 投递                         │
│                                                              │
│  1. 事件发布到 NATS JetStream（持久化存储）                    │
│  2. 消费者处理完成后发送 ACK                                  │
│  3. 未收到 ACK 的事件会被重新投递                              │
│  4. 消费者需要实现幂等处理（基于事件 ID 去重）                  │
│                                                              │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐            │
│  │ Publisher │────→│  NATS    │────→│ Consumer │            │
│  │          │     │ JetStream│     │          │            │
│  │          │     │          │     │ 处理完成  │            │
│  │          │     │          │←────│ 发送 ACK │            │
│  └──────────┘     └──────────┘     └──────────┘            │
│                                                              │
│  如果 ACK 未收到（超时/消费者崩溃）:                            │
│  NATS 会在 ack_wait 时间后重新投递事件                         │
└─────────────────────────────────────────────────────────────┘
```

### 幂等消费

```go
// 幂等消费实现
type IdempotentConsumer struct {
    store    KeyValueStore  // Redis
    delegate EventHandler
}

func (c *IdempotentConsumer) Handle(ctx context.Context, event *CloudEvent) (*EventResult, error) {
    // 检查事件是否已处理
    key := fmt.Sprintf("event:processed:%s", event.ID)
    
    processed, err := c.store.Exists(ctx, key)
    if err != nil {
        return nil, fmt.Errorf("check idempotency: %w", err)
    }
    
    if processed {
        // 事件已处理，直接返回成功
        return &EventResult{Action: EventActionAllow}, nil
    }
    
    // 处理事件
    result, err := c.delegate(ctx, event)
    if err != nil {
        return nil, err
    }
    
    // 标记事件已处理（设置 TTL 防止无限增长）
    c.store.Set(ctx, key, "1", 24*time.Hour)
    
    return result, nil
}
```

## 死信队列

### 设计

无法处理的事件（超过最大重试次数）会被转入死信队列（Dead Letter Queue, DLQ）。

```
正常事件流
    │
    ▼
事件消费失败
    │
    ▼
重试 5 次（指数退避）
    │
    ▼
仍然失败
    │
    ▼
┌─────────────────────────────────────────┐
│            Dead Letter Queue             │
│                                          │
│  存储:                                    │
│  - 原始事件                               │
│  - 最后一次错误信息                        │
│  - 重试次数                               │
│  - 最后重试时间                            │
│                                          │
│  处理方式:                                 │
│  1. 管理后台查看 DLQ 列表                  │
│  2. 手动重试                              │
│  3. 忽略并删除                            │
│  4. 修复后批量重放                         │
└─────────────────────────────────────────┘
```

### DLQ 数据结构

```go
// DeadLetter 死信事件
type DeadLetter struct {
    ID              string       `json:"id"`
    OriginalEvent   *CloudEvent  `json:"original_event"`   // 原始事件
    ConsumerID      string       `json:"consumer_id"`      // 消费者标识
    Error           string       `json:"error"`            // 最后一次错误信息
    RetryCount      int          `json:"retry_count"`      // 重试次数
    LastRetryAt     time.Time    `json:"last_retry_at"`    // 最后重试时间
    CreatedAt       time.Time    `json:"created_at"`       // 进入 DLQ 时间
    Status          string       `json:"status"`           // pending / retried / ignored
}
```

### 事件回放

```go
// 回放指定时间范围内的事件
func (eb *EventBus) Replay(ctx context.Context, opts ReplayOptions) error {
    // 创建临时消费者
    consumer, err := eb.js.CreateConsumer(ctx, "CAMPUSOS_EVENTS", &nats.ConsumerConfig{
        DeliverPolicy: nats.DeliverByStartTimePolicy,
        OptStartTime:  opts.StartTime,
        FilterSubject: opts.Subject,  // 可选: 过滤特定主题
    })
    if err != nil {
        return err
    }
    
    // 消费事件
    for {
        msgs, err := consumer.Fetch(100, nats.MaxWait(5*time.Second))
        if err == nats.ErrTimeout {
            break
        }
        if err != nil {
            return err
        }
        
        for _, msg := range msgs {
            event := &CloudEvent{}
            if err := json.Unmarshal(msg.Data, event); err != nil {
                continue
            }
            
            // 调用回放处理器
            if err := opts.Handler(ctx, event); err != nil {
                // 记录错误但继续回放
                log.Error("replay handler error", "event_id", event.ID, "error", err)
            }
            
            msg.Ack()
        }
    }
    
    return nil
}
```
